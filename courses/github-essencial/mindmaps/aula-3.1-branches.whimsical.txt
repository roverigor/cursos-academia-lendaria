Branches
	O Que Sao Branches
		Linha independente de desenvolvimento
		Analogia
			main = Rodovia principal (estavel)
			Branches = Vias paralelas (construcao)
			Merge = Retornar para rodovia
		Isolamento
			Mudancas nao afetam main
	Por Que Usar Branches
		1. Protecao codigo estavel
			main sempre funcional
			Experimente em branches
		2. Trabalho paralelo
			Alice, Bob, Carol
			Simultaneamente sem interferencia
		3. Code Review
			feature → PR → Review → Merge
		4. Experimentos seguros
			Se funcionar: merge
			Se falhar: delete branch
	Comandos Basicos
		Ver branches
			git branch (locais)
			git branch -a (todas)
			* indica branch atual
		Criar branch
			git branch nome
			git checkout -b nome
			git switch -c nome
		Mudar de branch
			git checkout main
			git switch main
		Deletar branch
			git branch -d (se merged)
			git branch -D (forcar)
			git push origin --delete (remota)
	Convencoes Nomenclatura
		Formato: type/description
		Types
			feature/
				Nova funcionalidade
			fix/ ou bugfix/
				Correcao de bug
			hotfix/
				Correcao urgente
			refactor/
				Refatoracao
			docs/
				Documentacao
			test/
				Testes
			chore/
				Manutencao
		Boas praticas
			kebab-case
			Descritivo mas conciso
			Prefixo com tipo
			Sem espacos
			Em ingles
		Exemplos bons
			feature/dark-mode-toggle
			fix/navbar-mobile-overflow
			hotfix/critical-security
		Evitar
			branch1 (sem contexto)
			test (generico)
			my-branch (pessoal)
	Workflow de Branch
		1. Atualizar main
			git checkout main
			git pull
		2. Criar branch
			git checkout -b feature/nome
		3. Trabalhar
			Editar arquivos
			git add .
			git commit -m
		4. Push branch
			git push -u origin feature/nome
		5. Abrir Pull Request
		6. Apos merge
			git checkout main
			git pull
			git branch -d feature/nome
	Branch main vs master
		Historia
			Git usava master ate 2020
		Mudanca
			GitHub mudou para main
			Mais inclusivo
		Hoje
			Novos: main
			Antigos: podem ter master
		Renomear
			git branch -m master main
			git push -u origin main
			Settings → Default branch
			git push origin --delete master
	Sincronizando Branches
		Atualizar com main
			Metodo 1: Merge
				git merge main
				Cria merge commit
			Metodo 2: Rebase
				git rebase main
				Historico linear
		Push de branch
			Primeira vez: -u
			Seguintes: git push
		Pull de branch
			git pull
	Ver Diferencas
		Commits diferentes
			git log main..feature/login
		Arquivos diferentes
			git diff main..feature/login
		Apenas nomes
			git diff --name-only
	Estrategias Branch
		Git Flow
			main (producao)
			develop (integracao)
			feature/*, release/*, hotfix/*
			Times grandes
		GitHub Flow
			main (producao)
			feature/*, fix/*, hotfix/*
			Deploy continuo
			Recomendado iniciar
			Regras
				main sempre estavel
				Branch para qualquer mudanca
				Merge via PR
				Delete apos merge
	Troubleshooting
		Please commit or stash
			git commit ou git stash
		Branch already exists
			git branch -d nome-existente
