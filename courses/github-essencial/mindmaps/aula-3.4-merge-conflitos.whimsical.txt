Merge + Conflitos
	O Que e Merge
		Integrar branches
		Commit de merge
			Junta historico
		Quando acontece
			PR aprovado
			Sincronizar branch
			git pull
	Merge Sem Conflitos
		Fast-Forward
			main nao mudou
			Apenas avanca ponteiro
			Historico linear
		Comando
			git checkout main
			git merge feature/branch
	Merge com Conflitos
		Cenario
			Mesma parte do mesmo arquivo
			Modificada diferentemente
		Git nao sabe
			Qual mudanca manter
			Voce decide
	Como Conflitos Acontecem
		Exemplo pratico
			Estado inicial
				function greet(name)
			Branch A (voce)
				"Good morning, " + name
			Branch B (colega)
				"Hey " + name
			Conflito!
				Ambos linha 2
	Detectando Conflitos
		Ao tentar merge
			git merge feature/branch
			CONFLICT in arquivo.js
			Automatic merge failed
		git status
			You have unmerged paths
			both modified: arquivo.js
	Anatomia Conflito
		Estrutura
			<<<<<<< HEAD
			[Versao atual]
			=======
			[Versao outra branch]
			>>>>>>> branch-name
		Marcadores
			<<<<<<< inicio conflito
			======= separador
			>>>>>>> fim conflito
	Resolvendo Conflitos
		Passo 1: Identificar
			git status
			Lista "both modified"
		Passo 2: Escolher versao
			Opcao 1: Manter sua
			Opcao 2: Manter deles
			Opcao 3: Combinar ambas
			Opcao 4: Solucao nova
		Passo 3: Testar
			npm test
			node index.js
			NUNCA commite sem testar
		Passo 4: Marcar resolvido
			git add arquivo.js
		Passo 5: Complete merge
			git commit
			Mensagem pre-preenchida
		Passo 6: Push
			git push
	Ferramentas Merge
		VS Code
			Accept Current Change
			Accept Incoming Change
			Accept Both Changes
			Compare Changes
		Git Merge Tool
			git config merge.tool vscode
			git mergetool
		Meld, KDiff3
			Ferramentas visuais
	Tipos de Conflito
		1. Conflito conteudo
			Mesma linha modificada
			Editar manualmente
		2. Conflito rename
			Arquivo renomeado diferente
			Escolher um nome
		3. Conflito delete
			Um deletou, outro modificou
			Decidir manter ou deletar
	Prevenindo Conflitos
		1. Sync frequente
			git pull origin main diario
		2. PRs pequenos
			Menos linhas = menos conflito
		3. Comunicar mudancas
			Avisar refactors grandes
		4. Dividir responsabilidades
			Cada dev area diferente
		5. Commits atomicos
			Faceis de resolver
	Abortando Merge
		Comando
			git merge --abort
		Quando usar
			Conflitos complexos
			Quer revisar antes
			Merge errado
	Conflitos em Pull Request
		GitHub mostra
			"This branch has conflicts"
		Opcao 1: Local (recomendado)
			git checkout main
			git pull
			git checkout feature/branch
			git merge main
			Resolver conflitos
			git push
		Opcao 2: GitHub web
			Resolve conflicts
			Editar interface
			Limitado: 1-2 arquivos simples
	Merge vs Rebase
		Merge
			Preserva historico
			Cria merge commit
		Rebase
			Reaplica commits
			Historico linear
		Quando usar
			Feature pessoal: Rebase
			Branch compartilhada: Merge
			Sincronizar: Rebase
			Finalizar PR: Merge/Squash
	Rebase com Conflitos
		Comando
			git rebase main
		Se conflito
			1. Resolver
			2. git add
			3. git rebase --continue
			4. Repetir ate terminar
		Abortar
			git rebase --abort
	Troubleshooting
		Unstaged changes
			git stash
			git merge
			git stash pop
		Status mostra conflito
			Certificar git add
			Ver "All conflicts fixed"
		Merge commit acidental
			git reset --soft HEAD~1
