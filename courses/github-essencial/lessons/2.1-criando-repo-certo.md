## GPS DA AULA

Ao final desta aula, voce vai:
- Ter um repositorio organizado no GitHub
- Entender os arquivos basicos de um projeto
- Saber baixar o projeto para seu computador

### ORIGEM (De onde voce vem)
Voce provavelmente:
- Tem Git instalado e conexao configurada (Modulo 1)
- Quer comecar a organizar seus projetos

### ROTA (Como vamos chegar)
1. Criar repositorio no GitHub

### [POR QUE ORGANIZACAO IMPORTA] - 1 minuto

[SLIDE 1 - Comparativo]

**[FALA]**

> Imagine duas pastas no seu computador.
>
> A primeira tem arquivos jogados: "documento1.txt", "versao_nova.docx", "backup_antigo"...
>
> A segunda tem uma estrutura clara: uma descricao do que tem ali, pastas organizadas por assunto, tudo no lugar.
>
> [Pausa]
>
> Qual voce prefere abrir quando precisa encontrar algo?
>
> [SLIDE 2 - Projeto organizado]
>
> Um repositorio bem organizado economiza tempo.
>
> Quando voce ou outra pessoa precisar encontrar algo, vai saber exatamente onde procurar.
>
> Vamos criar assim desde o comeco.

---

### [OS ARQUIVOS INICIAIS] - 1 minuto 30 segundos

[SLIDE 3 - Arquivos basicos]

**[FALA]**

> Todo repositorio bem organizado comeca com alguns arquivos basicos. Vou explicar cada um.
>
> **README.md**
>
> E o arquivo de apresentacao do projeto. Quando alguem abre seu repositorio, e a primeira coisa que aparece.
>
> Pense como um indice: explica o que tem ali, para que serve, como usar.
>
> O ".md" significa Markdown, que e uma forma simples de formatar texto.
>
> [SLIDE 4 - .gitignore]
>
> **.gitignore**
>
> Esse arquivo diz ao Git o que ele deve ignorar. Ou seja, quais arquivos NAO devem ser salvos.
>
> Por que isso e importante?
>
> Alguns arquivos nao devem ir para a nuvem. Por exemplo: arquivos temporarios, senhas, configuracoes pessoais do seu computador.
>
> O .gitignore protege voce de subir algo que nao deveria.
>
> [SLIDE 5 - LICENSE]
>
> **LICENSE**
>
> Esse arquivo define as regras de uso. Quem pode usar, como pode usar.
>
> Se voce criar algo que quer compartilhar, precisa de uma licenca.
>
> A mais comum e a MIT, que basicamente diz: "pode usar, so mantenha os creditos".

---

### [CRIANDO O REPOSITORIO] - 2 minutos

[SLIDE 6 - GitHub]

**[FALA]**

> Vamos criar. Acesse github.com e faca login.
>
> Clique no "+" no canto superior direito e depois em "New repository".
>
> [SLIDE 7 - Formulario]
>
> **Nome do repositorio:**
>
> Escolha um nome simples e descritivo. Use letras minusculas e tracos no lugar de espacos.
>
> Por exemplo: "meus-projetos" ou "documentos-empresa" ou "automacoes".
>
> **Descricao:**
>
> Escreva em uma frase o que vai guardar ali. Por exemplo: "Central de documentos e projetos".
>
> **Visibilidade:**
>
> - **Private** (privado): so voce ve. Use para coisas pessoais ou da empresa.
> - **Public** (publico): qualquer pessoa pode ver. Use se quiser compartilhar.
>
> [Pausa]
>
> Agora a parte importante. Marque essas opcoes:
>
> - **Add a README file** - cria o arquivo de apresentacao
> - **Add .gitignore** - escolha "None" por enquanto, ou selecione se souber a linguagem que vai usar
>
> Clique em "Create repository".
>
> Pronto. Seu repositorio foi criado.

---

### [BAIXANDO PARA O COMPUTADOR] - 1 minuto 30 segundos

[SLIDE 8 - Botao Code]

**[FALA]**

> Agora vamos baixar o repositorio para o seu computador. Isso se chama "clonar".
>
> Na pagina do seu repositorio no GitHub, procure o botao verde escrito "Code".
>
> Clique nele. Vai aparecer algumas opcoes.
>
> Clique na aba "SSH" (nao em HTTPS). Voce configurou isso na aula anterior.
>
> Copie o endereco que aparece. Comeca com "git@github.com:".
>
> [SLIDE 9 - Terminal]
>
> Abra o terminal no seu computador.
>
> Navegue ate a pasta onde quer guardar o projeto. Por exemplo, sua pasta de Documentos.
>
> Digite:
>
> ```
> git clone [cole o endereco aqui]
> ```
>
> Por exemplo:
>
> ```
> git clone git@github.com:seu-usuario/meus-projetos.git
> ```
>
> Aperte Enter.
>
> [Pausa]
>
> O Git vai baixar o repositorio. Vai criar uma pasta com o nome do projeto.
>
> Entre na pasta:
>
> ```
> cd meus-projetos
> ```
>
> Liste os arquivos:
>
> ```
> ls
> ```
>
> Voce deve ver o arquivo README.md que foi criado.

---

### [VERIFICANDO A CONEXAO] - 30 segundos

[SLIDE 10 - Verificacao]

**[FALA]**

> Para confirmar que tudo esta conectado corretamente, digite:
>
> ```
> git remote -v
> ```
>
> Voce deve ver algo assim:
>
> ```
> origin  git@github.com:seu-usuario/meus-projetos.git (fetch)
> origin  git@github.com:seu-usuario/meus-projetos.git (push)
> ```
>
> Se aparecer "git@github.com", esta usando a conexao segura (SSH). Perfeito.
>
> Se aparecer "https://", clonou de forma diferente. Funciona, mas vai pedir senha. Se preferir SSH, delete a pasta e clone novamente pela aba SSH.
