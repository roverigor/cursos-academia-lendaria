### [O QUE E GIT] - 1 minuto 30 segundos

[SLIDE 1 - Git = Programa no Computador]

**[FALA]**

> Git e um programa que voce instala no seu computador.
>
> Assim como voce instala Word, Excel, ou qualquer outro aplicativo.
>
> A diferenca e que o Git nao tem janelas ou botoes. Ele funciona no terminal - aquela tela preta onde voce digita comandos.
>
> [Pausa]
>
> O que o Git faz?
>
> Ele guarda o historico de tudo que voce muda nos seus arquivos.
>
> E como ter um "Ctrl+Z" que funciona para sempre. Nao importa se foi ontem, semana passada, ou mes passado.
>
> [Pausa]
>
> Mudou um arquivo e ficou pior? O Git te deixa voltar.
>
> Quer ver o que mudou desde a ultima versao? O Git mostra.
>
> Deletou algo sem querer? O Git recupera.
>
> [Pausa]
>
> E tem uma coisa importante: o Git funciona no seu computador, sem precisar de internet.
>
> Voce pode estar num aviao, na praia, em qualquer lugar sem wifi.
>
> O Git continua funcionando porque roda localmente, na sua maquina.

---

### [O QUE E GITHUB] - 1 minuto 30 segundos

[SLIDE 2 - GitHub = Servico na Nuvem]

**[FALA]**

> GitHub e diferente. GitHub e um site, um servico na internet.
>
> Assim como Google Drive ou Dropbox guardam seus arquivos na nuvem, o GitHub guarda seus projetos com todo o historico que o Git criou.
>
> [Pausa]
>
> Por que usar GitHub?
>
> **Backup seguro:** Se seu computador estragar, seus projetos estao salvos no GitHub.
>
> **Acesso de qualquer lugar:** Voce pode acessar de outro computador, do celular, de onde quiser.
>
> **Colaboracao:** Outras pessoas podem trabalhar com voce no mesmo projeto.
>
> [Pausa]
>
> Aqui vai um ponto importante que muita gente entende errado:
>
> Voce nao precisa de internet o tempo todo para usar GitHub.
>
> Como funciona na pratica? Voce baixa o projeto do GitHub para o seu computador. Trabalha offline o tempo que precisar, usando o Git. Quando tiver internet, voce sincroniza as mudancas de volta para o GitHub.
>
> E como baixar um email para ler offline e responder depois quando tiver conexao.

---

### [COMO ELES TRABALHAM JUNTOS] - 1 minuto

[SLIDE 3 - Diagrama Visual]

**[FALA]**

> Entao, resumindo:
>
> **Git** = programa no seu computador que guarda o historico
>
> **GitHub** = servico na nuvem que guarda uma copia de tudo
>
> [Pausa]
>
> Vou dar uma analogia que ajuda a lembrar:
>
> Pensa em fotos.
>
> **Git** e como a camera do seu celular. Ela tira e guarda as fotos no seu aparelho.
>
> **GitHub** e como o Google Fotos. Ele guarda suas fotos na nuvem, para voce ter backup e acessar de qualquer lugar.
>
> Voce pode tirar fotos sem Google Fotos? Pode. A camera funciona sozinha.
>
> Mas se voce quiser backup e acesso de qualquer lugar, ai voce usa o Google Fotos junto.
>
> [Pausa]
>
> Com Git e GitHub e a mesma logica.
>
> Git funciona sozinho. Mas junto com GitHub, voce tem backup, acesso remoto, e colaboracao.

### [POR QUE ISSO IMPORTA PRA VOCE] - 30 segundos

[SLIDE 4 - Produtividade]

**[FALA]**

> E por que voce deveria se importar com isso?
>
> Porque organizacao economiza tempo.
>
> Tempo que voce gastaria procurando "aquela versao que funcionava".
>
> Tempo refazendo trabalho que foi perdido.
>
> Tempo explicando o que mudou para outra pessoa.
>
> [Pausa]
>
> Tempo e dinheiro.
>
> Quem se organiza melhor, produz mais com menos estresse.
>
> Git e GitHub sao ferramentas de produtividade.
