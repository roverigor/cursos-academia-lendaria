# AULA 4.2: Perfil Profissional

**Módulo:** 4 - Sobrevivência e Portfolio
**Duração:** 5 minutos
**Tipo:** Prático

---

## GPS DA AULA

### DESTINO (Onde você vai chegar)
Ao final desta aula, você vai:
- Criar README de perfil atraente
- Destacar melhores repositórios (pinning)
- Configurar bio, foto e links profissionais
- Entender o que recrutadores procuram

### ORIGEM (De onde você vem)
Você provavelmente:
- Sabe o workflow completo de Git/GitHub (Módulos anteriores)
- Tem perfil GitHub vazio ou genérico
- Não sabe que recrutadores visitam seu GitHub antes da entrevista

### ROTA (Como vamos chegar)
1. Entender por que perfil importa
2. Configurar elementos básicos
3. Criar README de perfil
4. Destacar melhores repos

---

## ROTEIRO COMPLETO

### [ABERTURA] - 30 segundos

**[PROFESSOR DIZ:]**

> Seu GitHub é seu portfólio vivo.
>
> Recrutadores de tech visitam seu GitHub ANTES da entrevista. Gastam 30 segundos decidindo se você é interessante.
>
> Perfil bem feito? Você passa confiança. Perfil abandonado? "Será que essa pessoa sabe programar?"
>
> Nessa aula você vai construir um perfil que impressiona. Bora.

---

### [O QUE RECRUTADORES OLHAM] - 1 minuto

[SLIDE 1 - Checklist do recrutador]

**[PROFESSOR DIZ:]**

> O que recrutadores checam em 30 segundos:
>
> **Foto e bio:** Profissionalismo, clareza sobre o que você faz.
>
> **Repos pinados:** Seus melhores projetos em destaque.
>
> **README dos projetos:** Documentação clara mostra maturidade.
>
> **Contribuições:** A grama verde mostra atividade consistente.
>
> [SLIDE 2 - Bom vs Ruim]
>
> Perfil abandonado: sem foto, bio genérica, repos sem README, nenhuma atividade.
>
> Perfil profissional: foto clara, bio específica, projetos organizados, atividade consistente.
>
> [Pausa]
>
> A boa notícia: você configura isso em 10 minutos.

---

### [CONFIGURAÇÃO BÁSICA] - 1 minuto

[SLIDE 3 - Elementos básicos]

**[PROFESSOR DIZ:]**

> Primeiro, os elementos básicos.
>
> **Foto de perfil:**
>
> - Foto sua, profissional
> - Rosto visível, fundo neutro
> - Mesma foto que LinkedIn
>
> Evite: avatar padrão, foto de grupo, memes.
>
> [SLIDE 4 - Bio]
>
> **Bio (160 caracteres max):**
>
> Formato: `[Função] | [Tecnologias] | [Diferencial]`
>
> Exemplos:
>
> ```
> Full-Stack Developer | React, Node.js, PostgreSQL | Building web apps that scale
> ```
>
> ```
> Frontend Engineer | Vue.js, TypeScript | Passionate about UX and accessibility
> ```
>
> [Pausa]
>
> Seja específico. "React Developer" é melhor que "Programador".
>
> Adicione localização e link pro LinkedIn ou portfólio.

---

### [README DE PERFIL] - 1 minuto 30 segundos

[SLIDE 5 - Repo especial]

**[PROFESSOR DIZ:]**

> Agora o diferencial: README de perfil.
>
> Crie um repositório com o MESMO nome do seu username. Por exemplo, se seu username é `joaosilva`, cria repo `joaosilva`.
>
> Esse repo é especial. O README aparece no topo do seu perfil automaticamente.
>
> [SLIDE 6 - Template simples]
>
> Template básico:
>
> ```markdown
> # Olá! Eu sou [Seu Nome]
>
> ## Sobre Mim
>
> Desenvolvedor Full-Stack focado em aplicações web modernas.
> Atualmente aprendendo arquitetura de sistemas.
>
> ## Tecnologias
>
> **Frontend:** JavaScript, React, TypeScript
> **Backend:** Node.js, Python, PostgreSQL
> **Ferramentas:** Git, Docker, VS Code
>
> ## Como me Encontrar
>
> - LinkedIn: [seu-link]
> - Email: seu@email.com
> ```
>
> [Pausa]
>
> Simples, direto, profissional. Não precisa de firula.
>
> Quer algo mais visual? Use badges de shields.io e ícones de devicon.dev.

---

### [DESTACAR REPOSITÓRIOS (PINNING)] - 1 minuto

[SLIDE 7 - Pinned repos]

**[PROFESSOR DIZ:]**

> Você pode destacar até 6 repositórios no topo do perfil. Esses são os primeiros que recrutador vê.
>
> Como pinar:
>
> 1. Vai no seu perfil
> 2. Clica "Customize your pins"
> 3. Seleciona até 6 repos
> 4. Salva
>
> [SLIDE 8 - O que pinar]
>
> O que pinar:
>
> - Projeto mais impressionante (complexo, completo)
> - Projetos com tecnologias diferentes (mostra range)
> - Projetos com bom README (bem documentados)
> - Projetos recentes (mostra atividade)
>
> [Pausa]
>
> O que NÃO pinar:
>
> - Tutoriais que você seguiu (não é original)
> - Repos com nomes tipo "teste123"
> - Forks sem contribuição sua

---

### [FECHAMENTO] - 30 segundos

[SLIDE 9 - Resumo]

**[PROFESSOR DIZ:]**

> Recapitulando:
>
> - Foto profissional e bio específica
> - README de perfil com overview
> - 3-6 repos pinados de qualidade
> - Todos os repos pinados com README
>
> Seu GitHub é seu cartão de visita técnico. Invista esses 10 minutos.
>

>

## TROUBLESHOOTING

### README não aparece no perfil

> Checklist:
> - Nome do repo é EXATAMENTE igual ao seu username?
> - Repo é público?
> - Arquivo chama README.md?

### Não tenho projetos bons para pinar

> Comece projeto pessoal simples. Contribua em open source (mesmo documentação). Refatore projetos antigos com README profissional.

### Bio ficou muito longa

> Foque em 2-3 tecnologias principais. Use `|` em vez de "and". Remova palavras desnecessárias.
