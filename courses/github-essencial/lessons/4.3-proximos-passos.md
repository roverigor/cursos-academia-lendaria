### [GITHUB ACTIONS - AUTOMAÇÃO] - 1 minuto

[SLIDE 2 - GitHub Actions]

**[PROFESSOR DIZ:]**

> Primeiro recurso avançado: GitHub Actions.
>
> É automação direto no GitHub. Roda testes automaticamente. Faz deploy. Gera documentação.
>
> Exemplo: toda vez que você faz push, testes rodam automaticamente. Se passar, fica verde. Se falhar, fica vermelho.
>
> Você cria arquivo em `.github/workflows/test.yml` com instruções. GitHub executa.
>
> [Pausa]
>
> CI/CD (Continuous Integration / Continuous Deployment) é padrão em empresas. GitHub Actions é a forma mais fácil de começar.
>
> Quando aprender: depois que tiver projeto mais complexo com testes automatizados.

## ROADMAP DE EVOLUÇÃO

### Iniciante → Intermediário (1-3 meses)
- [ ] Contribuir em 1 projeto open source
- [ ] Configurar GitHub Actions em projeto
- [ ] Praticar rebases interativos
- [ ] Aprender git cherry-pick

### Intermediário → Avançado (3-6 meses)
- [ ] Dominar Git Flow ou GitHub Flow
- [ ] CI/CD completo (testes + deploy)
- [ ] Contribuir em projeto open source maior
- [ ] Mentorear colega em Git/GitHub

### Avançado → Expert (6-12 meses)
- [ ] Entender Git internals
- [ ] Configurar monorepos
- [ ] Criar GitHub Action customizado
- [ ] Ser maintainer de projeto open source
