**Duracao:** 10 minutos
**Tipo:** Pratico

## GPS DA AULA

### DESTINO (Onde voce vai chegar)
Ao final desta aula, voce vai:
- Ter o Git instalado no seu computador
- Ter seu nome e email configurados
- Estar pronto para comecar a usar

Voce provavelmente:
- Entendeu a diferenca entre Git e GitHub (aula anterior)

## ROTEIRO COMPLETO

### [ABERTURA] - 30 segundos

**[FALA]**

> Agora vamos instalar o Git no seu computador.
>
> E mais simples do que parece. Se voce ja instalou qualquer programa antes, consegue fazer isso.
>
> Vou mostrar passo a passo, para Windows, Mac e Linux.

---

### [POR QUE CONFIGURAR SEU NOME E EMAIL] - 1 minuto

[SLIDE 1 - Identificacao]

**[FALA]**

> Antes de instalar, deixa eu explicar uma coisa importante.
>
> Quando voce salva uma versao no Git, ele registra quem fez aquela mudanca.
>
> Se voce nao configurar seu nome e email, o Git nao sabe quem voce e.
>
> [Pausa]
>
> Isso importa por dois motivos:
>
> **Primeiro:** voce consegue ver o historico e saber "isso aqui fui eu que mudei em outubro".
>
> **Segundo:** quando voce trabalha com outras pessoas, todo mundo sabe quem fez o que.
>
> [Pausa]
>
> E uma configuracao simples que voce faz uma vez so.

---

### [VERIFICANDO SE JA TEM GIT] - 30 segundos

[SLIDE 2 - Terminal]

**[FALA]**

> Primeiro, vamos ver se voce ja tem Git instalado.
>
> Para isso, voce vai usar o terminal. O terminal e aquela tela onde voce digita comandos.
>
> **No Windows:** procure por "cmd" ou "Prompt de Comando" no menu iniciar.
>
> **No Mac:** abra o aplicativo "Terminal" (fica na pasta Utilitarios).
>
> Com o terminal aberto, digite:
>
> `git --version`
>
> E aperte Enter.
>
> [Pausa]
>
> Se aparecer algo como "git version 2.40.0", voce ja tem Git. Pode pular para a parte de configuracao.
>
> Se aparecer erro ou "comando nao encontrado", vamos instalar agora.

---

### [INSTALACAO - WINDOWS] - 1 minuto 30 segundos

[SLIDE 3 - Git for Windows]

**[FALA]**

> Se voce usa Windows, faca assim:
>
> Abra o navegador e acesse: git-scm.com
>
> Na pagina inicial, clique em "Download for Windows".
>
> Baixe o arquivo e execute o instalador.
>
> [SLIDE 4 - Telas de instalacao]
>
> Durante a instalacao, vai aparecer varias telas com opcoes.
>
> Na maioria delas, pode clicar em "Next" e aceitar o padrao.
>
> Preste atencao em duas coisas:
>
> **Opcao de branch:** se aparecer, escolha "main" como padrao.
>
> **Opcao de terminal:** escolha "Git Bash" quando perguntar.
>
> O Git Bash e um terminal especial que vem junto com o Git. Ele facilita o uso no Windows.
>
> [Pausa]
>
> Depois de instalar, abra o programa "Git Bash" (vai estar no menu iniciar).
>
> Digite:
>
> `git --version`
>
> Se aparecer a versao, a instalacao deu certo.

---

### [INSTALACAO - MAC] - 1 minuto

[SLIDE 5 - macOS]

**[FALA]**

> Se voce usa Mac, tem duas formas de instalar.
>
> **Forma mais simples:**
>
> Abra o Terminal e digite:
>
> `git --version`
>
> Se o Git nao estiver instalado, o Mac vai perguntar se voce quer instalar as "Command Line Tools".
>
> Sao ferramentas de desenvolvedor da Apple. Aceite e aguarde. Demora uns 2-3 minutos.
>
> Depois, digite novamente:
>
> `git --version`
>
> Deve mostrar a versao instalada.
>
> [Pausa]
>
> **Forma alternativa (com Homebrew):**
>
> Homebrew e um programa que facilita instalar outros programas no Mac. Muita gente usa.
>
> Se voce ja tem Homebrew instalado, pode digitar:
>
> `brew install git`
>
> Se voce nao sabe o que e Homebrew, use a primeira forma. E mais simples.

---

### [INSTALACAO - LINUX] - 30 segundos

[SLIDE 6 - Linux]

**[FALA]**

> Se voce usa Linux, provavelmente ja tem Git instalado.
>
> Para verificar, abra o terminal e digite:
>
> `git --version`
>
> Se precisar instalar, o comando depende da sua distribuicao.
>
> **Ubuntu ou Debian:**
>
> `sudo apt update && sudo apt install git`
>
> **Fedora:**
>
> `sudo dnf install git`
>
> Digite sua senha quando pedir. Pronto.

---

### [CONFIGURANDO NOME E EMAIL] - 2 minutos

[SLIDE 7 - Configuracao]

**[FALA]**

> Agora vem a parte mais importante: dizer ao Git quem voce e.
>
> No terminal, digite esses dois comandos. Substitua pelos seus dados:
>
> ```
> git config --global user.name "Seu Nome"
> git config --global user.email "seu@email.com"
> ```
>
> Por exemplo:
>
> ```
> git config --global user.name "Maria Silva"
> git config --global user.email "maria@empresa.com"
> ```
>
> [Pausa]
>
> **Dica importante:** use o mesmo email que voce cadastrou no GitHub.
>
> Assim o GitHub consegue conectar suas mudancas com sua conta.
>
> Se voce usou maria@empresa.com no GitHub, use maria@empresa.com aqui tambem.
>
> [Pausa]
>
> Essa configuracao fica salva no seu computador. Voce so precisa fazer uma vez.

---

### [CONFIGURACAO DO BRANCH PADRAO] - 30 segundos

[SLIDE 8 - Branch main]

**[FALA]**

> Uma ultima configuracao rapida.
>
> Digite:
>
> `git config --global init.defaultBranch main`
>
> Isso define que quando voce criar um novo projeto, ele vai usar "main" como nome do branch principal.
>
> Branch e como uma versao do seu projeto. Vamos falar mais sobre isso depois.
>
> Por enquanto, so precisa saber que "main" e o padrao que todo mundo usa hoje.

---

### [VERIFICANDO SE DEU CERTO] - 1 minuto

[SLIDE 9 - Verificacao]

**[FALA]**

> Vamos confirmar que tudo esta configurado.
>
> Digite:
>
> `git config --list`
>
> Voce deve ver algo parecido com isso:
>
> ```
> user.name=Seu Nome
> user.email=seu@email.com
> init.defaultbranch=main
> ```
>
> [Pausa]
>
> Se aparecer seu nome e email, parabens. Esta tudo pronto.
>
> Essa configuracao fica salva permanentemente no seu computador.
