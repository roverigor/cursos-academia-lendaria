### [O QUE E SSH] - 1 minuto 30 segundos

[SLIDE 1 - O que e SSH]

**[FALA]**

> SSH e uma forma segura de conectar dois computadores.
>
> O nome vem de "Secure Shell", que significa "conexao segura". Voce nao precisa decorar isso, so entender o conceito.
>
> [Pausa]
>
> Funciona assim: voce cria duas "chaves" no seu computador.
>
> Uma chave e **privada** - ela fica guardada no seu computador e voce nunca compartilha com ninguem. E como a chave da sua casa.
>
> A outra chave e **publica** - essa voce envia para o GitHub. E como se fosse um cadeado que so a sua chave privada consegue abrir.
>
> [SLIDE 2 - Analogia chave e cadeado]
>
> Quando voce se conecta ao GitHub, ele testa: "esse computador tem a chave certa para abrir o cadeado?"
>
> Se a chave combina, a conexao e liberada. Sem precisar de senha.
>
> [Pausa]
>
> O tipo de chave que vamos usar se chama **Ed25519**. E o mais moderno e seguro.
>
> O GitHub recomenda esse tipo desde 2022. E mais rapido e mais seguro que os tipos antigos.

---

### [PASSO 1: VERIFICAR SE JA TEM] - 30 segundos

[SLIDE 3 - Terminal]

**[FALA]**

> Primeiro, vamos verificar se voce ja tem uma chave configurada.
>
> Abra o terminal e digite:
>
> ```
> ls ~/.ssh/id_ed25519.pub
> ```
>
> Esse comando verifica se existe um arquivo de chave no seu computador.
>
> [Pausa]
>
> Se aparecer o nome do arquivo, voce ja tem uma chave. Pode pular para o passo de adicionar no GitHub.
>
> Se aparecer uma mensagem de erro como "No such file" ou "nao encontrado", voce precisa criar a chave. Vamos fazer isso agora.

---

### [PASSO 2: CRIAR A CHAVE] - 1 minuto 30 segundos

[SLIDE 4 - Criando a chave]

**[FALA]**

> Para criar sua chave, digite esse comando no terminal.
>
> Substitua pelo seu email - use o mesmo email que voce cadastrou no GitHub:
>
> ```
> ssh-keygen -t ed25519 -C "seu@email.com"
> ```
>
> Explicando o comando:
> - `ssh-keygen` e o programa que cria chaves
> - `-t ed25519` diz o tipo de chave (o mais seguro)
> - `-C "seu@email.com"` e um comentario para identificar a chave
>
> [Pausa]
>
> Depois de digitar, ele vai perguntar onde salvar. Pode apertar Enter para aceitar o local padrao.
>
> Depois pergunta se voce quer criar uma senha para a chave. Isso e chamado de "passphrase".
>
> [Pausa]
>
> Aqui voce decide:
>
> Se o computador e so seu, pode deixar em branco e apertar Enter. E mais pratico.
>
> Se outras pessoas usam o computador, coloque uma senha para mais seguranca.
>
> [Pausa]
>
> Pronto. Sua chave foi criada.
>
> Voce vai ver uma mensagem confirmando que o arquivo foi salvo.

---

### [PASSO 3: ATIVAR A CHAVE] - 45 segundos

[SLIDE 5 - Ativando a chave]

**[FALA]**

> Agora precisamos "ativar" a chave no seu computador.
>
> Isso e feito com dois comandos.
>
> **No Mac ou Linux:**
>
> ```
> eval "$(ssh-agent -s)"
> ssh-add ~/.ssh/id_ed25519
> ```
>
> **No Windows (usando Git Bash):**
>
> ```
> eval `ssh-agent -s`
> ssh-add ~/.ssh/id_ed25519
> ```
>
> [Pausa]
>
> O primeiro comando inicia o "agente" que gerencia suas chaves.
>
> O segundo comando adiciona sua chave ao agente.
>
> Se voce criou uma senha para a chave, ele vai pedir agora. Digite e aperte Enter.
>
> Pronto. Sua chave esta ativa.

---

### [PASSO 4: COPIAR A CHAVE PUBLICA] - 45 segundos

[SLIDE 6 - Copiando a chave]

**[FALA]**

> Agora voce precisa copiar a chave **publica** para enviar ao GitHub.
>
> Lembre: a chave publica e a que voce compartilha. A privada nunca sai do seu computador.
>
> **No Mac:**
>
> ```
> pbcopy < ~/.ssh/id_ed25519.pub
> ```
>
> Esse comando copia a chave automaticamente.
>
> **No Windows (Git Bash):**
>
> ```
> cat ~/.ssh/id_ed25519.pub | clip
> ```
>
> **No Linux:**
>
> ```
> cat ~/.ssh/id_ed25519.pub
> ```
>
> No Linux, selecione o texto que aparecer e copie manualmente.
>
> [Pausa]
>
> A chave comeca com "ssh-ed25519" e termina com seu email. Copie tudo, do inicio ao fim.

---

### [PASSO 5: ADICIONAR NO GITHUB] - 1 minuto

[SLIDE 7 - GitHub Settings]

**[FALA]**

> Agora vamos adicionar sua chave publica no GitHub.
>
> Acesse github.com e faca login na sua conta.
>
> Clique na sua foto no canto superior direito.
>
> Clique em "Settings" (Configuracoes).
>
> No menu lateral, clique em "SSH and GPG keys".
>
> [SLIDE 8 - Adicionando a chave]
>
> Clique no botao verde "New SSH key".
>
> No campo "Title", coloque um nome que identifique seu computador. Por exemplo: "Notebook Pessoal" ou "Computador do Escritorio".
>
> No campo "Key", cole a chave que voce copiou.
>
> Clique em "Add SSH key".
>
> [Pausa]
>
> O GitHub pode pedir sua senha para confirmar. Digite e confirme.
>
> Pronto. Sua chave esta cadastrada no GitHub.

---

### [PASSO 6: TESTAR A CONEXAO] - 1 minuto

[SLIDE 9 - Testando]

**[FALA]**

> Agora vamos testar se tudo esta funcionando.
>
> No terminal, digite:
>
> ```
> ssh -T git@github.com
> ```
>
> Se for a primeira vez que voce conecta, vai aparecer uma pergunta sobre "fingerprint" e se voce confia no servidor.
>
> Digite "yes" e aperte Enter.
>
> [Pausa]
>
> Se deu certo, voce vai ver uma mensagem assim:
>
> "Hi SeuUsuario! You've successfully authenticated, but GitHub does not provide shell access."
>
> [SLIDE 10 - Sucesso]
>
> Se voce viu essa mensagem com seu nome de usuario, parabens.
>
> A conexao segura esta funcionando.
>
> A partir de agora, todas as vezes que voce usar Git com o GitHub, nao vai precisar digitar senha.

## SE DER ERRO

### Erro: "Permission denied (publickey)"

Esse erro significa que a chave nao foi reconhecida. Pode ser:

1. A chave nao foi adicionada no GitHub - verifique em github.com/settings/keys
2. O agente nao esta ativo - rode os comandos:

```
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

### Erro: "Could not open a connection to your authentication agent"

O agente de chaves nao esta rodando. Inicie com:

```
eval "$(ssh-agent -s)"
```

## COMANDOS DA AULA (COPIAR E COLAR)

```bash
# Verificar se ja tem chave
ls ~/.ssh/id_ed25519.pub

# Criar chave (substitua pelo seu email)
ssh-keygen -t ed25519 -C "seu@email.com"

# Iniciar o agente (Mac/Linux)
eval "$(ssh-agent -s)"

# Iniciar o agente (Windows Git Bash)
eval `ssh-agent -s`

# Adicionar chave ao agente
ssh-add ~/.ssh/id_ed25519

# Copiar chave publica (Mac)
pbcopy < ~/.ssh/id_ed25519.pub

# Copiar chave publica (Windows Git Bash)
cat ~/.ssh/id_ed25519.pub | clip

# Copiar chave publica (Linux - copie manualmente o resultado)
cat ~/.ssh/id_ed25519.pub

# Testar conexao
ssh -T git@github.com
```
