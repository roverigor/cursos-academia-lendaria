# Aula 7.1 - Integração com Notion e Obsidian

**Módulo:** 7 - Workflows e Automação Profissional
**Duração:** 20 minutos
**Tipo:** Tutorial Técnico

---

## Objetivo da Aula

Criar fluxos de sincronização entre NotebookLM e suas ferramentas de produtividade (Notion, Obsidian).

---

## Roteiro de Gravação

### INTRODUÇÃO (0:00 - 1:30)

**[CENA: Instrutor em câmera]**

NotebookLM é poderoso, mas não é sua única ferramenta. Você provavelmente usa:

- **Notion** — Gestão de projetos, docs colaborativos
- **Obsidian** — Segundo cérebro pessoal, notas interligadas
- **Google Docs** — Documentos de trabalho

O problema: informação fragmentada.

A solução: **workflows integrados** que conectam tudo.

Nesta aula, vamos criar pipelines práticos.

---

### PARTE 1: NOTION → NOTEBOOKLM (1:30 - 8:00)

**[CENA: Screencast - Notion + NotebookLM]**

#### Caso de Uso:
Você tem uma base de conhecimento no Notion. Quer analisar com NotebookLM.

---

#### Método 1: Export Manual (Simples)

**Passo a passo:**

1. **No Notion:**
   - Abra a página/database
   - Clique "..." → Export
   - Escolha: Markdown & CSV ou PDF

2. **No NotebookLM:**
   - Upload do arquivo exportado
   - Ou: Copiar texto → Copied Text

**[DEMONSTRAR - exportar página Notion, importar no NotebookLM]**

**Quando usar:** Análise pontual, poucos documentos.

---

#### Método 2: Link Público (Semi-Automático)

**Passo a passo:**

1. **No Notion:**
   - Abra a página
   - Clique "Share" → "Share to web"
   - Copie o link público

2. **No NotebookLM:**
   - Add Source → Website
   - Cole o link público

**[DEMONSTRAR]**

**Vantagem:** Notion atualiza, re-adicione o link para atualizar source.
**Limitação:** Precisa ser público. Não ideal para docs confidenciais.

---

#### Método 3: Notion API + Automação (Avançado)

**Arquitetura:**
```
Notion Database → Notion API → n8n/Make → Export → NotebookLM
```

**Quando usar:** Volume alto, atualização frequente.
**Veremos em detalhes:** Aula 7.2 (Automação com n8n)

---

### PARTE 2: NOTEBOOKLM → NOTION (8:00 - 13:00)

**[CENA: Screencast - NotebookLM + Notion]**

#### Caso de Uso:
Você gerou análises no NotebookLM. Quer salvar/organizar no Notion.

---

#### Workflow Recomendado:

```
1. GERE OUTPUT NO NOTEBOOKLM
   Exemplo: Executive Summary, FAQ, Study Guide

2. COPIE O OUTPUT
   Selecione → Ctrl+C (ou botão Copy)

3. FORMATE NO NOTION
   • Crie página dedicada
   • Cole e organize
   • Adicione metadata (tags, data, projeto)

4. CONECTE
   • Link para notebook original (se relevante)
   • Relacione com outras páginas
```

**[DEMONSTRAR - copiar FAQ do NotebookLM, colar no Notion formatado]**

---

#### Template Notion para Outputs:

```markdown
# [Título da Análise]

**Fonte:** [Link para NotebookLM notebook]
**Data:** [Quando foi gerado]
**Tipo:** Executive Summary | FAQ | Study Guide | etc
**Projeto:** [Relacionar com projeto]

---

## Conteúdo

[Colar output aqui]

---

## Notas Adicionais

[Suas observações]

---

## Tags
#notebooklm #análise #[projeto]
```

**[MOSTRAR template no Notion]**

### PARTE 3: OBSIDIAN → NOTEBOOKLM (13:00 - 17:00)

**[CENA: Screencast - Obsidian + NotebookLM]**

#### Caso de Uso:
Você tem notas no Obsidian. Quer analisar conexões com NotebookLM.

---

#### Método 1: Export de Notas Específicas

**Passo a passo:**

1. **No Obsidian:**
   - Selecione notas relevantes
   - Export como PDF ou Markdown
   - Ou: Copie conteúdo diretamente

2. **No NotebookLM:**
   - Upload PDFs
   - Ou: Copied Text para cada nota

**[DEMONSTRAR]**

---

#### Método 2: Export de Vault Parcial

**Para análise de cluster de notas:**

1. **Identifique o tema:**
   - Use tags ou links no Obsidian
   - Exemplo: todas notas com #projeto-x

2. **Export em batch:**
   - Plugin: "Obsidian Export" ou manual
   - Gere PDFs ou compile em documento único

3. **Carregue no NotebookLM:**
   - Cada PDF como source separada
   - Ou: documento compilado

**[DEMONSTRAR - exportar cluster de notas]**

---

#### Método 3: Dataview Query → Export

**Para usuários avançados de Obsidian:**

```dataview
TABLE file.name, file.tags
FROM #projeto-x
SORT file.mtime DESC
```

- Execute query para listar notas
- Export resultado
- Use como guia para seleção

---

### PARTE 4: NOTEBOOKLM → OBSIDIAN (17:00 - 19:00)

**[CENA: Screencast - NotebookLM + Obsidian]**

#### Workflow de Captura:

```
1. ANÁLISE NO NOTEBOOKLM
   Gere insight/síntese

2. CAPTURE NO OBSIDIAN
   Crie nota: "AI Analysis - [Tema] - [Data]"

3. FORMATE PARA INTERLIGAÇÃO
   • Adicione links [[para outras notas]]
   • Adicione tags #ai-analysis #notebooklm
   • Adicione metadata YAML

4. CONECTE NO GRAFO
   • Link para notas de fonte
   • Link para projetos relacionados
```

---

#### Template Obsidian:

```markdown
---
type: ai-analysis
source: notebooklm
date: {{date}}
project:
tags: [ai-analysis, notebooklm]
---

# AI Analysis: [Tema]

## Fontes Analisadas
- [[Nota 1]]
- [[Nota 2]]
- Documento externo: [nome]

## Síntese

[Output do NotebookLM aqui]

## Meus Insights

[Suas observações adicionais]

## Ações

- [ ] [Ação derivada]

## Links Relacionados
- [[Projeto X]]
- [[Tema Y]]
```

**[MOSTRAR template no Obsidian]**

### FECHAMENTO (19:00 - 20:00)

**[CENA: Instrutor em câmera]**

Você agora conecta NotebookLM ao seu sistema de produtividade.

**Resumo dos workflows:**

| Direção | Método | Quando Usar |
|---------|--------|-------------|
| Notion → NotebookLM | Export/Link | Análise de docs Notion |
| NotebookLM → Notion | Copy + Template | Salvar análises |
| Obsidian → NotebookLM | Export cluster | Analisar notas |
| NotebookLM → Obsidian | Capture + Link | Integrar ao grafo |

**Missão:**
1. Escolha 5 notas/docs do seu Notion ou Obsidian
2. Carregue no NotebookLM
3. Gere uma análise
4. Salve de volta na ferramenta original
