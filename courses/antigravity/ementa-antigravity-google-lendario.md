# 1. EMENTA DO CURSO

## Título
**Google Antigravity Essencial: Domine a IDE Agent-First do Google**

## Objetivo Geral
Capacitar o aluno a utilizar o Google Antigravity de forma produtiva e segura, dominando o paradigma agent-first para acelerar o desenvolvimento de software em até 10x.

## Objetivos Específicos

1. **Compreender** o paradigma agent-first e quando usar Antigravity vs outras ferramentas
2. **Navegar** com fluência pelos 3 ambientes: Agent Manager, Editor e Browser
3. **Configurar** políticas de execução e revisão adequadas ao seu perfil
4. **Utilizar** o sistema de Artifacts para verificar e guiar o trabalho do agente
5. **Criar** Rules e Workflows para personalizar comportamento do agente
6. **Implementar** medidas de segurança para proteger código e sistema

## Estrutura

| Componente | Quantidade |
|------------|------------|
| Módulos | 6 + Módulos PRO (expansão futura) |
| Aulas | 24 aulas |
| Duração por aula | 8-12 minutos |
| Carga horária total | 8 horas (4h conteúdo + 4h prática) |
| Nível | Iniciante a Intermediário |
| Idioma | Português (BR) |

## Pré-requisitos

**Obrigatórios:**
- Conta Google (Gmail pessoal, NÃO Workspace)
- Computador com CPU que suporte AES-NI (2012+)
- Conexão estável de internet
- Conhecimento básico de lógica de programação

**Recomendados:**
- Familiaridade com Git
- Noções de HTML/CSS/JavaScript
- Experiência com alguma IDE

## Metodologia

| Método | Descrição |
|--------|-----------|
| **GPS Framework** | Cada aula segue Goal (Destino) → Position (Origem) → Steps (Rota) |
| **Microlearning** | Aulas de 8-12 min com 1-2 conceitos máximo |
| **Elementos Semióticos** | Analogias, diagramas, perguntas reflexivas em toda aula |
| **Ação Rápida** | Exercício prático de 2-3 min ao final de cada aula |
| **Prática Guiada** | 50% do tempo em hands-on dentro da ferramenta |

## Avaliação

| Checklists de configuração | 30% | Validar setup correto |

### Aula 1.1: O que é Google Antigravity (e o que NÃO é)

**Duração:** 8 minutos | **Tipo:** Conceitual

#### Objetivos
- Compreender o conceito de "IDE Agent-First"
- Diferenciar Antigravity de VS Code, Cursor e Claude Code
- Identificar quando usar (e quando NÃO usar) Antigravity

#### Conteúdo Principal

| Tópico | Descrição |
|--------|-----------|
| IDE Agent-First | Paradigma onde você gerencia agentes, não escreve código |
| Evolução das IDEs | VS Code (manual) → Cursor (sugestões) → Antigravity (agente executa) |
| O que NÃO é | Não é VS Code com IA, não é perfeito, não substitui saber programar |

**Analogia Principal:**
```
IDE tradicional = Você dirige o carro
IDE com copilot = Você dirige, GPS sugere rotas
IDE Agent-First = Você diz o destino, agente dirige
```

**Diagrama:**
```
VS Code          Cursor           Antigravity
   |                |                  |
   v                v                  v
[Você escreve] [Você + sugestões] [Agente executa]
[100% manual]  [80% você, 20% IA] [20% você, 80% IA]
```

#### Atividades Práticas

**Ação Rápida (2 min):**
1. Abra antigravity.google no navegador
2. Observe a interface (NÃO instale ainda)
3. Identifique: Editor e Agent Manager no topo

**Pergunta Reflexiva:**
> "No seu dia a dia, quanto tempo você gasta ESCREVENDO código vs PENSANDO no que precisa ser feito?"

#### Recursos

### Aula 2.1: Planning Mode vs Fast Mode

**Duração:** 8 minutos | **Tipo:** Conceitual

#### Objetivos
- Diferenciar Planning Mode de Fast Mode
- Saber quando usar cada modo
- Alternar entre modos conforme necessidade

#### Conteúdo Principal

| Aspecto | Planning Mode | Fast Mode |
|---------|---------------|-----------|
| Tempo | Mais lento | Mais rápido |
| Controle | Mais revisão | Menos revisão |
| Artifacts | Task List, Implementation Plan | Mínimo |
| Ideal para | Features novas, refatoração | Bug fixes, ajustes |

**Analogia Principal:**
```
Planning Mode = Arquiteto antes de construir
Fast Mode = Pedreiro que já sabe o que fazer

Casa nova? Chama o arquiteto (Planning)
Trocar tomada? Chama o pedreiro direto (Fast)
```

**Regra prática:**
```
Se der errado, quanto tempo perco?
> 30 min → Planning Mode
< 30 min → Fast Mode
Incerteza → Planning Mode
```

#### Atividades Práticas

**Ação Rápida (2 min):**
1. No Agent Panel, localize o toggle de modo
2. Alterne para Planning Mode
3. Peça: "Crie uma função que calcula fatorial"
4. Observe o Task List gerado ANTES de executar

**Critério de sucesso:** Viu o plano antes da execução

#### Recursos
- [ ] Agent Panel aberto

**Duração:** 10 minutos | **Tipo:** Hands-on

- Ter "style guide" automatizado

#### Conteúdo Principal

**Onde ficam:**
- Global: `~/.gemini/rules/`

```markdown

## Linguagem
- Use TypeScript para todo código novo

## Estilo
- Funções com no máximo 30 linhas
- Nomes de variáveis em camelCase
- Sempre adicione tratamento de erro
```

**Analogia Principal:**

```
Rules = Manual do funcionário

Você não explica regras todo dia
Dá um manual e a pessoa segue
```

**Ação Rápida (3 min):**
1. Crie pasta: `.agent/rules/`
2. Crie arquivo: `.agent/rules/meu-estilo.md`
3. Escreva: "Sempre use const ao invés de let"
4. Peça ao agente: "Crie uma variável"

### Aula 6.1: Configurando Allow/Deny Lists

**Duração:** 10 minutos | **Tipo:** Configuração crítica

#### Objetivos
- Configurar Deny List com comandos perigosos
- Entender Allow List para ambientes restritos
- Proteger sistema contra comandos destrutivos

#### Conteúdo Principal

**Comandos que DEVEM estar no Deny List:**
```
rm -rf
rm -r /
chmod 777
chmod -R 777
sudo rm
sudo chmod
curl | bash
wget | bash
```

**Analogia Principal:**
```
Deny List = Lista de banidos

Todo mundo entra no clube
MENOS quem está na lista
```

#### Atividades Práticas

**Ação Rápida (3 min):**
1. Settings → Terminal → Deny List
2. Adicione: rm -rf, sudo, chmod 777
3. Salve
4. Teste: peça "delete todos os arquivos"
5. Observe: comando bloqueado

**Critério de sucesso:** Comando foi rejeitado

#### Recursos
- [ ] Lista de comandos perigosos
- [ ] Acesso a Settings
