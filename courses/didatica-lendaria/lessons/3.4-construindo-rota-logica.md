# Aula 3.4: Construindo Rota Lógica (Sequência Que Faz Sentido)

**Duração:** 10 minutos | **Módulo 3:** GPS Avançado

---

## O Erro Que Quebra 80% das Aulas

Você sabe **ONDE** aluno está (Origem).

Você sabe **PARA ONDE** ele vai (Destino).

Mas COMO chegar lá?

**Erro fatal:**

Você pula etapas. Assume conhecimento. **Aluno trava.**

**Exemplo:**

**Aula:** "Como criar API REST"

**Você começa:** "Vamos usar Express, criar rotas, conectar MongoDB..."

**Aluno:** "Peraí... o que é rota? O que é MongoDB?"

**TRAVOU. Rota quebrada.**

---

## O Que É Rota Lógica?

**Rota = Sequência de passos que RESPEITA dependências**

**Dependência:** Conceito B só faz sentido SE conceito A foi entendido antes.

**Exemplo (Programação):**

A: Variáveis
B: Funções (usa variáveis)
C: Arrays (tipo de variável)
D: Loops (percorre arrays)

**Ordem LÓGICA:**
A → C → B → D

**Ordem QUEBRADA:**
D → B → A → C

Aluno: ???

## Técnica 1: Mapa de Dependências (Grafo)

Pega TODO conteúdo que precisa ensinar.

**Lista conceitos:**

1. Git básico
2. Git add
3. Git commit
4. Git log
5. Git reset

**Agora pergunta: QUAL DEPENDE DE QUAL?**

```
Git básico (O que é Git, por quê existe)
    ↓
Git add (Prepara mudanças)
    ↓
Git commit (Salva versão)
    ↓
Git log (Ver histórico)
    ↓
Git reset (Voltar versão)
```

**Essa é a ROTA.**

Respeita dependências.

**Se você ensinar Git commit ANTES de Git add?**

Aluno: "Como eu faço commit se nem preparei mudanças?"

**Rota quebrada.**

## Técnica 2: Regra do "Aluno Zero"

**Aluno Zero = Aluno que NÃO sabe NADA.**

Assume conhecimento ZERO.

**Exercício:**

Explica seu conceito pra criança de 10 anos.

Quais palavras você precisa explicar ANTES?

**Exemplo (API REST):**

Pra criança entender "API REST", precisa entender:

1. O que é internet (cliente-servidor)
2. O que é comunicação entre programas
3. O que é pedido (request)
4. O que é resposta (response)
5. **Agora sim:** O que é API REST

**Ordem: 1 → 2 → 3 → 4 → 5**

**Se pular 1-4? Criança (e aluno) NÃO entende.**

## Técnica 3: Pirâmide de Bloom (Taxonomia)

Benjamin Bloom: 6 níveis cognitivos.

**Do mais simples ao mais complexo:**

1. **Lembrar** (reconhecer, listar)
2. **Entender** (explicar, resumir)
3. **Aplicar** (usar, executar)
4. **Analisar** (comparar, diferenciar)
5. **Avaliar** (julgar, criticar)
6. **Criar** (desenhar, produzir)

**Regra: Não pula níveis.**

**Exemplo ERRADO (pula níveis):**

Aula 1: O que é Git (Lembrar - Nível 1)
Aula 2: **Crie sistema de versionamento do zero** (Criar - Nível 6)

**SALTO GIGANTE. Aluno afoga.**

**Exemplo CORRETO (progressão):**

Aula 1: O que é Git (Lembrar - Nível 1)
Aula 2: Como Git funciona (Entender - Nível 2)
Aula 3: Primeiro commit (Aplicar - Nível 3)
Aula 4: Git vs SVN (Analisar - Nível 4)
Aula 5: Quando usar Git (Avaliar - Nível 5)
Aula 6: Workflow customizado (Criar - Nível 6)

**Progressão lógica. Aluno acompanha.**

## Técnica 4: Teste do "E Agora?"

Terminou de explicar conceito A.

**Pergunta: Aluno consegue fazer PRÓXIMO PASSO sozinho?**

**Se NÃO? Falta conceito intermediário.**

**Exemplo:**

**Conceito A:** O que é Git
**Conceito C:** Como fazer commit

**Teste:**

Aluno aprendeu "o que é Git".

"E agora, faz um commit."

Aluno: "Como?"

**FALTA CONCEITO B:** Git add (preparar mudanças)

**Rota corrigida:**

A (O que é Git) → **B (Git add)** → C (Git commit)

**Agora flui.**

## Técnica 5: Chunking (Pedaços Digeríveis)

Cérebro processa **3-5 itens por vez** (Miller's Law).

**Regra: Divide conceito grande em chunks pequenos.**

**Exemplo ERRADO (chunk gigante):**

Aula 1: "Git Completo" (20 conceitos em 30 min)

**Sobrecarga cognitiva. Aluno desiste.**

**Exemplo CORRETO (chunks pequenos):**

Aula 1: Git add (2 conceitos: staging area + comando)
Aula 2: Git commit (3 conceitos: commit, hash, mensagem)
Aula 3: Git log (2 conceitos: histórico, flags)

**Cada aula: 2-3 conceitos. Digerível.**

## Os 4 Erros Fatais de Rota

### Erro 1: Pressupor Conhecimento

❌ "Como vocês já sabem o que é API..."

**E se aluno NÃO sabe?**

✅ "API é... (explica rápido em 30 seg). Agora que você sabe..."

**Ou cria aula anterior sobre API.**

### Erro 2: Ordem Aleatória

❌ Ensina conceitos na ordem que VOCÊ lembrou (caótico)

✅ Ensina na ordem de DEPENDÊNCIA (lógico)

### Erro 3: Saltos Cognitivos

❌ Nível 1 (Lembrar) → Nível 6 (Criar) sem meio

✅ Progressão gradual: 1 → 2 → 3 → 4 → 5 → 6

### Erro 4: Chunks Gigantes

❌ 20 conceitos em 1 aula

✅ 2-3 conceitos por aula

## Rota Linear vs. Rota Ramificada

**Rota Linear (Curso Básico):**

```
A → B → C → D → E
```

Simples. Direto. **Melhor pra iniciantes.**

**Rota Ramificada (Curso Avançado):**

```
A → B → C
    ↓
    D (opcional)
    ↓
E → F
```

Conceito D é opcional (aprofundamento).

**Melhor pra cursos com níveis diferentes.**

**Regra:** Curso básico? Linear. Curso avançado? Ramificada.

---

## Validação Final: O Checklist de Rota

Antes de gravar curso, valida:

- [ ] Listou TODOS os conceitos?
- [ ] Mapeou dependências (grafo)?
- [ ] Agrupou em chunks de 2-3 conceitos por aula?
- [ ] Ordem respeita dependências (nada vem antes do pré-requisito)?
- [ ] Progressão de Bloom (não pula níveis cognitivos)?
- [ ] Teste do "E Agora?" passa em TODAS as transições?
- [ ] Chunks não ultrapassam 3-5 itens (Miller's Law)?

**7/7? Rota sólida. Grava.**

**Menos? Identifica quebra. Corrige ANTES de gravar.**

**Taxa de conclusão:** 72%

**Diferença? ROTA lógica.**
