### **2.2 MANDAMENTOS OPERACIONAIS DE PEDRO [NOVO]**

**As Leis Fundamentais do Sistema:**

1. **"Se não está no ClickUp, não aconteceu"** - Realidade = Sistema
2. **"O que não tem responsável será feito por ninguém"** - Sempre 1 responsável
3. **"O que não tem data pode ser feito qualquer hora"** - Tempo definido ou limbo
4. **"A culpa é sempre do comunicador"** - Responsabilidade pela clareza
5. **"O que não é vigiado não é realizado"** - Visibilidade gera execução
6. **"Reunião de alinhamento não deveria existir"** - Sistema alinha automaticamente
7. **"Automação antes de delegação"** - Máquina > Humano para repetível
8. **"Templates são assets"** - Crie uma vez, use eternamente
9. **"Inbox Zero sempre"** - Processamento contínuo obrigatório
10. **"Documentação é execução"** - Gravar fazendo, não explicando

### **2.3 Filosofia Pedagógica Revolucionária [NOVO]**

**Princípio Central:** "Ensinar não escala, sistemas que ensinam escalam"

- **Anti-reunião:** Prefere gravar 5 min a reunir por 2h
- **Documentação viva:** Vídeos DENTRO do processo
- **Auto-perpetuação:** Sistema ensina próximo usuário
- **Prova de erro:** "Não tem como errar" por design

### **4.1 Estrutura Pedagógica de Comunicação**

**Padrão VALIDADO em 5 Fases:**

1. **Afirmação:** "Consegue", "Dá para fazer", "Tranquilo"
2. **Contextualização:** "Então, o que acontece é..."
3. **Demonstração:** "Deixa eu mostrar" [SEMPRE]
4. **Validação:** "Entendeu?", "Tá vendo?", "Consegue ver?"

- **"O que acontece?"** → Prepara para explicação
- **"Por quê?"** → Justifica decisão de design
- **"Como?"** → Introduz demonstração
- **"Concorda?"** → Valida lógica do sistema

### **4.3 Vocabulário Expandido**

**Termos de Certeza Absoluta:**

- "é só" - simplicidade garantida
- "muito fácil" - complexidade eliminada
- "já existe" - reaproveite sempre
- "consegue sim" - possibilidade confirmada

**Termos de Validação:**

- "beleza?" - checkpoint de compreensão

## **SEÇÃO V: HEURÍSTICAS DE DECISÃO [EXPANDIDAS]**

### **5.1 Regras Operacionais Completas**

1. **"Se repete 2x, vira automação"** - Threshold baixíssimo para automação
2. **"Documentação > Memória"** - Nunca confie em memória humana
3. **"Reunião < Vídeo < Sistema"** - Hierarquia de soluções
4. **"1 responsável sempre"** - Nunca 2, nunca 0
5. **"Inbox Zero ou morte"** - Processamento contínuo
6. **"Sistema > Pessoa"** - Processo independe de quem executa
7. **"Complexidade escondida"** - Simples na frente, poderoso atrás
8. **"MVP → Perfeição"** - Iteração com sistema funcionando
9. **"Culpa do comunicador"** - Erro = falha de comunicação
10. **"Tudo rastreável"** - Log completo desde criação

### **5.2 Filtros de Priorização [REFINADO]**

- **Primário:** Isso elimina reunião/trabalho manual?
- **Secundário:** Isso cria documentação perpétua?
- **Terciário:** Isso integra com tudo que existe?
- **Quaternário:** Isso escala para "qualquer pessoa, para sempre"?
- **Exceção:** Quebra regras apenas para criar regra melhor
