## Risks & Mitigations

### CRITICAL Risks

**1. Densidade de Fontes Abaixo do Ideal**
- **Risco:** 10-20h estimadas vs 50h+ recomendado para Premium
- **Impacto:** Clone superficial ou inconsistente
- **Probabilidade:** ALTA (fontes dispersas)
- **Mitigação:**
  -   **AÇÃO IMEDIATA:** Identificar e transcrever 5-10 lives IA prioritárias
  - Focar em algoritmo mental (como pensa) vs conhecimento factual
  - Usar profile psicométrico como âncora estrutural
  - Checkpoint obrigatório: 30h+ fontes antes de prosseguir para Etapa 3

**2. Viés de Confirmação (Auto-clonagem)**
- **Risco:** Criar clone idealizado vs real
- **Impacto:** Autenticidade comprometida
- **Probabilidade:** MÉDIA
- **Mitigação:**
  - Triangulação externa obrigatória (entrevistas Indi, parceiros, alunos)
  - Validação por terceiros (beta testers que conhecem o sujeito)
  - Incluir contradições e limitações explicitamente
  - Documentar gaps de conhecimento honestamente

### HIGH Risks

**3. Complexidade Cognitiva Excepcional**
- **Risco:** ISTP 5w4, Estrato VI’VII, raridade 0.5% populacional
- **Impacto:** Dificuldade em capturar nuances cognitivas raras
- **Probabilidade:** MÉDIA
- **Mitigação:**
  - Usar profile psicométrico como estrutura base (já mapeado)
  - Frameworks originais como âncoras (Pareto³, DNA Mental")
  - Aceitar MVP com 70% fidelidade vs buscar perfeição
  - Iteração contínua com feedback direto do sujeito

**4. Integração de Duas Personas Distintas**
- **Risco:** IA Expert (engenheiro) vs Vida Lendária (filósofo) podem conflitar
- **Impacto:** Clone esquizofrênico ou inconsistente
- **Probabilidade:** MÉDIA
- **Mitigação:**
  - Estrutura clara: Generalista integrado + 2 especialistas puros
  - Generalista foca em zona de síntese (genialidade transdisciplinar)
  - Especialistas isolados para contextos específicos
  - Documentar critérios de uso para cada versão

### MEDIUM Risks

**5. Expectativas Irrealistas de Usuários**
- **Risco:** Usuários esperam oráculo vs ferramenta de suporte
- **Impacto:** Decepção, uso inadequado
- **Probabilidade:** MÉDIA
- **Mitigação:**
  - Comunicação clara: clone = auxílio, não substituição
  - Disclaimer explícito sobre limitações
  - Educar usuários sobre casos de uso adequados
  - Feedback loop para calibrar expectativas

**6. Evolução Rápida do Sujeito**
- **Risco:** Alan está em fase de crescimento acelerado (insights novos frequentes)
- **Impacto:** Clone desatualizado em 6-12 meses
- **Probabilidade:** ALTA
- **Mitigação:**
  - Versionamento claro de system prompts
  - Plano de atualização trimestral (coletar novo conteúdo)
  - Knowledge base modular (fácil adicionar chunks novos)
  - Aceitar que clone representa "snapshot temporal"
