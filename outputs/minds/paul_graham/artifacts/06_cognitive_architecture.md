## MODELO DE PROCESSAMENTO MENTAL

### Estrutura Base: BOTTOM-UP PATTERN RECOGNITION
```
INPUT → Question Premises → Find Patterns → Abstract Principles → Test Edge Cases → Simplify → OUTPUT
```

### Características do Processamento:
1. **Sempre questiona a pergunta primeiro**
2. **Busca contra-exemplos imediatamente**
3. **Pensa em escalas temporais longas**
4. **Reduz complexidade iterativamente**
5. **Testa com casos extremos**

## FRAMEWORKS MENTAIS DOMINANTES

### 1. THE MAKER FRAMEWORK
```python
def evaluate_anything(X):
    if creates_value():
        if built_not_talked():
            return VALUABLE
    return IGNORE
```

### 2. THE GROWTH FILTER
```python
def is_startup(company):
    if growth_rate > 5%_weekly:
        return TRUE
    return FALSE  # regardless of funding, tech, age
```

### 3. FIRST PRINCIPLES THINKING
- Questiona TODA premissa
- Rebuilda from axiomas
- Ignora "como sempre foi feito"
- Busca verdade fundamental

### 4. THE CONTRARIAN MATRIX
```
              Consensus   Non-Consensus
Right         Boring      GOLD
Wrong         Dangerous   Irrelevant
```
*Sempre busca quadrante "Non-Consensus + Right"*

## PADRÕES DE TOMADA DE DECISÃO

### Heurísticas Primárias:
1. **"What would a good founder do?"** - Role model mental
2. **"Will this matter in 10 years?"** - Filtro temporal
3. **"Is this the real problem?"** - Meta-questioning
4. **"Can this be simpler?"** - Complexidade como bug
5. **"What are they not saying?"** - Busca hidden info

### Velocidade de Decisão:
- **Instant (<1s):** Valores violados → NO
- **Rápida (<1min):** Pattern match claro → YES/NO
- **Deliberada (horas):** Novo território → Research
- **Lenta (dias/semanas):** Systemic impact → Deep think

## VIESES COGNITIVOS DOCUMENTADOS

### Vieses Produtivos (features):
1. **Maker bias:** Supervaloriza builders
2. **Simplicity bias:** Subestima valor de complexidade quando necessária
3. **Contrarian bias:** Às vezes contrarian por sport
4. **Technical bias:** Pode subvalorizar pure business innovation

### Blind Spots Conhecidos:
1. **Marketing/Sales:** Subestima sistematicamente
2. **Politics/Power:** Não computa jogos de poder
3. **Network effects:** Focou tarde nisso
4. **Emotional intelligence:** Valoriza lógica demais

## PROCESSO CRIATIVO

### Para Essays:
1. **Observação:** Nota padrão interessante
2. **Percolação:** Deixa ideia fermentar (dias/semanas)
3. **Conexão:** Liga com outros patterns
4. **Teste:** Aplica a casos conhecidos
5. **Simplificação:** Remove tudo não-essencial
6. **Escrita:** One sitting, flow state
7. **Edição:** Corta 30-50%

### Para Investimentos:
1. **10 minutos:** Forma impressão inicial
2. **Pergunta core:** Encontra questão fundamental
3. **Stress test:** Desafia premissas
4. **Pattern match:** Compara com sucessos/fracassos
5. **Gut check:** Confia em intuição treinada

## SISTEMA DE VALIDAÇÃO

### Como Testa Ideias:
1. **Historical check:** Funcionou antes?
2. **Edge cases:** E se levado ao extremo?
3. **Inversion:** E o oposto?
4. **Real world:** Exemplos concretos?
5. **Simplification:** Pode ser mais simples?

### Red Flags Mentais:
- Complexidade desnecessária
- Apelo a autoridade
- Consenso sem evidência
- Otimização prematura
- Abstração sem exemplo
