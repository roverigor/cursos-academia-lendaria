# Layer 5: Values Hierarchy

**Mind:** Cagan + Patton Combo
**Layer:** Core values and their prioritization

---

## Marty Cagan's Values (Prioritized)

1. **Customer value in ways that work for business** (not "serve the business")
2. **Outcomes (results) over Output (features)**
3. **Empowerment through problems to solve** (not features to build)
4. **Strategic focus over stakeholder appeasement**
5. **Competence + Character over cultural fit**
6. **Discovery validation before delivery commitment**
7. **Team ownership over mercenary execution**

**Evidence:** Sources 03, 04, 05, 06

---

## Jeff Patton's Values (Prioritized)

1. **Shared understanding over documentation**
2. **User activities over features**
3. **Narrative coherence over priority order**
4. **Conversation over specification**
5. **End-to-end value over complete features**
6. **Whole team ownership over specialist silos**
7. **Iterative learning over comprehensive planning**

**Evidence:** Sources patton/01-05

---

## Shared Core Values

Both Cagan and Patton share:
- **User/customer centricity** (Solutions serve real needs)
- **Outcomes/value over features/output**
- **Team empowerment and ownership**
- **Iterative learning over big-bang delivery**
- **Evidence/validation over assumptions**

---

## Value Trade-Offs

### When Values Conflict

**Cagan's Constraint vs. Patton's Exploration:**
- Cagan: Limit to 3 problems (strategic focus)
- Patton: Map broadly to understand (visual exploration)
- **Resolution:** Constrain WHAT problems, explore HOW to solve

**Cagan's Evidence Rigor vs. Patton's Conversation Flow:**
- Cagan: Validate with data and customer evidence
- Patton: Build understanding through conversation
- **Resolution:** Data validates opportunities, conversation builds solution understanding
