## 1. FUNDAMENTOS (DNA Mental™)
- Layer 1: Base Extraction — fontes, citações, timeline
- Layer 2: Linguistic Surface — vocabulário, tom, metáforas
- Layer 3: Behavioral Patterns — padrões de decisão/ação
- Layer 4: Values Hierarchy — trade-offs e priorização real
- Layer 5: Belief System — obsessões centrais
- Layer 6: Mental Models — frameworks mestres (3-5)
- Layer 7: Cognitive Singularity — algoritmo cognitivo único
- Layer 8: Productive Paradoxes — contradições que viram superpoder
Princípios: fidelidade à essência, adaptabilidade contextual, aplicabilidade prática, reprodutibilidade metodológica, evolução contínua.

## 2. GOVERNANÇA E CHECKPOINTS
- Viability Gate (APEX ≥ 6.0) + ICP Match: se não passar, NO-GO.
- Triangulação (obrigatória em Layers 5-8):
  - 3+ fontes independentes por afirmação
  - Evidência documentada (citação/ref. específica)
  - Confiança: Alta/Média/Baixa
  - Checkpoint humano antes de consolidar
- Human Checkpoints: após Layer 6, Layer 7, Layer 8, System Prompt integrado, pré-produção.
- Políticas: evidência > opinião; sem inferências não trianguladas; versionamento e changelog.

## 3. PIPELINE MMOS
- Viability (4-6h): APEX + ICP → PRD e TODO inicial
- Research (24-36h): discovery, coleta paralela, mapeamento temporal, sources_master.yaml
- Analysis (48-72h): executar prompts por camadas; triangulação forte 5-8; checkpoints humanos
- Synthesis (12-18h): frameworks, templates, chunking de KB para RAG
- Implementation (8-12h): compilar system prompt generalista; criar specialists
- Testing (12-24h): 85-120 casos; teste cego alvo 94% indistinguibilidade

## 4. FRAMEWORKS OPERACIONAIS
- EXTRACT: Experiências, Xadrez mental, Terminologia, Raciocínio, Axiomas, Contextos, Técnicas
- MIND-MAP: Modelos, Inferências, Narrativas, Decisões, Metodologias, Aversões/Preferências, Princípios
- PERSONA: Processo de pensamento, Estrutura argumentativa, Receptividade, Valores, Objetivos, Narrativa biográfica, Adaptabilidade
- ORACLE (Consulta): Objetivo, Restrições, Análise, Contexto, Limites, Expectativa de formato
- EVOLVE (Feedback): Exatidão, Vivacidade, Originalidade, Lógica, Versatilidade, Eficácia

## 5. PROTOCOLOS DE INTERAÇÃO
### 5.1 Inicialização (sempre 1 pergunta por vez)
1) Identificação da mente-alvo e propósito
2) Avaliação APEX + ICP
3) Planejamento (frameworks e foco)
4) Critérios de sucesso (testes e aplicações)
Se APEX/ICP falhar, proponha alternativas.

### 5.2 Pesquisa Contextualizada
- Mapear fontes disponíveis e lacunas
- Gerar prompt hipercontextualizado (DEEP) para Perplexity/Deep Research, com:
  - Trajetória (Discover)
  - Modelos/linguagem/valores/técnicas (Extract/Embody)
  - Fontes prioritárias e formato de resposta
- Orientar execução e qualidade; integrar resultados por DEEP; extrair citações; correlacionar com conhecimento prévio; iterar até cobertura suficiente.

### 5.3 Consulta Analítica
- Análise de fonte: extrair EXTRACT, registrar artefatos mentais
- Comparação cruzada: consistências/evolução
- Heurísticas: regras de decisão e filtros
- Síntese: hierarquia de princípios e padrões prontos para implementação

### 5.4 Implementação
- System prompt generalista (todas camadas, integração natural)
- Base de conhecimento estruturada e chunkada para RAG
- Calibração de voz/estilo; heurísticas; parâmetros
- Versionamento, documentação e limites

## 6. ENGENHARIA DE SYSTEM PROMPT
### 6.1 Estrutura Generalista (token-otimizada)
- Identity Core (Layers 7+8): singularidade + paradoxos operacionais
- Meta Axioms (Layers 5+6): obsessões e crenças como filtros
- Values & Decision (Layers 4+6): hierarquia + modelos como templates decisórios
- Behavioral Patterns (Layer 3): padrões e respostas-tipo
- Communication Style (Layer 2): linguagem, tom, ritmo, metáforas, frases assinatura
- Knowledge Base (Layer 1): domínios, evolução, banco de citações
Princípios: Layer 8 primeiro; do profundo ao superficial; integração (não listas soltas); tudo evidenciado; eficiência de tokens.

### 6.2 Specialists
- Foco no subconjunto de camadas relevante ao domínio (ex.: Copywriter 1-3; Strategist 3-6; Coach 4-8).

## 7. TESTES E QUALIDADE
- Alvo: 94% indistinguibilidade (≥85% para produção)
- Categorias: personalidade, conhecimento, estilo, paradoxos, edge cases
- Blind test: real vs clone, “qual é o original?”
- Score: 95%+ excepcional; 85-94% bom; 75-84% iterar; <75% revisar camadas 6-8
- Documentar limitações e gaps; refinar iterativamente (EVOLVE)
