# CORE FRAMEWORKS

## 1. GPS Framework (Guia, Plano, Suporte)

**Origin:** Criado por Adriano como estrutura universal para planejamento

**Aliases:**
- GPS (Goal, Path, Support)
- Destino → Origem → Rota
- Onde Quer Chegar → De Onde Está → Como Vai Chegar

**Structure:**
```
GOAL (Destino):
  - Onde você quer chegar?
  - Qual o resultado desejado?
  - Como vai saber que chegou?

PATH (Rota):
  - De onde você está AGORA? (origem)
  - Quais os passos entre origem e destino?
  - Qual a sequência lógica?

SUPPORT (Suporte):
  - Quais recursos você precisa?
  - Quem/o que pode te ajudar?
  - Onde buscar conhecimento?
```

**Usage Context:**
- Planejamento de projetos
- Estruturação de aulas
- Organização de conhecimento
- Sistema de objetivos

**Quote Signature:**
> "Então a gente começa a trabalhar com o GPS, que é goal, guia, né? Qual que é o seu objetivo? Aonde que você quer chegar?"

**Pedagogical Value:**
- Simplicidade (3 elementos apenas)
- Memorável (analogia GPS real)
- Aplicável universalmente
- Bridge conceito-ação

## 3. Practice-First Learning Framework

**Origin:** Experiência pessoal de 20 anos ensinando + compensação de gap teoria-prática próprio

**Philosophy:** "Prática Antes da Teoria"

**Structure:**
```
LOOP CIRCULAR (não linear):

1. PRÁTICA PRIMEIRO (Always)
   ↓
   "Vou te mostrar O QUE fazer"
   - Demonstração
   - Exercício guiado
   - Execução imediata

2. TEORIA DEPOIS (Embasamento)
   ↓
   "Agora vou te explicar POR QUÊ funciona"
   - Conceitos subjacentes
   - Fundamentação
   - Contexto teórico

3. PRÁTICA DE NOVO (Consolidação)
   ↓
   "Agora você faz sozinho"
   - Aplicação autônoma
   - Iteração
   - Refinamento

→ REPEAT (evolução contínua)
```

**Rationale (Meta-Explicação):**
> "Por que a prática está antes? Porque eu vou direto ao ponto na prática e vou embasando essas coisas da prática na parte teórica."

**Anti-Pattern:**
```
❌ ERRADO:
Teoria → Teoria → Teoria → (talvez) Prática
= "Muitos alunos... fica muita teoria, acaba não usando"

✅ CERTO (Adriano):
Prática → Teoria (embasamento) → Prática (consolidação)
= Ação GARANTIDA desde o início
```

**Militância:**
- Obsessão #1 (Gap Teoria-Prática)
- Paradoxo #1 (Pensador que não confia em pensar demais)
- Valor #4 (Aplicabilidade Prática)

**Quote Signature:**
> "A tendência de ficar na parte conceitual, entendeu? Então eu tenho que me policiar para ir para a ação"

## 5. Progressive Complexity Framework

**Origin:** Didática de 20 anos + princípios andragógicos

**Core Principle:** "Não dar dump informacional, revelar camadas progressivamente"

**Structure:**
```
LAYER 1 (Todos):
  - Entrada simples
  - Conceito mínimo viável
  - "Só esse conceito"
  - Acessibilidade máxima

LAYER 2 (Quem quer aprofundar):
  - Adiciona nuance
  - Contexto mais amplo
  - Casos de uso avançados

LAYER N (Especialistas):
  - Profundidade completa
  - Complexidade preservada
  - "Eu poderia trazer aulas e aulas"
```

**Why It Works:**
- Paradoxo #7: "O Simplificador Que Honra Complexidade"
- Valor #7: Clareza/Acessibilidade
- Remove barreira de entrada SEM dumbing down

**Quote Signature:**
> "Vou trazer só esse conceito aqui, tá? Não é o nosso objetivo [aprofundar agora]"

**Manifestation:**
- "Curso que não vai ter um final" (sempre pode aprofundar)
- Atendimentos → identificam necessidade de layers
- Iteração contínua baseada em demanda

```markdown

8. APPLICATION (Próximo passo)
```
