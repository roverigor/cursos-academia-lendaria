## 3. TECHNICAL REQUIREMENTS

### 3.1 AI Model Specifications

- **Minimum:** GPT-4 or Claude 3.5 Sonnet
- **Optimal:** Claude 3.5 Opus or GPT-4 Turbo
- **Context Window:** 50,000+ tokens required
- **Temperature:** 0.7-0.8 for creativity, 0.3-0.4 for analysis
- **Response Time:** <3 seconds for standard queries

### 3.2 Knowledge Base Requirements

- **Core Knowledge:** 50+ structured concepts
- **Examples Database:** 100+ annotated examples
- **Case Studies:** 20+ detailed analyses
- **Templates:** 30+ reusable structures
- **Update Frequency:** Quarterly enrichment

### 3.3 Integration Capabilities

- API accessibility
- Webhook support
- Batch processing
- Export formats (MD, TXT, JSON)
- Version control for outputs

## 6. CONSTRAINTS & LIMITATIONS

### 6.1 Technical Constraints

- Cannot access real-time market data
- No direct integration with ad platforms
- Limited to text-based interactions
- Cannot create visual designs
- No direct testing capability

### 6.2 Ethical Boundaries

- Clear identification as AI
- No false claims of being Schwartz
- Respect for intellectual property
- No harmful or deceptive content
- Privacy protection for users

### 6.3 Scope Limitations

- Focus on Schwartz methodology only
- English language only (initially)
- Text-based copy only
- B2C and B2B (no B2G)
- Legal industries only
