# 1.2 - Instalação + Config

**Duração:** 10 min | **Nível:** Iniciante

## Objetivo

Instalar Git no seu sistema operacional e configurar identidade profissional (nome, email, editor).

## Por que isso importa

Uma configuração correta desde o início evita commits com "unknown author" e garante que suas contribuições sejam creditadas corretamente no GitHub. Sua identidade Git aparece em cada commit que você faz.

## Conteúdo

### Instalação por Sistema Operacional

#### Windows

**Opção recomendada:** Git for Windows

1. Acesse [git-scm.com/download/win](https://git-scm.com/download/win)
2. Baixe o instalador (64-bit)
3. Execute e aceite as opções padrão **exceto**:
   - Editor: Escolha VS Code (se tiver instalado) ou Nano
   - Branch inicial: `main` (não `master`)
   - Terminal: Use Git Bash

4. Verifique instalação:
```bash
git --version
# Deve mostrar: git version 2.43.0 ou superior
```

#### macOS

**Opção 1 - Homebrew (recomendado):**
```bash
# Instale Homebrew primeiro se não tiver: https://brew.sh
brew install git
```

**Opção 2 - Xcode Command Line Tools:**
```bash
xcode-select --install
```

Verifique:
```bash
git --version
# Deve mostrar: git version 2.39.0 ou superior
```

#### Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install git
```

**Linux (Fedora):**
```bash
sudo dnf install git
```

Verifique:
```bash
git --version
```

### Configuração Essencial

Após instalar, configure sua identidade global. Essas informações aparecem em **todos** os seus commits.

#### 1. Nome e Email

```bash
# Use seu nome real (empresas valorizam transparência)
git config --global user.name "Maria Silva"

# Use o email da sua conta GitHub
git config --global user.email "maria.silva@email.com"
```

**Importante:** Use o mesmo email que você cadastrou no GitHub. Se usar email diferente, suas contribuições não aparecem no seu perfil.

**Dica pro:** GitHub permite email privado:
- Vá em Settings → Emails
- Marque "Keep my email addresses private"
- Use o email no formato: `username@users.noreply.github.com`

```bash
git config --global user.email "mariasilva@users.noreply.github.com"
```

#### 2. Editor Padrão

Git abre editor para mensagens de commit longas. Configure seu favorito:

**VS Code:**
```bash
git config --global core.editor "code --wait"
```

**Nano (simples, bom para iniciantes):**
```bash
git config --global core.editor "nano"
```

**Vim (se você já sabe sair dele):**
```bash
git config --global core.editor "vim"
```

#### 3. Branch Padrão

Configure `main` como branch padrão (padrão atual da indústria):

```bash
git config --global init.defaultBranch main
```

#### 4. Cores no Terminal

Facilita leitura dos comandos:

```bash
git config --global color.ui auto
```

### Verificando Configuração

Liste todas as configurações:

```bash
git config --list
```

Saída esperada:
```
user.name=Maria Silva
user.email=maria.silva@email.com
core.editor=code --wait
init.defaultbranch=main
color.ui=auto
```

Ver configuração específica:
```bash
git config user.name
# Retorna: Maria Silva
```

### Onde Fica Salvo?

**Global** (para todos os repos): `~/.gitconfig`

```bash
# Ver arquivo de config
cat ~/.gitconfig
```

**Local** (só para um repo específico): `.git/config` dentro do repo

```bash
# Sobrescrever config só neste repo
git config user.email "outro@email.com"  # sem --global
```

### Configurações Extras (Opcional mas Recomendado)

```bash
# Sempre fazer pull com rebase (evita merge commits desnecessários)
git config --global pull.rebase true

# Usar credencial helper (evita digitar senha toda hora)
# Windows:
git config --global credential.helper wincred

# macOS:
git config --global credential.helper osxkeychain

# Linux:
git config --global credential.helper cache
```

## Exercício Prático

1. **Instale Git** no seu sistema operacional
2. **Configure sua identidade**:
   ```bash
   git config --global user.name "Seu Nome"
   git config --global user.email "seu@email.com"
   git config --global core.editor "code --wait"
   git config --global init.defaultBranch main
   ```

3. **Verifique tudo**:
   ```bash
   git config --list
   ```

4. **Tire screenshot** da saída e salve (você vai precisar confirmar isso depois)
