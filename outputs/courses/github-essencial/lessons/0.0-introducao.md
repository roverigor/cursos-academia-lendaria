## O Que É Git — A História Completa

Antes de falar de GitHub, você precisa entender **Git**. E para entender Git, precisa conhecer **Linus Torvalds** e **Linux**.

### Linus Torvalds e Linux

**1991:** Linus Torvalds, um estudante finlandês de 21 anos, cria o **Linux** — um sistema operacional de código aberto.

```
┌─────────────────────────────────────────────────────────────┐
│                    SISTEMAS OPERACIONAIS                     │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Windows        macOS          Linux                         │
│  (Microsoft)    (Apple)        (Linus Torvalds)             │
│                                                              │
│  Pago           Pago           Gratuito e aberto            │
│  Código fechado Código fechado Código aberto                │
│                                                              │
│  Usado em:      Usado em:      Usado em:                     │
│  • PCs          • Macs         • 90%+ dos servidores        │
│  • Empresas     • iPhones      • Android                     │
│                                • Supercomputadores           │
│                                • Nuvem (AWS, Google, Azure)  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**Fato:** Quando você acessa Google, Netflix, Amazon, Instagram — você está usando Linux nos bastidores.

### O Problema de Linus (1991-2002)

Linux cresceu. Milhares de desenvolvedores no mundo todo queriam contribuir.

**Como gerenciar código de milhares de pessoas?**

```
1991-2002: O CAOS ORGANIZADO

Desenvolvedor A ──────┐
Desenvolvedor B ──────┼──► Linus ──► "Deixa eu juntar isso manualmente..."
Desenvolvedor C ──────┤
Desenvolvedor D ──────┘

• Contribuições enviadas por email
• Linus juntava tudo manualmente
• Sem histórico organizado
• Conflitos resolvidos na mão
• Funcionava, mas era trabalhoso
```

**Por 11 anos**, Linus gerenciou as contribuições assim. Heroico, mas insustentável.

### BitKeeper: A Solução Temporária (2002-2005)

Em 2002, Linux adotou **BitKeeper** — um sistema de controle de versão proprietário (pago, código fechado).

**O acordo:** BitKeeper seria gratuito para Linux, mas com restrições:
- Não podia trabalhar em ferramentas concorrentes
- Não podia fazer engenharia reversa
- Licença podia ser revogada a qualquer momento

**A controvérsia:** Richard Stallman (fundador do movimento software livre) criticou: *"Um projeto livre usando ferramenta proprietária?"*

A comunidade aceitou. Funcionou por 3 anos.

### A Crise de Abril de 2005

Em 2005, um desenvolvedor chamado **Andrew Tridgell** criou uma ferramenta que se comunicava com BitKeeper. A empresa dona (BitMover) considerou isso violação da licença.

**Resultado:** Licença gratuita **revogada**. Linux ficou sem ferramenta.

```
ABRIL 2005: A CRISE

BitMover: "Vocês violaram nossa licença. Acabou o BitKeeper gratuito."

Comunidade Linux: "E agora? Voltamos para email?"

Linus: "Não. Vou criar minha própria ferramenta. Uma que ninguém possa tirar de nós."
```

### Git Nasce em 10 Dias

Linus Torvalds fez o que faz melhor: resolveu o problema ele mesmo.

| Data | Marco |
|------|-------|
| **3 Abril 2005** | Linus começa a desenvolver Git |
| **7 Abril 2005** | Primeiro commit (Git gerenciando a si mesmo) |
| **18 Abril 2005** | Primeiro merge de múltiplos branches |
| **16 Junho 2005** | Git gerencia lançamento do Linux Kernel 2.6.12 |
| **26 Julho 2005** | Linus passa manutenção para Junio Hamano |

**Em aproximadamente 10 dias**, Linus criou o protótipo funcional de uma ferramenta que revolucionaria o desenvolvimento de software.

> "O truque não foi tanto a codificação, mas descobrir como organizar os dados."
> — Linus Torvalds

### O Significado do Nome "Git"

**A piada de Linus:**

> "Eu sou um bastardo egoísta, e nomeio todos os meus projetos com meu nome. Primeiro 'Linux', agora 'git'."
> — Linus Torvalds

**"Git" em gíria britânica significa:** pessoa desagradável, teimosa, que acha que está sempre certa. Linus estava fazendo uma piada autodepreciativa.

**No primeiro commit, Linus escreveu no README:**

```
"GIT - the stupid content tracker"
(GIT - o rastreador de conteúdo estúpido)

"'git' pode significar qualquer coisa, dependendo do seu humor"
```

**As 4 interpretações oficiais do nome:**

| Seu Humor | Git Significa |
|-----------|---------------|
| **Neutro** | Combinação aleatória de 3 letras pronunciável |
| **Negativo** | Estúpido, desprezível, simples (gíria britânica) |
| **Positivo** | **G**lobal **I**nformation **T**racker |
| **Com raiva** | **G**oddamn **I**diotic **T**ruckload of sh*t |

### O Que Git Realmente É

**Git é um sistema de controle de versão distribuído.**

```
CENTRALIZADO (antigo)              DISTRIBUÍDO (Git)
─────────────────────              ─────────────────────

      [Servidor]                   [GitHub]     [GitLab]
          │                            │            │
    ┌─────┼─────┐                     │            │
    │     │     │               ┌─────┴─────┬──────┴─────┐
    ▼     ▼     ▼               ▼           ▼            ▼
   PC1   PC2   PC3          [PC1 FULL]  [PC2 FULL]  [PC3 FULL]
  (cópia (cópia (cópia      (histórico  (histórico  (histórico
  parcial)parcial)parcial)   COMPLETO)   COMPLETO)   COMPLETO)

• Servidor cai = ninguém      • Cada PC tem backup completo
  trabalha                    • Funciona 100% offline
• Sempre precisa de conexão   • Servidor é conveniência, não
                                dependência
```

**Por que isso importa:**
- Você trabalha offline no avião? Git funciona.
- Servidor caiu? Você tem tudo local.
- Quer experimentar sem medo? Crie um branch local.

## A História do GitHub

Agora que você entende Git, vamos ao GitHub.

### O Problema (2007)

Era 2007. Git existia há 2 anos. Era poderoso, rápido, revolucionário.

**Mas tinha um problema:** era difícil de usar colaborativamente.

> "Git was a really cool way of working with other people on code, but **there wasn't an easy way to get up and running with a group of developers**."
> — Tom Preston-Werner, co-fundador do GitHub

**Dores reais:**
- Compartilhar repositórios era complicado
- Não existia interface visual amigável
- Code review era trabalhoso
- Descobrir projetos open source era difícil

Git era para experts de linha de comando. E a maioria dos desenvolvedores não era expert.

### A Faísca — Um Bar em San Francisco

**2007.** Tom Preston-Werner e Chris Wanstrath se encontraram em um bar para falar sobre "vida e software". A conversa virou para Git.

**A pergunta que mudou tudo:**
> "Por que não tornar Git mais fácil e colaborativo?"

Naquela noite, nasceu a ideia do GitHub.

### O Significado do Nome

**Git** + **Hub** = **GitHub**

| Termo | Significado |
|-------|-------------|
| **Git** | O sistema de controle de versão |
| **Hub** | Centro, ponto de encontro |
| **GitHub** | O ponto de encontro para repositórios Git |

```
Git = Motor do carro          GitHub = Concessionária + Oficina +
      (roda local)                    Comunidade de motoristas
                                      (na nuvem, com serviços)
```

### A Filosofia

> "Mergulhe em algo, descubra onde dói, e então escreva algo para resolver essa dor."
> — Tom Preston-Werner

Os fundadores eram desenvolvedores. Eles sentiram a dor de usar Git. E criaram a solução.

**Decisão crucial:** Tom recusou **$300.000 de bônus + stock options da Microsoft** para focar no GitHub. Uma aposta que valeu bilhões.

### A Ironia da Microsoft

Em 2008, Tom recusou bônus da Microsoft para construir o GitHub.

Em 2018, a Microsoft comprou o GitHub por **$7.5 bilhões**.

**O que mudou?**

Satya Nadella (CEO da Microsoft desde 2014) entendeu que:
- Desenvolvedores são o futuro
- Open source venceu
- GitHub era a comunidade

**Resultado:** GitHub continua operando de forma independente, agora com recursos da Microsoft. VS Code, Copilot, Actions — tudo integrado.

## Seu Primeiro Passo

Antes de ir para a próxima lição, faça isso:

### Exercício: Prepare Seu Ambiente

1. **Abra o terminal:**
   - Windows: PowerShell ou Git Bash
   - Mac: Terminal
   - Linux: Terminal

2. **Verifique se Git está instalado:**
   ```bash
   git --version
   ```

   - Se aparecer versão (ex: `git version 2.43.0`): Você está pronto
   - Se aparecer erro: Não se preocupe, instalaremos na próxima lição

3. **Crie conta no GitHub** (se ainda não tem):
   - Acesse [github.com](https://github.com)
   - Use email profissional
   - Escolha username que você usará por anos

4. **Prepare um projeto simples:**
   - Pode ser qualquer código que você tenha
   - Ou crie uma pasta vazia para praticar
   - Vamos versioná-lo durante o curso
