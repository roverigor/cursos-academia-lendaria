## PASSO 2: Aplicar GPS (10 min)

Agora abre o template GPS (tem nos recursos).

### 2.1: DESTINO (Para onde vai?)

**Pergunta:** "O que o aluno vai SABER FAZER após esta aula?"

Exemplo: "Salvar versões do código usando git commit"

**Agora aplica 5 Porquês:**

1. Por quê salvar versões? → Não perder código
2. Por quê não perder código? → Evitar refazer
3. Por quê evitar refazer? → Economizar tempo
4. Por quê economizar tempo? → Focar no que importa
5. Por quê focar? → **Crescer, ter vida, família**

**ESSÊNCIA: "Ter tempo pra vida"**

### 2.2: ORIGEM (Onde aluno está?)

**Pergunta:** "O que ele JÁ sabe? O que NÃO sabe?"

Exemplo:
- JÁ sabe: O que é Git (aula anterior)
- NÃO sabe: Como salvar versão (commit)
- DOR: Já perdeu código, refez trabalho

### 2.3: ROTA (Como chegar lá?)

**Sequência lógica:**
1. Por quê commit importa (motivação)
2. O que é commit (conceito)
3. Como fazer commit (prática)

**GPS pronto. 10 minutos.**

---

## PASSO 3: Definir Conceitos (5 min)

**Quantos conceitos vou ensinar?**

**Lembra: 2-3 MÁXIMO.**

Exemplo (Git Commit):
1. **Conceito 1:** O que é commit
2. **Conceito 2:** Comando git commit
3. **Conceito 3:** Mensagem de commit (boas práticas)

**3 conceitos. OK.**

Se tivesse 4? **DIVIDIR.**

---

## PASSO 4: Criar Semiótica (10 min)

**Para CADA conceito: pelo menos 1 ferramenta.**

**Conceito 1: O que é commit**
- **Analogia:** "Commit é como foto do código. Você tira foto. Se estragar, volta pra foto."

**Conceito 2: Comando git commit**
- **Exemplo prático:** "git add . (prepara) → git commit -m 'mensagem' (salva)"

**Conceito 3: Mensagem de commit**
- **Contra-exemplo:** ❌ "mudanças" ✅ "Adiciona validação de email no formulário"

**Semiótica pronta. 10 minutos.**

---

## PASSO 5: Criar Prática (5 min)

**Exercício CLARO e ESPECÍFICO.**

Exemplo:
"Pausa o vídeo. Faz o seguinte:
1. Cria arquivo teste.txt
2. git add teste.txt
3. git commit -m 'Adiciona arquivo teste'
4. git log (confere que salvou)
5. Volta aqui"

**Prática: 5-10 min pro aluno fazer.**

---

## PASSO 6: Escrever Script dos 7 Elementos (20 min)

Agora JUNTA TUDO na estrutura de 7 partes:

### Elemento 1: MOTIVAÇÃO (1-2 min de script)
"Você já perdeu código? Trabalhou 3 horas. Fechou sem salvar. Ou pior: salvou ERRADO, estragou tudo, não tem como voltar? Hoje acaba. Git commit."

### Elemento 2: CONCEITO (2-3 min)
"Commit é [explicação técnica limpa]"

### Elemento 3: SEMIÓTICA (1-2 min)
"Pensa assim: commit é foto do código..."

### Elemento 4: PRÁTICA (2-3 min)
"Agora VOCÊ. Pausa..."

### Elemento 5: LINK (1 min)
"Olha. Commit não é sobre Git. É sobre TEMPO DE VOLTA. Toda vez que você não precisa refazer código..."

### Elemento 6: REVISÃO (30 seg)
"Recap: Commit salva versão (conceito). É como foto (semiótica). Você fez seu primeiro commit (prática)."

### Elemento 7: HOOK (30 seg)
"Mas commit salva LOCAL. E se seu HD queimar? Perde tudo. Próxima aula: Git push. Backup na nuvem."

**Script completo. 20 minutos.**
