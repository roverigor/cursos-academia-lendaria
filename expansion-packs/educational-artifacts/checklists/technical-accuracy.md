# Technical Accuracy Validation Checklist

Validate that all technical content is correct, current, and appropriate for the target audience.

## Code Examples & Snippets

- [ ] All code examples are syntactically correct
- [ ] Code examples follow best practices for the language/framework
- [ ] Code examples use current/stable versions (not deprecated syntax)
- [ ] Code comments are clear and helpful
- [ ] Code can be copy-pasted and executed without modifications
- [ ] Error handling is demonstrated (when relevant)
- [ ] Security best practices demonstrated (when applicable)
- [ ] Performance considerations noted (when significant)

## Technical Terminology

- [ ] All technical terms are defined before use
- [ ] Terminology is used consistently throughout
- [ ] Acronyms are explained at first use
- [ ] Jargon is avoided or translated to plain language
- [ ] Industry-standard terms are used (not invented synonyms)
- [ ] Context makes terminology clear

## References & Citations

- [ ] External references are current and accessible
- [ ] Links are not broken (spot-check critical ones)
- [ ] Citations are accurate and attributed correctly
- [ ] Statistics and data are from credible sources
- [ ] Publication dates are reasonable (not outdated)
- [ ] No misleading attribution or out-of-context citations

## Technical Depth

- [ ] Content depth appropriate for target audience
- [ ] Beginner content doesn't overwhelm with details
- [ ] Advanced content doesn't oversimplify
- [ ] Edge cases mentioned when important
- [ ] Common misconceptions addressed
- [ ] Limitations explained (where applicable)

## Libraries, Frameworks, Tools

- [ ] Correct versions referenced
- [ ] API usage is current and accurate
- [ ] Configuration examples are valid
- [ ] Installation instructions are up-to-date
- [ ] Alternatives or competing solutions mentioned (when relevant)

## Mathematics & Formulas

- [ ] All formulas are mathematically correct
- [ ] Variables are clearly defined
- [ ] Units are specified (where applicable)
- [ ] Calculations checked manually
- [ ] Assumptions stated explicitly

## System & Architecture Concepts

- [ ] System descriptions are accurate
- [ ] Diagrams match described architecture
- [ ] Data flow is correctly represented
- [ ] Dependencies are correctly shown
- [ ] Scalability implications noted (where relevant)
- [ ] Trade-offs mentioned (performance vs. simplicity, etc.)

## Real-World Applicability

- [ ] Examples reflect actual use cases
- [ ] Edge cases and limitations mentioned
- [ ] Compatibility considerations noted
- [ ] Performance characteristics disclosed
- [ ] Deployment/testing guidance provided
- [ ] Common pitfalls warned about

## Standards & Best Practices

- [ ] Industry standards referenced accurately
- [ ] Best practices are current
- [ ] Rationale given for practices recommended
- [ ] Alternatives discussed (when appropriate)
- [ ] Trade-offs explained
- [ ] Context for when not to follow practices

## Fact-Checking

- [ ] Technical facts verified against authoritative sources
- [ ] Historical information is accurate
- [ ] Current state of technology correctly described
- [ ] Future projections clearly marked as speculation
- [ ] Version-specific information clearly tagged

## Audience Alignment

- [ ] Content matches audience's technical level
- [ ] Prerequisite knowledge not assumed without stating
- [ ] Leap-of-faith explanations avoided
- [ ] Analogies are technically sound
- [ ] No dumbing down of legitimate complexity

## Common Technical Errors

- [ ] Off-by-one errors in examples
- [ ] Incorrect scope/visibility of variables
- [ ] Type mismatches not mentioned
- [ ] Null/undefined handling not glossed over
- [ ] Race conditions/concurrency issues addressed (when relevant)
- [ ] Resource leaks not introduced (in examples)

## Overall Technical Quality

- [ ] Content feels authoritative and knowledgeable
- [ ] No technical red flags or inconsistencies
- [ ] Would a subject matter expert approve this?
- [ ] Could someone new to the field trust this content?
- [ ] Missing important caveats or warnings?

---

**Validation Notes:**
When in doubt about technical accuracy, consult authoritative sources or subject matter experts. Better to under-teach than to teach incorrectly.
