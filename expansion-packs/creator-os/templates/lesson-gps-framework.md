## ðŸ“Š GPS Validation Checklist

Every lesson MUST have:

### Goal (Destino)
- [ ] Clear promise in first 60 seconds (what they'll achieve)
- [ ] Tangible, specific outcomes (not vague "understand")
- [ ] Motivation stated (why it matters)

### Position (Origem)
- [ ] Acknowledges different starting points
- [ ] Validates student concerns/struggles (empathy)
- [ ] Offers multiple learning paths/approaches

### Steps (Rota)
- [ ] Starts with "why" before "what"
- [ ] Clear, sequential steps
- [ ] Immediate practice/application exercise

### Semiotic Elements (Required)
- [ ] **Analogies:** Minimum 1, maximum 3 (too many = confusing)
- [ ] **Diagrams/Visuals:** Minimum 1 described or included
- [ ] **Reflective Questions:** Minimum 2, maximum 4
- [ ] **Visual Summary:** Bullets, not paragraphs

### Voice Fidelity (If instructor persona defined)
- [ ] Uses instructor's signature phrases (if applicable)
- [ ] Matches instructor's tone (formal/casual, warm/technical)
- [ ] Maintains instructor's teaching style (theory-first vs practice-first)

### Accessibility
- [ ] No jargon without explanation
- [ ] Complex concepts translated to familiar analogies
- [ ] Multiple representations (text + visual + practice)

## ðŸŽ¨ Semiotic Strategies Library

### Powerful Analogies by Domain

**Technology/Tools:**
- Files = Documents in a filing cabinet
- Variables = Labeled drawers/boxes
- Functions = Recipes (input ingredients, output dish)
- APIs = Restaurant menu (you order, kitchen prepares)
- Databases = Organized library with card catalog
- Cache = Sticky notes on your desk (quick access)

**Productivity/PKM:**
- Notes = Thoughts externalized
- Links = Neural connections in brain
- Tags = Colored labels/sticky tabs in book
- Folders = Rooms in a house (not too many!)
- Search = Asking librarian for help
- Templates = Cookie cutters (same shape, different cookies)

**Business/Strategy:**
- Marketing funnel = Water filtering system
- Customer journey = GPS navigation route
- Value proposition = Movie trailer (show best parts)
- Product-market fit = Key fitting lock perfectly
- Iteration = Sculptor refining statue (remove what doesn't fit)

**Learning/Education:**
- Spaced repetition = Watering plants regularly (not drowning once)
- Active recall = Closing book and explaining to friend
- Elaboration = Teaching concept to 5-year-old
- Interleaving = Mixing practice problems (not blocking)

### Diagram Types to Use

1. **Flowcharts:** Decision trees, workflows, processes
2. **Before/After:** Show transformation visually
3. **Concept Maps:** Relationships between ideas
4. **Timelines:** Sequential processes, learning paths
5. **Venn Diagrams:** Compare/contrast, overlaps
6. **Pyramids/Hierarchies:** Levels, priorities, foundations
7. **Cycle Diagrams:** Iterative processes, habits
8. **Annotated Screenshots:** Tool interfaces, step-by-step

## ðŸš€ Pro Tips for Lesson Writers

1. **Write Goal FIRST:** If you can't write a clear promise, the lesson isn't focused enough
2. **Analogies from Universal Experiences:** Don't use niche references (e.g., "like a blockchain merkle tree" - most don't know that)
3. **One Concept = One Analogy:** Don't mix metaphors (confuses instead of clarifies)
4. **Questions Create Engagement:** Every 5-7 minutes, force student to pause and reflect
5. **Show, Don't Just Tell:** Diagram > Paragraph for spatial/process concepts
6. **Voice Consistency:** If instructor says "TÃ¡?" every 3 sentences, AI should too (within reason)
7. **Actionable Immediately:** Student should be able to DO something in <2 minutes after lesson
