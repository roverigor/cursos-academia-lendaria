Data: 2025-10-20

Modo: Expert-Led

---

## Objetivo
Guia rápido para avaliar desempenho dos alunos por lesson, com rubricas objetivas, checkpoints e critérios de aprovação.

---

## Rubricas por Lesson (Resumo)

- Lesson 1.1 — A Máquina que Você Não Sabia Operar (10 min)
  - Evidências de aprendizagem:
    - Explica HTML/CSS/JS/API com a metáfora Casa FullStack
    - Identifica diferença Protótipo (Artifacts) vs Produção (Bolt)
  - Avaliação rápida:
    - Pergunta oral curta (2-3 min)
    - Mini-quiz: q1–q4 (ver Quiz M1)

- Lesson 1.2 — Mapa da Clareza (20 min)
  - Evidências:
    - Formulário criado e estilizado
    - API key configurada (Anthropic)
    - App processa texto e exibe categorias
  - Rubrica (0–10): 3/infra + 4/função + 3/clareza UI

- Lesson 2.1 — Bolt: Protótipo → Produção (15 min)
  - Evidências:
    - Landing publicada com URL acessível
    - Seções mínimas: Header, Hero, Benefícios, Footer
  - Rubrica (0–10): 4/publicação + 3/personalização + 3/estrutura

- Lesson 2.2 — Supabase (20 min)
  - Evidências:
    - Tabela criada no Supabase
    - App salva e lista dados
  - Rubrica (0–10): 4/schema + 3/env + 3/CRUD

- Lesson 2.3 — Auth (25 min)
  - Evidências:
    - Cadastro/login funcionais
    - Área protegida
  - Rubrica (0–10): 4/login + 3/sessão + 3/proteção rota

- Lesson 3.1 — MicroSaaS (20 min)
  - Evidências:
    - Assistants conectados (OpenAI)
    - Stripe modo teste funcionando
    - Fluxo Cadastro → Paga → Acessa
  - Rubrica (0–10): 3/IA + 4/Stripe + 3/fluxo

- Lesson 3.2 — Landing que Vende (10 min)
  - Evidências:
    - Estrutura Isca → Captura → Pitch
    - Form salva leads no Supabase
  - Rubrica (0–10): 4/estrutura + 3/copy CTA + 3/captura

## Checkpoints Sugeridos

- Após Módulo 1: validar quebra de barreira + primeiro app
- Após Módulo 2: validar publicação, DB e Auth funcionando
- Após Módulo 3: validar funil completo + plano de preços

---

## Boas Práticas

- Mantenha a voz do instrutor: use marcadores “Olha só…”, “Vem comigo…”, celebração de vitórias
- Reforce prática imediata (80% hands-on)
- Use troubleshooting para reduzir fricção
- Traga preço/valor sempre que possível (contexto de monetização)

## Critérios de Certificação

- Completar 7 lessons
- Atingir ≥80% em cada quiz de módulo
- Projeto final ≥70 pts
- Submeter 3 apps obrigatórios (Mapa da Clareza, App com Auth, Hub de GPTs)
