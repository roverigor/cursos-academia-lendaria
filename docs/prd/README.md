### Template Structure
1. **Overview** - Vision, objectives, target users
2. **Features** - Core capabilities and functionality
3. **Requirements** - Technical and business requirements
4. **Success Metrics** - KPIs and measurement criteria
5. **Timeline** - Phases and milestones

### Best Practices
- Focus on **why** (not just **what**)
- Define clear success criteria
- Include user personas and use cases
- Link to technical architecture docs
- Keep PRDs living documents (update as product evolves)
