### O Problema que Ninguém te Conta

Você já criou um clone básico. Ele funciona. Às vezes.

Mas tem um problema: ele é **inconsistente**.

Hoje ele responde uma coisa. Amanhã, a mesma pergunta, resposta completamente diferente.

Por quê?

Porque você está **descrevendo** quem a IA é, mas não está **prescrevendo** o que ela deve fazer.

É como contratar um funcionário e dizer: "Você é criativo e dedicado". Legal. Mas... o que diabos ele deve FAZER quando chega um cliente?

Nesta aula você vai aprender a diferença que separa um clone de R$ 800 de um de R$ 8.000.

---

### CONCEITO CENTRAL: O Poder da Prescrição

**DESCRITIVO = Dizer O QUE a IA é** **PRESCRITIVO = Dizer O QUE a IA deve fazer**

Vamos usar uma analogia simples:

**Imagine que você está ensinando alguém a fazer café.**

**Versão Descritiva (Ruim):** "Você é um barista experiente que faz cafés deliciosos e tem paixão por qualidade."

O que essa pessoa faz quando chega na máquina? Adivinha.

**Versão Prescritiva (Boa):** "QUANDO um cliente pedir café:

1. Pergunte: espresso, filtrado ou cappuccino?
    
2. Use 18g de café para espresso, 60g para filtrado
    
3. Água a 93°C sempre
    
4. Tempo de extração: 25-30 segundos para espresso
    
5. Nunca sirva café que passou de 30 minutos"
    

Vê a diferença? A segunda versão dá um **algoritmo**. Um passo a passo. Uma receita.

É exatamente isso que faremos com seu clone.

### TABELA COMPARATIVA: Descritivo vs Prescritivo

Aspecto Clone Descritivo (R$ 800) Clone Prescritivo (R$ 8.000) **Identidade** "Você é um especialista em vendas" "QUANDO receber pergunta sobre vendas, EXECUTE protocolo V3" **Estilo** "Você escreve de forma direta e clara" "Use frases de 10-15 palavras. Máximo 3 por parágrafo. Sempre termine com pergunta." **Conhecimento** "Você conhece estratégias de marketing" "ANTES de sugerir estratégia: 1) Identifique budget 2) Defina prazo 3) Aplique Framework ROAS" **Consistência** Degrada após 5 mensagens Mantém por 20+ com checkpoints **Resultado** "Às vezes funciona" "Funciona sempre (se bem feito)"

### OS 3 NÍVEIS DE INSTRUÇÕES

Pense em instruções como se fossem ordens militares. Existem 3 níveis:

**NÍVEL 1: Sugestão (Fraco)** "Seria legal se você fosse educado"

- A IA pode ignorar
    
- Depende do humor do modelo
    
- Inconsistente
    

**NÍVEL 2: Diretriz (Médio)** "Você deve ser educado nas respostas"

- A IA tenta seguir
    
- Mas pode esquecer em conversas longas
    
- Semi-consistente
    

**NÍVEL 3: Comando Prescritivo (Forte)** "OBRIGATÓRIO: Toda resposta DEVE começar com saudação e terminar com pergunta de follow-up. SE não souber a resposta, diga 'Não tenho informação suficiente sobre isso' e NUNCA invente."

- A IA é forçada a seguir
    
- Estrutura clara
    
- Altamente consistente
    

**No Clone Profissional, usamos APENAS Nível 3.**

### EXEMPLO PRÁTICO 1: Transformando Descrição em Prescrição

**ANTES (Descritivo - Clone de R$ 800):**

```
Você é Alex Hormozi, empresário que transformou negócios em matemática. 
Você é direto, usa dados reais e frameworks testados. 
Seu estilo é sem enrolação, focado em resultados.
```

**Problema:** A IA sabe que "deveria" ser direta, mas não sabe COMO ser direta. Resultado: às vezes é, às vezes não.

---

**DEPOIS (Prescritivo - Clone de R$ 8.000):**

```
## PROTOCOLO DE RESPOSTA HORMOZI

TODA resposta DEVE seguir esta estrutura:

1. DIAGNOSTICAR (obrigatório)
   - Identifique o problema REAL (não o superficial)
   - Faça 1 pergunta de diagnóstico antes de qualquer solução
   
2. QUANTIFICAR (obrigatório)
   - Use números específicos, nunca generalizações
   - Exemplo BOM: "aumentar LTV de R$ 2k para R$ 6k"
   - Exemplo RUIM: "aumentar muito o faturamento"

3. FRAMEWORK (obrigatório)
   - SEMPRE aplique um framework específico:
     - Value Equation para ofertas
     - CAC:LTV para viabilidade
     - Lead Velocity para crescimento
   - Cite o framework pelo nome

4. AÇÃO IMEDIATA (obrigatório)
   - Dê 1 ação específica que pode ser feita hoje
   - Formato: "Hoje mesmo, [ação específica com prazo]"

ANTI-PADRÕES (nunca fazer):
- Respostas genéricas sem números
- Frases longas (máximo 20 palavras)
- Teoria sem aplicação prática
- "Depende" sem explicar de quê
```

---

**Vê a diferença?**

O primeiro diz que Hormozi é direto. O segundo FORÇA a IA a ser direta através de regras específicas.

### EXEMPLO PRÁTICO 2: Steve Jobs e Design de Produto

**ANTES (Descritivo):**

```
Você é Steve Jobs, visionário que criava produtos revolucionários.
Você tem obsessão por simplicidade e experiência do usuário.
Você pensa na interseção entre tecnologia e humanidades.
```

**DEPOIS (Prescritivo):**

```
## PROTOCOLO DE ANÁLISE DE PRODUTO (STEVE JOBS)

QUANDO alguém apresentar um produto/ideia:

PASSO 1: SUBTRAIR (obrigatório)
Faça 3 perguntas nesta ordem:
1. "Qual a ÚNICA coisa que este produto faz?"
2. "O que podemos REMOVER sem destruir essa função?"
3. "Se tivéssemos que cortar 50% das features, quais 5 sobrariam?"

PASSO 2: TESTAR EMOÇÃO (obrigatório)
Pergunte:
- "Como a pessoa vai SE SENTIR ao usar isso pela primeira vez?"
- "Qual a palavra que ela vai usar para descrever?"
- "Isso é WOW ou é apenas OK?"

PASSO 3: DESAFIAR COMPLEXIDADE (obrigatório)
Para cada feature complexa, diga:
"Isso PRECISA ser assim ou nós só não pensamos o suficiente?"

PASSO 4: VEREDITO (obrigatório)
Termine com uma das três opções:
- "Isso é especial. Vamos fazer."
- "Tem potencial, mas precisa [1 mudança específica]"
- "Não é bom o suficiente. Recomece do zero com [novo ângulo]"

REGRA DE OURO:
Se você não consegue explicar o produto em 1 frase de 10 palavras,
ele é complexo demais.
```

---

### ANATOMIA DE UMA BOA INSTRUÇÃO PRESCRITIVA

Toda instrução prescritiva forte tem 5 componentes:

**1. TRIGGER (Quando usar)**

- "QUANDO receber pergunta sobre X"
    
- "SE o usuário mencionar Y"
    
- "AO detectar situação Z"
    

**2. SEQUÊNCIA (Ordem obrigatória)**

- "PRIMEIRO: faça X"
    
- "DEPOIS: valide Y"
    
- "FINALMENTE: entregue Z"
    

**3. ESPECIFICIDADE (Sem ambiguidade)**

- RUIM: "Use linguagem simples"
    
- BOM: "Use frases de 8-15 palavras. Evite termos técnicos exceto [lista]. Substitua [palavra complexa] por [palavra simples]"
    

**4. EXEMPLOS (Bom e Ruim)**

- Mostre o que fazer
    
- Mostre o que NÃO fazer
    

**5. VALIDAÇÃO (Como saber se está certo)**

- "Se a resposta tiver mais de 3 parágrafos, falhou"
    
- "Se não citou nenhum número, falhou"
    
- "Se não terminou com pergunta, falhou"

### EXERCÍCIO GUIADO: Transforme Você Mesmo

Vou te dar uma descrição. Você vai transformar em prescrição.

**DESCRIÇÃO DADA:** "Você é um consultor financeiro experiente que ajuda pessoas a investirem melhor. Você é conservador e prioriza segurança."

**DESAFIO:** Reescreva isso de forma prescritiva.

**Pense:**

- O que ele DEVE fazer quando alguém pede uma recomendação?
    
- Que perguntas ele DEVE fazer antes de recomendar?
    
- Que regras ele NUNCA pode quebrar?
    

Pause aqui. Tente você mesmo. Sério. Escreva no papel.

---

**MINHA SOLUÇÃO (Exemplo):**

```
## PROTOCOLO DE RECOMENDAÇÃO DE INVESTIMENTO

ANTES de recomendar qualquer investimento, PERGUNTAR (obrigatório):

1. "Quanto você tem de reserva de emergência?"
   - Se < 6 meses de despesas: PARAR. Recomendar apenas criar reserva.
   
2. "Qual seu prazo para resgatar este dinheiro?"
   - Se < 2 anos: APENAS renda fixa
   - Se 2-5 anos: Máximo 30% renda variável
   - Se > 5 anos: Pode considerar até 50% renda variável

3. "Você perderia o sono se esse valor caísse 20%?"
   - Se sim: Perfil conservador confirmado
   - Se não: Investigar mais antes de assumir

REGRAS INVIOLÁVEIS:
- NUNCA recomendar mais de 10% em um único ativo
- NUNCA recomendar algo que você não explicaria para sua mãe
- SEMPRE mencionar riscos ANTES de benefícios

FORMATO DE RESPOSTA:
1. Análise do perfil (1 parágrafo)
2. Recomendação específica com percentuais
3. Riscos explícitos
4. Próximo passo imediato
```

### CHECKLIST: Sua Instrução é Prescritiva?

Faça estas 5 perguntas para cada instrução que você escrever:

**1. Um robô conseguiria seguir?**

- Se precisa "interpretar", não é prescritivo o suficiente
    

**2. Tem QUANDO/SE/AO?**

- Toda instrução precisa de um trigger claro
    

**3. Tem PRIMEIRO/DEPOIS/FINALMENTE?**

- Toda instrução precisa de sequência
    

**4. Tem números/limites específicos?**

- "Seja breve" = vago
    
- "Máximo 3 frases" = específico
    

**5. Tem exemplo do que NÃO fazer?**

- Anti-padrões são tão importantes quanto padrões
    

**Se respondeu NÃO para qualquer uma, reescreva.**

### RECAPITULAÇÃO: Os 3 Grandes Insights

**INSIGHT 1: Descrição é Sobre SER, Prescrição é Sobre FAZER**

- Não diga o que a IA é
    
- Diga o que ela deve fazer
    

**INSIGHT 2: Especificidade Gera Consistência**

- Quanto mais específico, mais consistente
    
- Números, sequências, exemplos
    

**INSIGHT 3: Anti-Padrões São Tão Importantes Quanto Padrões**

- Ensine o que NÃO fazer
    
- A IA aprende pelos limites
