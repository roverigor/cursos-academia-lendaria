## METODOLOGIA

### FASE 1: DESTILAÇÃO PROFUNDA

1. Analise todos os inputs disponíveis da etapa de Analysis
2. Identifique elementos INVARIANTES (que nunca mudam)
3. Extraia componentes NUCLEARES (centrais à identidade)
4. Mapeie OPERADORES FUNDAMENTAIS (regras de funcionamento)
5. Sintetize ESSÊNCIA DISTINTIVA (o que torna único)

### FASE 2: CONSOLIDAÇÃO ESTRUTURAL

Organize elementos core em categorias operacionais prontas para implementação.

### FASE 3: VALIDAÇÃO DE COERÊNCIA

Verifique consistência interna e ausência de contradições não resolvidas.

## ALERTAS CRÍTICOS

### PRINCÍPIOS DE EXTRAÇÃO

**DESTILAÇÃO, NÃO CÓPIA:**
- Extraia ESSÊNCIA, não detalhes
- 20% dos elementos que geram 80% da identidade
- Foco em INVARIANTES e NUCLEARES
- Elimine redundâncias e ruído

**OPERACIONALIZAÇÃO:**
- Elementos devem ser IMPLEMENTÁVEIS
- Descrições devem ser EXECUTÁVEIS
- Prioridades devem ser CLARAS
- Output deve ser ACIONÁVEL

**COERÊNCIA INTERNA:**
- Valores não podem conflitar (exceto paradoxos documentados)
- Comportamentos devem alinhar com valores
- Arquitetura deve suportar comportamentos
- Tudo deve convergir para essence statement

### VALIDAÇÃO DE ELEMENTOS CORE

**Para cada elemento extraído, pergunte:**

1. É verdadeiramente NUCLEAR (central à identidade)?
2. É INVARIANTE (presente em todos os contextos)?
3. É DISTINTIVO (diferencia de outras personalidades)?
4. É OPERACIONAL (pode ser implementado)?
5. Tem EVIDÊNCIAS (múltiplas fontes confirmam)?

**Se resposta for NÃO para qualquer pergunta:** elemento não é core.

### TESTES DE CONSISTÊNCIA

**Teste 1: Coerência de Valores**
- Valor #1 nunca deve conflitar com essence statement
- Top 5 valores devem ser mutuamente suportivos
- Anti-valores devem ser opostos diretos de valores core

**Teste 2: Alinhamento Comportamental**
- Invariantes devem manifestar valores core
- Nunca-comportamentos devem violar anti-valores
- Padrões devem ser consistentes com arquitetura cognitiva

**Teste 3: Implementabilidade**
- Cada elemento core deve ter path claro para implementação
- Operadores devem ser traduzíveis em instruções
- Frameworks devem ser codificáveis em prompts

### REFINAMENTO ITERATIVO

Se scores de qualidade < 7/10:

1. **Completeness baixa:** Revisar análises para elementos faltantes
2. **Coherence baixa:** Resolver contradições ou documentar como paradoxos
3. **Distinctiveness baixa:** Buscar elementos mais únicos e específicos
4. **Readiness baixa:** Tornar elementos mais operacionais e executáveis

**LEMBRE-SE:** Qualidade > Quantidade. Melhor ter 5 elementos core perfeitos que 50 elementos medianos.

### ERROS COMUNS A EVITAR

1. Copiar análises completas em vez de destilar essência
2. Incluir elementos superficiais ou contextuais como "core"
3. Ter valores conflitantes não resolvidos
4. Descrições vagas e não operacionais
5. Elementos genéricos que se aplicam a qualquer pessoa
6. Paradoxos não documentados gerando inconsistências
7. Limitações não explicitadas criando expectativas irreais
