# # **CATEGORIA 1: HIPERCONTEXTUALIZAÇÃO**

# ## **T001 - Ancoragem por Camadas Progressivas**

```markdown
# # TÉCNICA: Estratificação Contextual
# ## Aplicação: Estabelecer contexto robusto em prompts

<context_foundation>
CAMADA 1 - ESSÊNCIA: [Identidade fundamental do sistema]
CAMADA 2 - DOMÍNIO: [Área de atuação e fronteiras]
CAMADA 3 - OPERAÇÃO: [Como o sistema funciona]
</context_foundation>

# ## Exemplo de Uso:
Substitua os colchetes pelos valores específicos do clone.
```

# ## **T002 - Delimitação Semântica por Tags**

```xml
# # TÉCNICA: Fronteiras Cognitivas
# ## Aplicação: Separar espaços conceituais claramente

<operational_domain>
[Definir claramente O QUE está no escopo]
</operational_domain>

<non_operational_domain>
[Definir claramente O QUE NÃO está no escopo]
</non_operational_domain>

# ## Quando Usar:
- Prompts que precisam de fronteiras claras
- Sistemas que podem "vazar" para áreas indesejadas
- Clones com expertise muito específica
```

```markdown

# ## Aplicação: Estabelecer padrão de qualidade esperado
```

# # **CATEGORIA 2: ESTRUTURAÇÃO MODULAR**

# ## **T004 - Pseudo-código Cognitivo**

```javascript
# # TÉCNICA: Programação Mental
# ## Aplicação: Definir fluxos de processamento precisos

function processar_[TAREFA](input) {
    // FASE 1: ANÁLISE
    let contexto = analisar_contexto(input);
    let objetivos = identificar_objetivos(contexto);

    // FASE 2: PROCESSAMENTO
    let estrategia = selecionar_abordagem(objetivos);
    let resultado = aplicar_estrategia(estrategia, input);

    // FASE 3: VALIDAÇÃO
    if (!validar_qualidade(resultado)) {
        resultado = refinar(resultado);
    }

    return estruturar_output(resultado);
}

# ## Adaptação:
Substitua [TAREFA] pela função específica do prompt.
```

# ## **T005 - Modularização Hierárquica**

```markdown
# # TÉCNICA: Organização em Camadas
# ## Aplicação: Estruturar prompts complexos

# NÍVEL 1: SISTEMA GLOBAL
# # 1. MÓDULO PRINCIPAL: [Nome do Módulo]
   ### 1.1 Submódulo: [Nome do Submódulo]
      #### 1.1.1 Componente: [Nome do Componente]
         - Detalhe específico
         - Outro detalhe específico

# # 2. MÓDULO SECUNDÁRIO: [Nome do Módulo]
   ### 2.1 Submódulo: [Nome do Submódulo]
      #### 2.1.1 Componente: [Nome do Componente]

# ## Regra:
Cada nível deve ser auto-contido mas conectado aos outros.
```

# ## **T006 - Mapeamento de Dependências**

```markdown
# # TÉCNICA: Fluxo de Dependências
# ## Aplicação: Sequenciar processos complexos

SEQUÊNCIA DE PROCESSAMENTO:
1. ETAPA A: [Descrição]
   → OUTPUT: [O que produz]

2. ETAPA B: [Descrição]
   → DEPENDE DE: Output da Etapa A
   → OUTPUT: [O que produz]

3. ETAPA C: [Descrição]
   → DEPENDE DE: Outputs das Etapas A e B
   → OUTPUT: [Resultado final]

# ## Aplicação:
Use quando o processo tem dependências claras entre etapas.
```

```markdown
□ [Pergunta específica sobre qualidade 2]
□ [Pergunta específica sobre qualidade 3]
```

# ## **T015 - Refinamento Iterativo**

```markdown
# # TÉCNICA: Melhoria Contínua
# ## Aplicação: Otimização de outputs

CICLO_DE_REFINAMENTO:
1. GERAR versão inicial
2. AVALIAR contra critérios de qualidade
3. IDENTIFICAR pontos específicos de melhoria
4. APLICAR correções direcionadas
5. RE-AVALIAR resultado
6. REPETIR até aprovação ou limite de iterações

CRITÉRIOS_DE_PARADA:
- Qualidade satisfatória atingida
- Número máximo de iterações alcançado
- Melhoria marginal insuficiente
```

# # **GUIA DE USO DAS TÉCNICAS**

# ## **Para Escolher a Técnica Certa:**

1. **Identifique o objetivo**: O que você quer que o prompt faça?
2. **Mapeie a categoria**: Qual aspecto precisa ser melhorado?
3. **Selecione a técnica**: Qual técnica melhor atende ao objetivo?
4. **Adapte ao contexto**: Personalize para o clone específico
5. **Teste e refine**: Valide a efetividade e ajuste

# ## **Combinação de Técnicas:**

- **T001 + T007**: Para estabelecer contexto robusto
- **T004 + T009**: Para processos complexos com validação
- **T008 + T010**: Para análises profundas e criativas
- **T012 + T013**: Para máximo controle de atenção

# ## **Adaptação por Domínio:**

- **Técnicos**: Enfatizar T004, T006, T014
- **Criativos**: Enfatizar T008, T010, T013
- **Estratégicos**: Enfatizar T005, T011, T015
