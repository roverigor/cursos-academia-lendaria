### BLOCO 1: O QUE É VERCEL (2 minutos)

```
[TELA: Animação simples - código → nuvem → site funcionando]
```

**NARRAÇÃO:**

> Vercel é uma plataforma de deploy. Traduzindo: é onde você coloca seu projeto pra rodar na internet.
>
> Mas não é qualquer plataforma. É uma plataforma que foi CRIADA especificamente pra frameworks modernos como Next.js, Nuxt, SvelteKit...
>
> E aqui vem o detalhe importante: **a Vercel é a mesma empresa que criou o Next.js**. Então quando você usa Next.js com Vercel, é tipo usar iPhone com iCloud. Funciona junto de forma mágica.

```
[TELA: Logo Vercel + Logo Next.js lado a lado]
```

> Se eu tivesse que explicar Vercel em uma frase seria:
>
> **"Você dá um git push e seu site tá no ar."**
>
> Sério. É isso. Sem configurar servidor, sem se preocupar com infraestrutura. Push e pronto.

---

### BLOCO 2: ANALOGIA - A GARAGEM INTELIGENTE (2 minutos)

```
[TELA: Animação de garagem com robô estacionando carro]
```

**NARRAÇÃO:**

> Deixa eu te dar uma analogia pra você nunca mais esquecer.
>
> Imagina que seu projeto é um carro. Você construiu ele, tá funcionando lindo na sua garagem - que é o seu computador, o localhost.
>
> Agora você quer que outras pessoas vejam seu carro. Você precisa de um lugar pra estacionar ele em público.
>
> **Hospedagem tradicional** é tipo um estacionamento comum. Você chega, procura vaga, estaciona você mesmo, cuida de tudo.
>
> **Vercel** é tipo uma garagem inteligente com robô. Você chega na entrada, entrega a chave, e o robô estaciona pra você. Automaticamente. No melhor lugar. Com segurança.

```
[TELA: Comparação visual - Estacionamento manual vs Garagem robô]
```

> Você não precisa saber onde o carro tá estacionado. Só precisa saber que quando alguém quiser ver, vai estar lá, funcionando.
>
> Isso é Vercel.

### BLOCO 3: O QUE VERCEL FAZ POR VOCÊ (2 minutos)

```
[TELA: Lista animada aparecendo item por item]
```

**NARRAÇÃO:**

> Então o que a Vercel faz automaticamente quando você dá um push?
>
> **Primeiro:** Detecta qual framework você tá usando. Next.js? React? Astro? Ela descobre sozinha.
>
> **Segundo:** Faz o build do seu projeto. Compila, otimiza, prepara tudo.
>
> **Terceiro:** Coloca numa CDN global. Isso significa que seu site vai estar em servidores no mundo todo. Se alguém no Japão acessar, vai ser rápido. Se alguém em São Paulo acessar, também.
>
> **Quarto:** Configura HTTPS automaticamente. Aquele cadeadinho verde de segurança? Vem de graça.
>
> **Quinto:** Te dá uma URL funcionando. Tipo seuprojeto.vercel.app.

```
[TELA: Diagrama - Git Push → Build → CDN → URL]
```

> Tudo isso em segundos. Sem você configurar nada.

### BLOCO 4: O QUE VERCEL NÃO É (1 minuto)

```
[TELA: Ícones com X vermelho]
```

**NARRAÇÃO:**

> Agora, pra ficar claro, o que Vercel NÃO é:
>
> **Não é um servidor tradicional.** Você não vai acessar via SSH, não vai instalar coisas manualmente.
>
> **Não é pra qualquer tipo de aplicação.** É focada em frontend e aplicações JAMstack. Se você precisa de um banco de dados PostgreSQL rodando, backend pesado em Python... Vercel não é o lugar. Aí você vai precisar de outra coisa.
>
> **Não é só pra Next.js.** Funciona com vários frameworks, mas Next.js é onde ela brilha mais.

---

### BLOCO 5: POR QUE VERCEL PRA NEXT.JS (1 minuto)

```
[TELA: Logos Vercel + Next.js com coração entre eles]
```

**NARRAÇÃO:**

> Se você tá usando Next.js, Vercel é a escolha óbvia. Por quê?
>
> **A Vercel CRIOU o Next.js.** Guillermo Rauch, o CEO da Vercel, é o criador do Next.js. A equipe que desenvolve o framework é a mesma que desenvolve a plataforma.
>
> Isso significa que quando sai uma feature nova no Next.js, tipo Server Components, ISR, Edge Functions... a Vercel já suporta no mesmo dia. Às vezes antes.
>
> É como usar o carro da montadora no test drive. Funciona perfeito porque foi feito pra funcionar junto.
