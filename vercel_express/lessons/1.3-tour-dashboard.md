# Aula 1.3: Tour pelo Dashboard

**Módulo:** 1 - Entendendo Vercel
**Duração:** 5 minutos
**Tipo:** Screencast

---

## Objetivo da Aula

Ao final desta aula, o aluno vai:
- Navegar pelo dashboard com confiança
- Identificar as seções principais
- Saber onde encontrar o que precisa

---

## Roteiro

### ABERTURA (15 segundos)

```
[SCREENCAST: Dashboard da Vercel]
```

**NARRAÇÃO:**

> Antes de deployar qualquer coisa, deixa eu te mostrar onde fica cada coisa no dashboard. Assim você não vai ficar perdido depois.

---

### BLOCO 1: VISÃO GERAL (1 minuto)

```
[SCREENCAST: Dashboard com destaques nas áreas]
```

**NARRAÇÃO:**

> O dashboard da Vercel é bem limpo. Vamos por partes.
>
> **No topo**, você tem a barra de navegação. Aqui fica:

```
[DESTAQUE: Barra superior]
```

> - Seu avatar e nome da conta
> - O seletor de team (se você tiver mais de um)
> - Notificações
> - Busca
>
> **No centro**, é onde aparecem seus projetos. Agora tá vazio, mas quando você deployar algo, vai aparecer aqui em formato de cards.

```
[DESTAQUE: Área central]
```

> **Na lateral esquerda**, dependendo da versão, você pode ter um menu com atalhos.

---

### BLOCO 2: ABA OVERVIEW (1 minuto)

```
[SCREENCAST: Clicando em um projeto (ou mostrando exemplo)]
```

**NARRAÇÃO:**

> Quando você clicar num projeto, vai entrar na página dele. A primeira aba é **Overview**.
>
> Aqui você vê:

```
[DESTAQUE: Cada elemento]
```

> - **Production Deployment** - A versão que tá no ar agora
> - **Preview Deployments** - Versões de teste de outras branches
> - **Domains** - Os domínios conectados
> - **Recent Activity** - Últimos deploys e mudanças
>
> É tipo o resumo executivo do seu projeto.

---

### BLOCO 3: ABA DEPLOYMENTS (1 minuto)

```
[SCREENCAST: Clicando em Deployments]
```

**NARRAÇÃO:**

> A aba **Deployments** é onde você vê o histórico de todos os deploys.
>
> Cada deploy mostra:
> - Se foi sucesso ou falhou
> - Quanto tempo levou
> - Qual branch gerou
> - Quem fez o commit
>
> Se alguma coisa der errado, é aqui que você vem investigar. Você pode clicar num deploy pra ver os logs de build.

```
[AÇÃO: Clicar num deploy pra mostrar os logs]
```

> Esses logs são seu melhor amigo quando dá erro. Guarda essa informação.

---

### BLOCO 4: ABA SETTINGS (1 minuto)

```
[SCREENCAST: Clicando em Settings]
```

**NARRAÇÃO:**

> A aba **Settings** é onde você configura tudo do projeto.
>
> As mais importantes pra você agora são:

```
[DESTAQUE: Cada seção]
```

> - **General** - Nome do projeto, framework, região
> - **Domains** - Adicionar domínio customizado
> - **Environment Variables** - As famosas env vars (vamos ver em detalhe depois)
> - **Git** - Configurações de integração com GitHub
>
> A maioria das coisas você não precisa mexer. A Vercel configura automaticamente. Mas é bom saber onde fica.

---

### BLOCO 5: MENU DO USUÁRIO (30 segundos)

```
[SCREENCAST: Clicando no avatar]
```

**NARRAÇÃO:**

> Por último, clicando no seu avatar lá no canto, você acessa:
>
> - **Account Settings** - Configurações da sua conta
> - **Usage** - Quanto você tá usando do seu plano (importante!)
> - **Billing** - Pagamentos e plano
> - **Logout** - Sair da conta
>
> A seção **Usage** é onde você monitora se tá chegando perto dos limites do plano gratuito. Vamos falar mais sobre isso no módulo 5.

---

### FECHAMENTO (15 segundos)

```
[TELA: Mapa visual do dashboard]
```

**NARRAÇÃO:**

> Pronto! Agora você conhece o território.
>
> - **Overview** - Resumo do projeto
> - **Deployments** - Histórico e logs
> - **Settings** - Configurações
> - **Usage** - Consumo do plano
>
> No próximo módulo, a gente vai finalmente fazer o primeiro deploy. A parte divertida! Até já!

## Pontos-Chave para Memorização

- Overview = resumo do projeto
- Deployments = histórico e logs (ir aqui quando der erro)
- Settings = configurações (domínio, env vars)
- Usage = monitorar limites do plano
