### BLOCO 1: O QUE É DNS (2 minutos)

```
[TELA: Analogia visual - lista telefônica]
```

**NARRAÇÃO:**

> DNS é a sigla pra Domain Name System. Traduzindo: **Sistema de Nomes de Domínio**.
>
> Vou te dar a melhor analogia possível:
>
> **DNS é a lista telefônica da internet.**
>
> Pensa assim: quando você quer ligar pra alguém, você não decora o número de telefone. Você procura o nome da pessoa na agenda.
>
> - Nome: "Maria"
> - Número: 11-99999-8888
>
> Na internet é igual:
>
> - Nome: "google.com"
> - Número: 142.250.218.14 (endereço IP)

```
[TELA: Diagrama nome → número]
```

> Quando você digita `google.com` no navegador, o DNS traduz isso pro endereço IP real onde o servidor do Google tá.
>
> Você não precisa decorar `142.250.218.14`. Só `google.com`.
>
> **DNS é a tradução entre nomes fáceis de lembrar e endereços técnicos de servidor.**

---

### BLOCO 2: REGISTROS DNS (2.5 minutos)

```
[TELA: Tabela de registros]
```

**NARRAÇÃO:**

> Na "lista telefônica" do seu domínio, existem diferentes tipos de entradas. Os dois mais importantes são:
>
> **Registro A (Address)**

```
[DESTAQUE: Registro A]
```

> Liga um nome diretamente a um endereço IP.
>
> Exemplo:
> - `meusite.com.br` → `76.76.21.21`
>
> É o mais básico. Nome aponta pra número.
>
> **Registro CNAME (Canonical Name)**

```
[DESTAQUE: Registro CNAME]
```

> Liga um nome a OUTRO nome.
>
> Exemplo:
> - `www.meusite.com.br` → `meusite.com.br`
>
> Ou:
> - `meusite.com.br` → `cname.vercel-dns.com`
>
> É tipo um redirecionamento. "Esse nome aqui? Vai procurar nesse outro nome."

```
[TELA: Diagrama A vs CNAME]
```

> **Quando usar cada um:**
>
> - **A:** Quando você tem um IP fixo
> - **CNAME:** Quando você quer apontar pra outro domínio (como a Vercel)
>
> A Vercel geralmente pede pra você configurar um CNAME apontando pra `cname.vercel-dns.com`.

### BLOCO 3: PROPAGAÇÃO (2 minutos)

```
[TELA: Mapa mundial com servidores DNS]
```

**NARRAÇÃO:**

> Quando você muda algo no DNS, não é instantâneo. Leva um tempo pra "espalhar" pelo mundo.
>
> Isso se chama **propagação**.
>
> Por que demora?
>
> A internet tem milhares de servidores DNS pelo mundo. Quando você muda um registro, essa mudança precisa se espalhar de servidor em servidor.

```
[TELA: Animação de propagação - onda se espalhando]
```

> É tipo uma fofoca. Começa num lugar e vai se espalhando até todo mundo saber.
>
> **Quanto tempo demora?**
>
> - Geralmente: 5 minutos a 2 horas
> - Às vezes: até 48 horas (raro, mas acontece)
> - Fatores: TTL (Time To Live) do registro, cache dos provedores
>
> **Dica:** Se você mudou o DNS e não tá funcionando, espera um pouco antes de entrar em pânico.

```
[TELA: whatsmydns.net]
```

> **Como verificar propagação:**
>
> Acessa **whatsmydns.net**, digita seu domínio, e ele mostra se a mudança já chegou em servidores pelo mundo.
>
> Verde = propagou. Vermelho = ainda não.

### BLOCO 4: RESUMO VISUAL (1 minuto)

```
[TELA: Infográfico completo]
```

**NARRAÇÃO:**

> Resumindo tudo:
>
> **DNS** = Lista telefônica da internet
> - Traduz nomes (domínios) em números (IPs)
>
> **Registro A** = Nome → IP direto
> **Registro CNAME** = Nome → Outro nome
>
> **Propagação** = Tempo pra mudança se espalhar (minutos a horas)
