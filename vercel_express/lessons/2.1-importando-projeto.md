# Aula 2.1: Importando Projeto do GitHub

**Módulo:** 2 - Primeiro Deploy
**Duração:** 8 minutos
**Tipo:** Screencast + explicação

---

## Objetivo da Aula

Ao final desta aula, o aluno vai:
- Importar repositório do GitHub na Vercel
- Entender o framework detection automático
- Saber quando e como ajustar configurações de build

---

## Roteiro

### ABERTURA (20 segundos)

```
[SCREENCAST: Dashboard da Vercel]
```

**NARRAÇÃO:**

> Chegou a hora. Vamos colocar seu primeiro projeto no ar.
>
> Eu vou usar um projeto Next.js simples, mas o processo é o mesmo pra React, Vue, ou qualquer outro framework que a Vercel suporta.

---

### BLOCO 1: INICIANDO A IMPORTAÇÃO (1.5 minutos)

```
[SCREENCAST: Dashboard → Add New → Project]
```

**NARRAÇÃO:**

> No seu dashboard, clica em **Add New** e depois em **Project**.

```
[AÇÃO: Clicar em Add New → Project]
```

> Você vai ver uma lista dos seus repositórios do GitHub. São os repos que você deu acesso quando conectou a conta.

```
[TELA: Lista de repositórios]
```

> Se o repo que você quer não tá aparecendo, clica em **Adjust GitHub App Permissions** lá embaixo pra adicionar mais repos.
>
> Eu vou selecionar esse aqui... [nome do projeto].

```
[AÇÃO: Clicar em Import no repositório escolhido]
```

---

### BLOCO 2: FRAMEWORK DETECTION (2 minutos)

```
[SCREENCAST: Tela de configuração do projeto]
```

**NARRAÇÃO:**

> Olha que mágica. A Vercel já detectou que meu projeto é **Next.js**.

```
[DESTAQUE: Framework detected]
```

> Isso é o **Framework Detection**. A Vercel olha seu código, identifica qual framework você tá usando, e configura tudo automaticamente.
>
> Ela suporta mais de 35 frameworks:
> - Next.js
> - React (Create React App)
> - Vue.js / Nuxt
> - Svelte / SvelteKit
> - Angular
> - Astro
> - E vários outros...

```
[TELA: Lista de frameworks suportados]
```

> Se ela detectar errado, ou se você estiver usando algo que ela não reconhece, você pode mudar aqui no dropdown "Framework Preset".

```
[DESTAQUE: Dropdown de framework]
```

> Mas na maioria das vezes, ela acerta de primeira. Deixa no automático.

---

### BLOCO 3: CONFIGURAÇÕES DE BUILD (2.5 minutos)

```
[SCREENCAST: Expandir "Build and Output Settings"]
```

**NARRAÇÃO:**

> Vamos abrir as configurações de build pra entender o que tá acontecendo.

```
[AÇÃO: Clicar em "Build and Output Settings"]
```

> Aqui você tem:
>
> **Build Command** - O comando que a Vercel vai rodar pra compilar seu projeto.

```
[DESTAQUE: Build Command]
```

> Por padrão, pra Next.js, é `next build` ou `npm run build`. Geralmente você não precisa mudar.
>
> **Output Directory** - A pasta onde o projeto compilado fica.

```
[DESTAQUE: Output Directory]
```

> Pra Next.js é `.next`, pra Create React App é `build`, pra Vue é `dist`. A Vercel já sabe isso.
>
> **Install Command** - Como instalar as dependências.

```
[DESTAQUE: Install Command]
```

> Por padrão é `npm install` ou `yarn install`. Também não precisa mudar.

```
[TELA: Resumo visual das 3 configurações]
```

> **Regra de ouro:** Se tá funcionando no seu computador com `npm run build`, vai funcionar na Vercel sem mudar nada.

---

### BLOCO 4: ROOT DIRECTORY (1 minuto)

```
[SCREENCAST: Mostrar Root Directory]
```

**NARRAÇÃO:**

> Uma configuração importante é o **Root Directory**.

```
[DESTAQUE: Root Directory]
```

> Se seu projeto Next.js não tá na raiz do repositório - tipo, tá dentro de uma pasta `frontend/` ou `apps/web/` - você precisa especificar aqui.
>
> Exemplo: se sua estrutura é assim:

```
[TELA: Estrutura de pastas]
meu-repo/
├── backend/
└── frontend/    ← seu Next.js tá aqui
    ├── package.json
    └── ...
```

> Você coloca `frontend` no Root Directory.
>
> Se seu Next.js tá na raiz, deixa em branco.

---

### BLOCO 5: DEPLOY! (1 minuto)

```
[SCREENCAST: Botão Deploy]
```

**NARRAÇÃO:**

> Tudo configurado? Então é só clicar em **Deploy**.

```
[AÇÃO: Clicar em Deploy]
```

> Isso. Agora a Vercel tá:
> 1. Clonando seu repositório
> 2. Instalando dependências
> 3. Rodando o build
> 4. Publicando na CDN global
>
> Vamos acompanhar isso na próxima aula. Mas olha... já tá quase pronto!

```
[TELA: Progresso do build]
```

---

### FECHAMENTO (15 segundos)

```
[TELA: Resumo visual]
```

**NARRAÇÃO:**

> Recapitulando:
>
> - Seleciona o repositório
> - Framework é detectado automaticamente
> - Configurações de build geralmente não precisam mudar
> - Root Directory só se o projeto não tá na raiz
>
> Na próxima aula, a gente vai entender o que tá acontecendo durante o build. Até já!

## Pontos-Chave para Memorização

- Framework Detection = automático na maioria dos casos
- Build Command = `npm run build` (não mexer)
- Root Directory = só se projeto não tá na raiz
- Regra: funciona local → funciona na Vercel
