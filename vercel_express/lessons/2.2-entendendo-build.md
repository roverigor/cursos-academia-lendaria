# Aula 2.2: Entendendo o Build

**Módulo:** 2 - Primeiro Deploy
**Duração:** 10 minutos
**Tipo:** Conceitual + screencast

---

## Objetivo da Aula

Ao final desta aula, o aluno vai:
- Entender o que é o processo de build
- Saber ler logs de build
- Identificar as etapas do processo

---

## Roteiro

### ABERTURA (20 segundos)

```
[SCREENCAST: Build em progresso]
```

**NARRAÇÃO:**

> Seu projeto tá buildando. Mas o que diabos isso significa?
>
> Vamos entender o que tá acontecendo por baixo dos panos. Isso vai te ajudar MUITO quando der erro.

---

### BLOCO 1: O QUE É BUILD (2 minutos)

```
[TELA: Animação - código fonte → build → código otimizado]
```

**NARRAÇÃO:**

> Pensa assim: você escreve código de um jeito que faz sentido pra você, desenvolvedor. Com componentes, imports, TypeScript, CSS modular...
>
> Mas o navegador não entende tudo isso diretamente. Ele precisa de HTML, CSS e JavaScript simples.
>
> **Build é o processo de transformar seu código de desenvolvimento em código de produção.**

```
[TELA: Analogia visual]
```

> Analogia: é como uma cozinha de restaurante.
>
> - Você (desenvolvedor) escreve a **receita** (código fonte)
> - O **build** é o chef preparando o prato
> - O resultado é o **prato pronto** (código de produção)
>
> O cliente (usuário) não quer ver a receita, quer o prato pronto na mesa.

---

### BLOCO 2: O QUE ACONTECE NO BUILD (2.5 minutos)

```
[SCREENCAST: Logs de build rolando]
```

**NARRAÇÃO:**

> Vamos olhar os logs. A Vercel mostra tudo que tá fazendo.

```
[DESTAQUE: Etapa 1]
```

> **Etapa 1: Cloning**
>
> Primeiro, ela clona seu repositório do GitHub. Baixa todo o código.
>
> Você vai ver algo tipo: "Cloning github.com/seuuser/seuprojeto..."

```
[DESTAQUE: Etapa 2]
```

> **Etapa 2: Installing Dependencies**
>
> Depois, ela roda `npm install` (ou yarn/pnpm). Baixa todas as bibliotecas que seu projeto precisa.
>
> Aqui você vê as dependências sendo instaladas: "added 500 packages in 30s"

```
[DESTAQUE: Etapa 3]
```

> **Etapa 3: Building**
>
> Agora é o build de verdade. Ela roda seu comando de build - geralmente `npm run build`.
>
> Pra Next.js, você vai ver:
> - Compilando páginas
> - Otimizando imagens
> - Gerando rotas estáticas
> - Minificando JavaScript

```
[DESTAQUE: Etapa 4]
```

> **Etapa 4: Deploying**
>
> Por fim, ela pega o resultado e distribui nos servidores pelo mundo.
>
> "Deployment completed" = sucesso!

---

### BLOCO 3: COMO LER OS LOGS (2.5 minutos)

```
[SCREENCAST: Logs detalhados]
```

**NARRAÇÃO:**

> Os logs parecem intimidadores, mas você não precisa entender tudo. Só precisa saber procurar.
>
> **Dica #1: Role até o final**

```
[AÇÃO: Rolar até o fim dos logs]
```

> O erro mais importante geralmente tá nas últimas linhas. Se deu erro, vai ter uma mensagem em vermelho bem clara no final.
>
> **Dica #2: Procure por "error"**

```
[AÇÃO: Ctrl+F → "error"]
```

> Usa Ctrl+F e busca por "error". Vai direto pro problema.
>
> **Dica #3: Leia a linha completa do erro**

```
[DESTAQUE: Exemplo de erro]
```

> Não olha só a palavra "error". Lê a frase completa. Tipo:
>
> `Error: Cannot find module './components/Header'`
>
> Isso te diz EXATAMENTE o problema: tem um import pra um arquivo que não existe.
>
> **Dica #4: Compare com local**

```
[TELA: Terminal local vs Vercel]
```

> Se deu erro, tenta rodar `npm run build` no seu computador. Se der o mesmo erro, o problema tá no código, não na Vercel.

---

### BLOCO 4: TEMPOS DE BUILD (1.5 minutos)

```
[SCREENCAST: Tempo de build no dashboard]
```

**NARRAÇÃO:**

> Você vai notar que cada build mostra quanto tempo levou.
>
> **Primeiro build:** Pode demorar mais (1-3 minutos) porque precisa baixar tudo do zero.
>
> **Builds seguintes:** Geralmente mais rápidos porque a Vercel cacheia as dependências.
>
> Se seu build tá demorando muito (mais de 5 minutos pra um projeto simples), provavelmente tem algo errado:
> - Dependências desnecessárias
> - Build não otimizado
> - Imagens muito grandes

```
[TELA: Barra de tempo com referências]
```

> Pra um portfolio simples em Next.js, espere entre 30 segundos e 2 minutos. Mais que isso, investiga.

---

```
[SCREENCAST: Build completo com sucesso]
```

**NARRAÇÃO:**

> Olha aí! "Build Succeeded"!

```
[DESTAQUE: Mensagem de sucesso]

> Quando você vê isso, significa:

> - Seu site tá no ar!
>
> A Vercel já mostra a URL onde você pode acessar. Geralmente é algo tipo:

> `seuprojeto-git-main-seuuser.vercel.app`

[AÇÃO: Clicar na URL]
```

> Na próxima aula, a gente vai acessar e celebrar seu primeiro deploy!

```

```

**NARRAÇÃO:**

> Recapitulando as etapas do build:
>
