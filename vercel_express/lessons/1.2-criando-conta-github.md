# Aula 1.2: Criando Conta e Conectando GitHub

**Módulo:** 1 - Entendendo Vercel
**Duração:** 7 minutos
**Tipo:** Screencast passo a passo

---

## Objetivo da Aula

Ao final desta aula, o aluno vai:
- Ter conta criada no plano Hobby (gratuito)
- GitHub conectado à Vercel
- Entender as permissões solicitadas

---

## Roteiro

### ABERTURA (20 segundos)

```
[TELA: Navegador aberto em vercel.com]
```

**NARRAÇÃO:**

> Agora que você entende o que é Vercel, vamos criar sua conta.
>
> Vai levar menos de 2 minutos. Acompanha comigo na tela.

---

### BLOCO 1: ACESSANDO O SITE (1 minuto)

```
[SCREENCAST: Navegador em vercel.com]
```

**NARRAÇÃO:**

> Abre o navegador e vai em **vercel.com**.
>
> Você vai ver a página inicial com um botão grande escrito "Start Deploying" ou "Get Started". Clica nele.

```
[AÇÃO: Clicar no botão]
```

> Agora você tem três opções pra criar conta:
> - Continuar com GitHub
> - Continuar com GitLab
> - Continuar com Bitbucket
>
> A gente vai usar **GitHub** porque é o mais comum e provavelmente você já tem conta lá.

```
[AÇÃO: Clicar em "Continue with GitHub"]
```

---

### BLOCO 2: AUTORIZANDO O GITHUB (2 minutos)

```
[SCREENCAST: Tela de autorização do GitHub]
```

**NARRAÇÃO:**

> Vai abrir uma tela do GitHub pedindo autorização. Aqui é importante você entender o que tá autorizando.
>
> A Vercel vai pedir acesso pra:

```
[TELA: Destacar cada permissão]
```

> **Ler seus repositórios** - Pra ela saber quais projetos você tem
>
> **Webhooks** - Pra ela saber quando você faz push e deployar automaticamente
>
> **Email** - Pra criar sua conta na Vercel
>
> Isso é seguro. A Vercel não vai mexer no seu código, não vai deletar nada. Só vai ler pra poder fazer deploy.

```
[AÇÃO: Clicar em "Authorize Vercel"]
```

> Clica em **Authorize Vercel**.

---

### BLOCO 3: ESCOLHENDO O PLANO (1.5 minutos)

```
[SCREENCAST: Tela de seleção de plano]
```

**NARRAÇÃO:**

> Agora vem a tela de escolher o plano. Você vai ver:
>
> - **Hobby** - Gratuito, pra projetos pessoais
> - **Pro** - Pago, pra times
> - **Enterprise** - Empresas grandes
>
> **Escolhe Hobby.** É gratuito e tem tudo que você precisa pra começar.

```
[AÇÃO: Selecionar Hobby]
```

> O Hobby inclui:
> - 100GB de bandwidth por mês
> - HTTPS automático
> - Deploys ilimitados
> - Preview de branches
>
> Pra um portfolio ou projeto pessoal, é mais que suficiente.

```
[AÇÃO: Clicar em "Continue" ou equivalente]
```

---

### BLOCO 4: INSTALANDO O APP NO GITHUB (2 minutos)

```
[SCREENCAST: Tela de instalação do Vercel app no GitHub]
```

**NARRAÇÃO:**

> Agora a Vercel vai pedir pra instalar um "app" no seu GitHub. Isso é pra ela poder acessar seus repositórios.
>
> Você tem duas opções:

```
[TELA: Destacar opções]
```

> **All repositories** - Ela vai ter acesso a todos os seus repos. Mais fácil, mas talvez você não queira isso.
>
> **Only select repositories** - Você escolhe quais repos ela pode ver. Mais seguro, você tem controle.
>
> **Minha recomendação:** Começa com "Only select repositories" e seleciona só o projeto que você quer deployar agora. Depois você pode adicionar mais.

```
[AÇÃO: Selecionar "Only select repositories"]
```

> Seleciona o repositório do seu projeto...

```
[AÇÃO: Selecionar um repo da lista]
```

> E clica em **Install**.

```
[AÇÃO: Clicar em Install]
```

---

### BLOCO 5: CONTA CRIADA! (30 segundos)

```
[SCREENCAST: Dashboard da Vercel]
```

**NARRAÇÃO:**

> Pronto! Você tá dentro.
>
> Essa é a tela do seu dashboard. Tá vazio porque você ainda não tem projetos deployados, mas isso vai mudar em breve.
>
> Sua conta tá criada, GitHub conectado, e você tá pronto pra fazer seu primeiro deploy.

---

### FECHAMENTO (20 segundos)

```
[TELA: Checklist visual]
```

**NARRAÇÃO:**

> Recapitulando o que você fez:
>
> ✅ Criou conta na Vercel
> ✅ Conectou com GitHub
> ✅ Escolheu plano Hobby (gratuito)
> ✅ Instalou o app no repositório
>
> Na próxima aula, vou te mostrar o dashboard pra você saber onde encontrar tudo. Até já!

## Pontos-Chave para Memorização

- Usar "Continue with GitHub"
- Escolher plano Hobby (gratuito)
- "Only select repositories" é mais seguro
- Permissões são apenas leitura
