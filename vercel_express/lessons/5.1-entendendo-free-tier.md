# Aula 5.1: Entendendo o Free Tier

**Módulo:** 5 - Evitando Problemas
**Duração:** 8 minutos
**Tipo:** Conceitual com exemplos

---

## Objetivo da Aula

Ao final desta aula, o aluno vai:
- Conhecer os limites do plano Hobby
- Saber o que está incluído
- Avaliar se seu projeto cabe no free tier

### BLOCO 1: O QUE É O PLANO HOBBY (1.5 minutos)

```
[TELA: Comparativo de planos]
```

**NARRAÇÃO:**

> A Vercel tem três planos principais:
>
> - **Hobby** - Gratuito, pra projetos pessoais
> - **Pro** - $20/mês por membro, pra times
> - **Enterprise** - Preço customizado, pra empresas grandes
>
> O **Hobby** é o que você tá usando. E é bem generoso.

```
[DESTAQUE: Plano Hobby]
```

> **O que tá incluído no Hobby:**
>
> ✅ Deploys ilimitados
> ✅ HTTPS automático
> ✅ CDN global
> ✅ Preview deployments
> ✅ Integrações com Git
> ✅ 100GB de bandwidth/mês
> ✅ Serverless functions
>
> Pra um portfolio ou projeto pessoal, isso é MUITO mais do que você precisa.

---

### BLOCO 2: OS LIMITES REAIS (2.5 minutos)

```
[TELA: Tabela de limites]
```

**NARRAÇÃO:**

> Agora os limites. Isso é importante.
>
> **Bandwidth: 100GB/mês**

```
[DESTAQUE: Bandwidth]
```

> Bandwidth é a quantidade de dados transferidos. Cada vez que alguém acessa seu site, gasta um pouquinho.
>
> 100GB parece pouco? Faz a conta:
> - Página média: 2MB
> - 100GB = 100.000MB
> - 100.000 / 2 = **50.000 pageviews/mês**
>
> Cinquenta mil acessos por mês. Seu portfolio vai ter isso? Provavelmente não.
>
> **Serverless Function Execution: 100GB-hours/mês**

```
[DESTAQUE: Function Execution]
```

> Se você usa API routes (tipo `/api/contato`), esse é o limite de execução.
>
> 100GB-hours é suficiente pra MILHARES de requests por dia pra maioria dos casos.
>
> **Builds: Ilimitados, mas...**

```
[DESTAQUE: Builds]
```

> Você pode fazer quantos deploys quiser. Mas se seu build demora muito ou você deploya demais, pode entrar numa fila.
>
> Não é cobrança, só espera um pouco mais.

```
[TELA: Resumo dos limites]
```

> | Recurso | Limite Hobby |
> |---------|--------------|
> | Bandwidth | 100GB/mês |
> | Function Execution | 100GB-hours/mês |
> | Builds | Ilimitados |
> | Projetos | 200 |
> | Membros do time | Só você |

---

### BLOCO 3: O QUE NÃO TÁ INCLUÍDO (1.5 minutos)

```
[TELA: Lista com X vermelho]
```

**NARRAÇÃO:**

> O que você NÃO tem no Hobby:
>
> ❌ **Times/Colaboração**
> - Só você pode acessar o projeto
> - Quer adicionar outro dev? Precisa do Pro
>
> ❌ **Analytics Avançado**
> - O básico tem, mas insights detalhados são Pro
>
> ❌ **Suporte Prioritário**
> - Hobby usa comunidade e docs
> - Pro tem suporte por ticket
>
> ❌ **Password Protection Avançada**
> - Proteger previews com senha é Pro
>
> ❌ **SLA (Garantia de Uptime)**
> - Hobby não tem garantia formal
> - Na prática funciona bem, mas sem compromisso

```
[TELA: "O que realmente importa"]
```

> **Pra um portfolio ou projeto pessoal, o Hobby tem TUDO que você precisa.**
>
> Você só vai precisar do Pro quando:
> - Trabalhar em time
> - Ter tráfego muito alto
> - Precisar de suporte empresarial

[TELA: Exemplos de projetos]
