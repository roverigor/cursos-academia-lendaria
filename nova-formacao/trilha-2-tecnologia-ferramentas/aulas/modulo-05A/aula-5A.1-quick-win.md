## GPS

### Goal (30s)
Aprender a avaliar se uma proposta técnica faz sentido em 5 minutos — sem saber programar.

### Position (60s)
Você não precisa virar programador. Precisa saber identificar furada.

### Steps
1. Os 5 sinais de alerta (3 min)
2. As 5 perguntas matadoras (4 min)
3. Praticar com exemplo real (3 min)

## Os 5 Red Flags

### Sinais de que algo está errado

| Red Flag | O que significa | Exemplo |
|----------|-----------------|---------|
| **Prazo impossível** | Vai entregar mal ou não entregar | "Faço em 2 dias" para sistema complexo |
| **Preço muito baixo** | Código lixo ou abandono | R$500 para app completo |
| **Preço muito alto** | Over-engineering ou má-fé | R$50k para landing page |
| **Sem perguntas** | Não entendeu o projeto | Aceita tudo sem questionar |
| **Tecnicalês excessivo** | Enrolação ou insegurança | Usa 10 termos que você não entende |

### Regra de Ouro

```
Se você não entende a proposta → O problema é DELE, não seu.
```

Profissional competente explica para leigo entender.

## As 5 Perguntas Matadoras

### Faça SEMPRE antes de contratar:

**1. "Por que você escolheu essa tecnologia?"**
```
Bom: "React porque seu projeto precisa de interatividade e
      é a tecnologia que mais devs dominam, facilita manutenção"

Ruim: "Porque é o que eu uso" / "É a melhor" (sem explicar por quê)
```

**2. "O que acontece se você sumir?"**
```
Bom: "Código fica documentado no seu GitHub, qualquer dev
      consegue dar continuidade"

Ruim: "Fica comigo" / "Não vai acontecer" / Evita responder
```

**3. "Como vou saber se está funcionando?"**
```
Bom: "Vou entregar com testes automatizados e documentação.
      Você pode verificar em [ambiente de teste]"

Ruim: "Você vai ver quando estiver pronto"
```

**4. "O que está fora do escopo?"**
```
Bom: Lista clara do que NÃO está incluído

Ruim: "Fazemos tudo" / Não define limites
```

**5. "Posso falar com um cliente anterior?"**
```
Bom: Fornece contato ou portfolio verificável

Ruim: "Confie em mim" / Evita / Fica defensivo
```

## Exercício: Avalie Esta Proposta

### Proposta Recebida

```
Olá! Vi seu projeto de criar um marketplace.

Consigo fazer isso usando as melhores tecnologias do mercado
(Node.js, React, MongoDB, AWS). Tenho 5 anos de experiência e
entrego em 3 semanas por R$8.000.

O sistema vai ter cadastro de vendedores, produtos, carrinho,
pagamento e painel admin.

Me avisa se quiser fechar!
```

### Sua Análise

| Critério | Avaliação | Por quê? |
|----------|-----------|----------|
| Prazo realista? | [ ] Sim [ ] Não [ ] ? | |
| Preço coerente? | [ ] Sim [ ] Não [ ] ? | |
| Fez perguntas? | [ ] Sim [ ] Não | |
| Explicou escolhas? | [ ] Sim [ ] Não | |
| Definiu escopo? | [ ] Sim [ ] Não | |

### Resposta Ideal

```
Obrigado pela proposta! Algumas dúvidas:

1. Por que Node + React + Mongo para esse caso?
2. O que exatamente está incluído em "pagamento"?
   (gateway específico, split, etc.)
3. Pode compartilhar um projeto similar que fez?
4. Se surgir algum problema depois, como funciona suporte?

Aguardo retorno!
```

## O Framework RAP

Para avaliar qualquer proposta técnica:

| Letra | Significado | Verificar |
|-------|-------------|-----------|
| **R** | Realista | Prazo e preço fazem sentido? |
| **A** | Articulado | Explica bem o que vai fazer? |
| **P** | Profissional | Portfolio, referências, contrato? |

```
Se falhar em qualquer um → Sinal de alerta
Se falhar em dois → Próximo candidato
Se falhar nos três → Fuja
```
