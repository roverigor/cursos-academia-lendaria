# Aula 4.2: Antes, Durante e Depois da Call

> **Tipo:** Teoria Mínima
> **Duração:** 15 minutos
> **Conceito:** Sistema de Call em 3 fases

---

## O Problema (3 min)

Calls de vendas falham por:

| Fase | Erro Comum | Consequência |
|------|------------|--------------|
| **Antes** | Não preparar | Perde tempo descobrindo óbvio |
| **Durante** | Improvisar | Esquece perguntas-chave |
| **Depois** | Não documentar | Repete erros, perde insights |

### O Ciclo Vicioso

```
Call ruim → Não sabe por quê → Repete erro → Call ruim → ...
```

### O Ciclo Virtuoso

```
Prepara → Executa com estrutura → Analisa → Melhora → Call melhor → ...
```

---

## Fase 1: PRE-CALL (5 min)

### O que Levantar Antes

| Categoria | Informações |
|-----------|-------------|
| **Dados básicos** | Nome, cargo, empresa, faturamento |
| **Contexto** | Dor principal, interações anteriores |
| **Score** | Quente/morno/frio, qualificação |
| **Objetivo** | O que você quer ao final da call |

### Checklist Pre-Call

- [ ] Revisei histórico de interações
- [ ] Sei qual a dor principal
- [ ] Tenho 3 perguntas preparadas
- [ ] Previ 2 objeções prováveis
- [ ] Defini próximo passo ideal

### Prompt IA para Pre-Call

```
Com base nesses dados do lead:
[dados]

Prepare:
1. Objetivo da call (1 frase)
2. 3 perguntas-chave
3. 2 objeções prováveis + respostas
4. Próximo passo ideal
5. Armadilhas a evitar
```

---

## Fase 2: DURANTE - Script Adaptativo (5 min)

### Estrutura de 30 Minutos

| Fase | Tempo | O que fazer |
|------|-------|-------------|
| **Abertura** | 2 min | Contexto + permissão para perguntar |
| **Descoberta** | 10 min | Perguntas sobre dor e situação |
| **Apresentação** | 10 min | Conectar dor com solução |
| **Objeções** | 5 min | Ouvir, validar, responder |
| **Fechamento** | 3 min | Resumir + próximo passo |

### Frases-Chave

**Abertura:**
> "[Nome], obrigado por reservar esse tempo. Vi que você [contexto]. Posso fazer algumas perguntas para entender melhor?"

**Descoberta:**
> "Como funciona [processo] hoje?"
> "O que te fez buscar uma solução agora?"
> "De 0-10, qual a prioridade disso?"

**Objeções:**
> "Entendo sua preocupação com [X]. Posso te mostrar como resolvemos isso?"

**Fechamento:**
> "Resumindo: você precisa de [X] até [prazo]. O próximo passo seria [ação]. Funciona?"

## Fase 3: PÓS-CALL (2 min)

### O que Documentar

| Campo | Exemplo |
|-------|---------|
| **Diagnóstico** | "Dor real é falta de processo, não falta de leads" |
| **Objeções** | "Preço alto" → respondido com ROI |
| **Probabilidade** | 70% de fechar |
| **Próximo passo** | Enviar proposta até sexta |
| **Aprendizado** | Perguntar sobre decisor mais cedo |

### Prompt IA para Pós-Call

```
Resuma esta call:
[anotações ou transcrição]

Entregue:
1. Diagnóstico do lead (qual a real dor)
2. Objeções levantadas
3. Probabilidade de fechamento (0-100%)
4. Próxima ação recomendada
5. O que fazer diferente na próxima
```
