---

## Precificação por Valor

### Custo vs Valor

| Custo (errado) | Valor (certo) |
|----------------|---------------|
| "Quanto custa fazer?" | "Quanto vale resolver?" |
| Horas × Taxa | Problema × 10 |
| Margem baixa | Margem justa |

### Fórmula

```
Se o problema custa R$100k/ano para o cliente
E você resolve em 90 dias
Seu preço pode ser R$10-30k (10-30% do valor gerado)
```

**Regra**: Preço = 10-30% do valor que você gera

## Estratégia de 3 Pacotes

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│   ESSENCIAL  │  │ RECOMENDADO  │  │   PREMIUM    │
│              │  │  ★ MELHOR ★  │  │              │
│   Básico     │  │  Completo    │  │  Completo    │
│              │  │  + suporte   │  │  + suporte   │
│              │  │  + bônus     │  │  + exclusivo │
│   R$ 5.000   │  │  R$ 8.000    │  │  R$ 15.000   │
└──────────────┘  └──────────────┘  └──────────────┘
```

**Objetivo**: Maioria escolhe o do meio (ancoragem)

**Proporção**: Recomendado = 1.5x Essencial / Premium = 2-3x Essencial

---

## Apresentar vs Enviar

| Enviar (evitar) | Apresentar (preferir) |
|-----------------|----------------------|
| Cliente lê sozinho | Você guia a leitura |
| Vai direto ao preço | Constrói valor antes |
| Sem rebater objeções | Responde na hora |
| "Vou pensar" | Fecha na call |

**Regra**: Sempre que possível, apresente ao vivo.

---

## Resumo

1. Proposta tem 6 seções estruturadas
2. Precifique por valor, não por custo
3. Use 3 pacotes com ancoragem
4. Apresente ao vivo sempre que possível
