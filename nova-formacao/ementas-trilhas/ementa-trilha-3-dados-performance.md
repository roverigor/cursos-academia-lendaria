## MODULO 1: Mapa de Dados do Negocio (O Que Voce Tem vs O Que Precisa)

### Ficha Tecnica

| Atributo | Valor |
|----------|-------|
| **Duracao** | 90 minutos |
| **Bloom** | Nivel 4 - Analisar |
| **Formato** | Video (30min) + Build Sprint (60min) |

### Objetivo

Criar visibilidade total sobre **onde seus dados vivem** e **quais decisoes eles podem informar**.

### Principio Central

> **"Voce nao precisa de mais dados. Precisa saber ONDE estao os dados que ja tem e PARA QUE servem."**

### Conteudo (Estrutura de Aula)

| Etapa | Duracao | Conteudo |
|-------|---------|----------|
| **1. Contexto de Negocio** | 5 min | "Quantas fontes de dados voce tem? Se nao sabe, ja tem problema." |
| **2. Tipos de dados de negocio** | 10 min | Financeiro, comercial, operacional, cliente |
| **3. Onde dados vivem** | 10 min | Planilha, CRM, ERP, plataformas, banco |
| **4. Decisoes vs Dados** | 5 min | Qual decisao precisa de qual dado? |
| **5. Build Sprint** | 45 min | Criar Mapa de Dados |
| **6. Validacao** | 10 min | Checklist + gaps identificados |
| **7. Proxima acao 48h** | 5 min | Consolidar 1 fonte de dados |

### Entregavel Obrigatorio

**Mapa de Dados do Negocio**, contendo:

| Coluna | Descricao |
|--------|-----------|
| **Categoria** | Financeiro, comercial, operacional, cliente |
| **Dado** | Nome especifico (faturamento, leads, tickets) |
| **Fonte** | Onde vive (planilha X, CRM, plataforma Y) |
| **Frequencia** | Tempo real, diario, semanal, mensal |
| **Qualidade** | Confiavel, parcial, inexistente |
| **Decisao** | Qual decisao esse dado informa? |

### Template

```
| Categoria | Dado | Fonte | Freq | Qualidade | Decisao |
|-----------|------|-------|------|-----------|---------|
| Financeiro | Faturamento | Planilha | Mensal | Confiavel | Investir em mkt? |
| Financeiro | Custo fixo | ERP | Mensal | Confiavel | Cortar despesa? |
| Comercial | Leads/dia | CRM | Diario | Parcial | Aumentar trafego? |
| Comercial | Conversao | Planilha | Semanal | Parcial | Melhorar funil? |
| Cliente | Churn | Nenhum | - | Inexistente | Prevenir saida? |
| Operacional | Tickets | Zendesk | Tempo real | Confiavel | Contratar suporte? |

- [ ] Pelo menos 10 dados listados
- [ ] Fonte identificada para cada dado
- [ ] Gaps sinalizados (dados que faltam)
- [ ] Conexao dado â†’ decisao clara

### Prompt IA (Analise de Gaps)

```

[colar tabela]

## MODULO 3: Alertas Inteligentes (Saber Antes do Problema Virar Crise)

### Ficha Tecnica

| Atributo | Valor |
|----------|-------|
| **Duracao** | 90 minutos |
| **Bloom** | Nivel 5 - Avaliar |
| **Formato** | Video (30min) + Build Sprint (60min) |

### Objetivo

Criar **alertas automaticos** que avisam ANTES do problema virar crise.

### Principio Central

> **"Dashboard e retrovisqr - mostra o que JA aconteceu. Alerta e farol - avisa o que VAI acontecer."**

### Tipos de Alerta

| Tipo | Exemplo | Acao |
|------|---------|------|
| **Limite critico** | Faturamento < 80% da meta | Investigar imediato |
| **Tendencia negativa** | 3 dias seguidos de queda | Analisar causa |
| **Anomalia** | Churn 2x maior que normal | Acionar CS |
| **Oportunidade** | Lead quente parado 2 dias | Priorizar contato |
| **Lembrete** | Meta do mes em risco | Reuniao de ajuste |

### Conteudo (Estrutura de Aula)

| Etapa | Duracao | Conteudo |
|-------|---------|----------|
| **1. Contexto de Negocio** | 5 min | "Quando foi a ultima vez que voce descobriu um problema TARDE demais?" |
| **2. Dashboard vs Alerta** | 10 min | Retrovisqr vs farol |
| **3. Tipos de alerta** | 10 min | Limite, tendencia, anomalia, oportunidade |
| **4. Configurando alertas** | 5 min | n8n, Zapier, ou script simples |
| **5. Build Sprint** | 45 min | Criar 5 alertas criticos |
| **6. Testar** | 10 min | Simular trigger |
| **7. Proxima acao 48h** | 5 min | Conectar ao WhatsApp/Slack |

### Entregavel Obrigatorio

**Sistema de Alertas Inteligentes**, contendo:

| Componente | Descricao |
|------------|-----------|
| **5 alertas configurados** | Limite, tendencia, anomalia |
| **Trigger definido** | Condicao que dispara |
| **Canal de notificacao** | WhatsApp, email, Slack |
| **Acao sugerida** | O que fazer quando receber |
| **Responsavel** | Quem recebe e age |

### Template: Configuracao de Alertas

```
## ALERTAS CRITICOS

### 1. Faturamento Diario Baixo
- Trigger: Faturamento < R$ 3K (60% da media)
- Canal: WhatsApp do dono
- Acao: Verificar vendas do dia, analisar funil
- Responsavel: [Nome]

### 2. Conversao Caindo
- Trigger: Taxa < 7% por 3 dias seguidos
- Canal: Email + Slack comercial
- Acao: Revisar leads, verificar qualidade, checar concorrencia
- Responsavel: Gerente comercial

### 3. Churn Alto
- Trigger: Cancelamentos > 5 na semana
- Canal: WhatsApp CS
- Acao: Ligar para cancelados, identificar padrao
- Responsavel: CS Manager

### 4. Lead Quente Parado
- Trigger: Score > 80 sem contato ha 48h
- Canal: Notificacao CRM + WhatsApp SDR
- Acao: Contato imediato
- Responsavel: SDR do lead

### 5. Meta em Risco
- Trigger: Dia 20 com < 70% da meta
- Canal: Reuniao automatica agendada
- Acao: Revisao de estrategia, acoes de recuperacao
- Responsavel: Time completo
```

### Checklist de Validacao

- [ ] 5 alertas configurados
- [ ] Trigger claro e mensuravel
- [ ] Canal de notificacao funcionando
- [ ] Acao sugerida para cada alerta
- [ ] Testado com simulacao

### Prompt IA (Definir Alertas)

```
Minhas metricas principais:
[lista do dashboard]

Meus maiores riscos:
[lista de problemas que ja tive]

Me ajude a criar 5 alertas criticos:
1. Para cada alerta, defina trigger (condicao numerica)
2. Sugira frequencia de verificacao
3. Recomende canal de notificacao
4. Defina acao imediata ao receber
5. Indique prioridade (critico/importante/informativo)
```

## Sistema de Inteligencia de Dados

### Descricao

Ao final da trilha, o aluno tera um **sistema completo** para:
1. Saber onde seus dados estao e quais faltam
2. Visualizar metricas em tempo real
3. Receber alertas antes de problemas
4. Ter IA interpretando numeros
5. Tomar decisoes diarias com confianca

### Entregaveis do Projeto Final

| Entregavel | Origem | Criterio de Aprovacao |
|------------|--------|----------------------|
| Mapa de Dados | Modulo 1 | 4 categorias, 10+ dados, gaps identificados |
| Dashboard Automatizado | Modulo 2 | 5-7 metricas, 2+ automaticas |
| Sistema de Alertas | Modulo 3 | 5 alertas configurados e testados |
| Analista IA | Modulo 4 | Prompt master + 1 analise real |
| Rotina de Decisao | Modulo 5 | Checklist + 5 dias executados |
| **Relatorio de 1 semana** | Integracao | Decisoes tomadas com dados |

### Rubrica de Avaliacao

| Criterio | Peso | Aprovado | Parcial | Reprovado |
|----------|------|----------|---------|-----------|
| Completude | 30% | Todos entregaveis | 70%+ | <70% |
| Qualidade | 30% | Passa em todos checklists | 70%+ | <70% |
| Aplicabilidade | 20% | Sistema rodando | Parcialmente | So teoria |
