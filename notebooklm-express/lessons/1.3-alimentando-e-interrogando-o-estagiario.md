### PARTE 1: ANATOMIA DE UMA BOA PERGUNTA (1:00 - 3:30)

**[CENA: Screencast - interface de chat do NotebookLM]**

O NotebookLM tem um chat onde você conversa com suas sources. Mas nem toda pergunta funciona igual.

#### Perguntas FRACAS vs FORTES:

**❌ FRACA:** "O que tem nesse documento?"
- Muito genérico
- Resposta vaga e superficial

**✅ FORTE:** "Quais são os 3 principais argumentos do autor sobre [tema]?"
- Específico
- Pede estrutura (3 principais)
- Direciona para o que você quer

---

**❌ FRACA:** "Resuma isso"
- Genérico demais
- Não diz o foco

**✅ FORTE:** "Resuma as conclusões práticas deste documento em formato de bullet points"
- Define o foco (conclusões práticas)
- Define o formato (bullet points)

---

**❌ FRACA:** "Explica"
- Explica o quê?

**✅ FORTE:** "Explique o conceito de [X] mencionado na página 15, usando uma analogia simples"
- Específico (conceito X)
- Localizado (página 15)
- Define estilo (analogia simples)

### PARTE 2: OS 5 TIPOS DE PERGUNTA ESSENCIAIS (3:30 - 8:00)

**[CENA: Screencast com exemplos práticos]**

Existem 5 tipos de pergunta que cobrem 90% do que você vai precisar:

#### TIPO 1: EXTRAÇÃO (3:30 - 4:30)
**Objetivo:** Puxar informações específicas

**Exemplos:**
```
"Liste todas as estatísticas mencionadas no documento"

"Quais são os nomes das empresas citadas como exemplo?"

"Extraia os passos do processo descrito na seção 3"
```

**[DEMONSTRAR NO NOTEBOOKLM]**

---

#### TIPO 2: SÍNTESE (4:30 - 5:30)
**Objetivo:** Resumir e condensar

**Exemplos:**
```
"Resuma o argumento central em 3 frases"

"Quais são os pontos-chave deste capítulo?"

"Condense as recomendações do autor em uma lista de ações"
```

**[DEMONSTRAR NO NOTEBOOKLM]**

---

#### TIPO 3: COMPARAÇÃO (5:30 - 6:30)
**Objetivo:** Contrastar informações entre sources

**Exemplos:**
```
"Compare as abordagens de [Source A] e [Source B] sobre [tema]"

"Quais são as diferenças entre os modelos descritos nos dois artigos?"

"Onde os autores concordam? Onde discordam?"
```

**[DEMONSTRAR NO NOTEBOOKLM]**

---

#### TIPO 4: APLICAÇÃO (6:30 - 7:30)
**Objetivo:** Transformar conhecimento em ação

**Exemplos:**
```
"Como posso aplicar essas técnicas no meu [contexto]?"

"Crie um plano de ação baseado nas recomendações"

"Adapte essa estratégia para uma empresa de [segmento]"
```

**[DEMONSTRAR NO NOTEBOOKLM]**

---

#### TIPO 5: ANÁLISE CRÍTICA (7:30 - 8:00)
**Objetivo:** Avaliar e questionar

**Exemplos:**
```
"Quais são os pontos fracos do argumento apresentado?"

"O que está faltando nesta análise?"

"Quais premissas o autor assume sem provar?"
```

**[DEMONSTRAR NO NOTEBOOKLM]**

---

### PARTE 3: TÉCNICA DO FUNIL (8:00 - 10:00)

**[CENA: Screencast - sequência de perguntas]**

A técnica mais poderosa é o **Funil**: começar amplo e ir afunilando.

**Exemplo prático:**

**Pergunta 1 (Ampla):**
```
"Qual é o tema central deste documento?"
```
→ Resposta: "O documento trata de estratégias de precificação para SaaS"

**Pergunta 2 (Média):**
```
"Quais são as 3 estratégias de precificação mais recomendadas?"
```
→ Resposta: Lista as 3 estratégias

**Pergunta 3 (Específica):**
```
"Detalhe a estratégia de precificação baseada em valor. Quais os passos para implementar?"
```
→ Resposta: Guia detalhado

**Pergunta 4 (Aplicada):**
```
"Como aplicar essa estratégia em um SaaS de produtividade com ticket médio de R$ 97?"
```
→ Resposta: Recomendação personalizada

**[DEMONSTRAR TODA A SEQUÊNCIA NO NOTEBOOKLM]**

### PARTE 4: ERROS COMUNS (10:00 - 11:00)

**[CENA: Screencast com exemplos de erro]**

**ERRO 1: Perguntar sobre o que não está nas sources**
```
❌ "Qual é a previsão de mercado para 2025?"
(Se não há documento com essa informação)
```
→ NotebookLM vai inventar ou dizer que não sabe

**ERRO 2: Perguntas muito longas e confusas**
```
❌ "Considerando o contexto do mercado brasileiro e as tendências
globais, levando em conta também a situação econômica, qual seria
a melhor estratégia considerando todos os fatores mencionados nos
documentos e também o que não está escrito mas pode ser inferido?"
```
→ Quebre em perguntas menores

**ERRO 3: Não especificar o formato da resposta**
```
❌ "Me conta sobre precificação"
✅ "Liste 5 estratégias de precificação em formato de tabela com prós e contras"
```
