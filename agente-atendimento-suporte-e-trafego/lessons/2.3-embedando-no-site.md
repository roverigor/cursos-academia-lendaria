**[INSTRUTOR NA CÂMERA]**

>
> Agora é hora de colocar no seu site de verdade.

### [OPÇÃO 1: WORDPRESS] - 5 minutos

**[TELA: WordPress admin]**

> Vamos começar pelo WordPress, que é o mais comum.
>
> **[LOGA NO WORDPRESS]**
>
> Primeiro, entra no painel administrativo do seu site.
>
> **[MOSTRA PAINEL]**
>
> Agora, você tem algumas opções de onde colocar o código:
>
> 1. **Em todas as páginas** — usando um plugin ou o arquivo header/footer
> 2. **Em uma página específica** — usando um bloco de HTML
>
> Vou mostrar as duas formas.

---

#### [WORDPRESS - MÉTODO 1: PLUGIN] - 2 minutos

**[TELA: WordPress > Plugins]**

> Método mais fácil: usar um plugin de header/footer.
>
> Vai em Plugins > Adicionar novo.
>
> **[NAVEGA]**
>
> Busca por "Insert Headers and Footers" ou "WPCode".
>
> **[BUSCA]**
>
> Esse aqui: WPCode. Instala e ativa.
>
> **[INSTALA]**
>
> Agora vai em Code Snippets > Header & Footer.
>
> **[NAVEGA]**
>
> Cola o código do chat na seção "Footer".
>
> **[COLA]**
>
> Por que no footer? Porque o chat aparece no canto inferior da tela. Faz sentido carregar por último.
>
> Salva.
>
> **[SALVA]**
>
> Pronto! Agora abre o site em uma aba anônima.
>
> **[ABRE O SITE]**
>
> Olha ali no canto! O chat apareceu.
>
> Testa: manda uma mensagem.
>
> **[TESTA]**
>
> Funcionando!

---

#### [WORDPRESS - MÉTODO 2: BLOCO HTML] - 2 minutos

**[TELA: WordPress > Editar página]**

> Se você quer o chat só em uma página específica — tipo a página de vendas — faz assim:
>
> Vai em Páginas > Todas as páginas.
>
> **[NAVEGA]**
>
> Escolhe a página. Clica em Editar.
>
> **[ABRE EDIÇÃO]**
>
> Adiciona um novo bloco.
>
> **[CLICA NO +]**
>
> Procura por "HTML personalizado" ou "Custom HTML".
>
> **[BUSCA]**
>
> Arrasta esse bloco pro final da página.
>
> **[ARRASTA]**
>
> Cola o código do chat aqui dentro.
>
> **[COLA]**
>
> Atualiza a página.
>
> **[ATUALIZA]**
>
> Agora visualiza.
>
> **[VISUALIZA]**
>
> O chat aparece só nessa página. Legal pra segmentar.

---

### [OPÇÃO 2: SITE HTML PURO] - 3 minutos

**[TELA: Editor de código / VS Code]**

> Se você tem um site em HTML puro — sem WordPress — é ainda mais simples.
>
> Abre o arquivo HTML da sua página. Geralmente é `index.html`.
>
> **[ABRE O ARQUIVO]**
>
> Procura a tag `</body>`. É bem no final do arquivo.
>
> **[MOSTRA]**
>
> Cola o código do chat ANTES dessa tag.
>
> **[COLA]**
>
> ```html
> <!-- Chat de IA -->
> [código do chat aqui]
> </body>
> </html>
> ```
>
> Salva o arquivo.
>
> **[SALVA]**
>
> Agora abre o site no navegador.
>
> **[ABRE]**
>
> O chat está lá!
>
> **[MOSTRA]**
>
> Se você quer em todas as páginas, cola o código em todos os HTMLs. Ou, se tiver um arquivo de template comum, coloca lá.

---

### [OPÇÃO 3: ELEMENTOR/CONSTRUTORES] - 2 minutos

**[TELA: Elementor ou outro construtor]**

> Se você usa Elementor, Divi, ou outro construtor de páginas, a lógica é parecida:
>
> **[MOSTRA ELEMENTOR]**
>
> Procura um widget de HTML.
>
> No Elementor: arrasta o widget "HTML" pra página.
>
> **[ARRASTA]**
>
> Cola o código.
>
> **[COLA]**
>
> Atualiza.
>
> **[ATUALIZA]**
>
> Pronto!
>
> **[DICA]**
>
> Dica: alguns construtores têm uma seção de "Scripts personalizados" nas configurações. Você pode colocar lá pra aparecer em todas as páginas.

---

### [VERIFICANDO SE FUNCIONA] - 2 minutos

**[TELA: Site com o chat]**

> Depois de colocar o código, sempre testa:
>
> **[CHECKLIST]**
>
> 1. **O chat aparece?** Olha no canto da tela.
>
> 2. **Abre quando clica?** Clica no ícone do chat.
>
> 3. **Responde mensagens?** Manda um "oi" e espera.
>
> 4. **Em mobile também?** Abre no celular ou usa o modo responsivo do navegador.
>
> **[TESTA CADA UM]**
>
> Se tudo isso funciona, parabéns! Seu chat está no ar.

---

### [TROUBLESHOOTING] - 2 minutos

**[INSTRUTOR NA CÂMERA]**

> Alguns problemas comuns e como resolver:
>
> **Chat não aparece:**
> - Verifica se o workflow está ativo no N8N
> - Verifica se o Chat URL está correto
> - Limpa o cache do navegador
> - Verifica se o código foi colado corretamente (sem cortar nada)
>
> **Chat aparece mas não responde:**
> - O workflow provavelmente está inativo
> - Ou a API key do Groq expirou
> - Abre o N8N e verifica
>
> **Erro de CORS:**
> - Isso acontece quando o domínio não está autorizado
> - No N8N, verifica as configurações de CORS do chat
>
> **Chat demora muito pra responder:**
> - Normal na primeira mensagem (cold start)
> - Se demorar sempre, pode ser o servidor ou a LLM
>
> **[DIRETO PARA CÂMERA]**
>
> Se tiver outro problema, comenta na aula que eu ajudo.

## NOTAS DE PRODUÇÃO

### Recursos necessários:
- Acesso a um site WordPress de teste
- Código do chat gerado
- Screenshots de cada plataforma

### Pontos de verificação:
- [ ] Código colado no lugar certo
- [ ] Chat aparece no site
- [ ] Chat responde mensagens
- [ ] Funciona em mobile

### Erros comuns e soluções:
| Problema | Solução |
|----------|---------|
| Chat não aparece | Verificar workflow ativo, limpar cache |
| Não responde | Verificar API key, workflow status |
| Erro CORS | Configurar domínio no N8N |
| Lento | Normal no cold start |

- Prático e direto
- Solucionador nos problemas
- Celebrar quando funciona
