## 2. EMENTA

Introducao ao conceito de controle de versao e sua importancia para organizacao de projetos. Historia e contexto do Git e GitHub como ferramentas de colaboracao. Diferenca entre Git (ferramenta local) e GitHub (plataforma na nuvem). Instalacao e configuracao inicial do ambiente. Configuracao de conexao segura via SSH com chave Ed25519. Criacao de repositorios e estrutura de pastas para automacoes e documentos. Ciclo basico de trabalho: area de trabalho, area de preparacao e repositorio. Sincronizacao com a nuvem (push e pull). Boas praticas para mensagens de commit. Protecao de informacoes sensiveis com gitignore. Conceito de branches como versoes paralelas. Pull Requests como proposta formal de mudancas. Revisao de trabalho e feedback construtivo. Resolucao de conflitos em arquivos editados simultaneamente. Comandos para desfazer erros (restore, stash, reset, revert). Organizacao do perfil e projetos no GitHub. Integracao com outras ferramentas de produtividade.

---

## 3. OBJETIVO GERAL

Capacitar o aluno a utilizar Git e GitHub para organizar, versionar e colaborar em projetos de automacao, documentos e SOPs, transformando pastas desorganizadas em uma central estruturada com backup automatico e historico completo de mudancas.

---

## 4. OBJETIVOS ESPECIFICOS

Ao final do curso, o aluno sera capaz de:

1. Instalar e configurar Git no computador com conexao segura ao GitHub
2. Criar repositorios organizados com estrutura de pastas adequada
3. Executar o ciclo basico de trabalho: adicionar, salvar versao e sincronizar
4. Escrever mensagens de commit que explicam claramente as mudancas
5. Proteger informacoes sensiveis usando arquivos gitignore
6. Criar branches para testar mudancas sem afetar a versao principal
7. Abrir e revisar Pull Requests para colaboracao organizada
8. Resolver conflitos quando duas pessoas editam o mesmo arquivo
9. Recuperar versoes anteriores e desfazer erros com seguranca
10. Integrar o GitHub com outras ferramentas de trabalho
