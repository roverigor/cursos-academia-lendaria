# Analise de Reposicionamento: GitHub Essencial

## De "Devs Iniciantes" para "Empresarios"

---

> **Objetivo:** Adaptar o curso GitHub Essencial para o ICP da Nova Formacao Lendaria - empresarios que faturam R$ 50K-250K/mes e querem usar GitHub para gerenciar projetos de automacao/IA em suas empresas.

## 5. NOVOS EXEMPLOS PRATICOS

### Exemplos que Ressoam com Empresarios

| Contexto | Exemplo |
|----------|---------|
| **Automacao de vendas** | "Voce criou um fluxo n8n que qualifica leads. Semana que vem quer mudar a logica. Como voltar se der errado?" |

| **Backup inteligente** | "Seu computador morreu. Suas automacoes estavam onde?" |

```
minha-empresa-automacoes/
├── README.md           # Indice de tudo que existe aqui
├── automacoes/

│   ├── atendimento/    # Bots e agentes
│   └── marketing/      # Automacoes de conteudo
├── sops/
│   ├── onboarding.md   # SOP de onboarding
│   ├── atendimento.md  # SOP de atendimento
│   └── vendas.md       # SOP comercial
├── prompts/
│   ├── agente-vendas.md    # Prompt do agente de vendas
│   └── agente-suporte.md   # Prompt do agente de suporte
└── docs/
    └── decisoes.md     # Por que fizemos cada escolha
```

## 7. CHECKLIST DE VALIDACAO

### Cada Aula Deve:

- [ ] Comecar com contexto de NEGOCIO (nao carreira)
- [ ] Usar exemplos de AUTOMACOES/SOPs (nao codigo)
- [ ] Falar em RESULTADOS (nao portfolio)
- [ ] Conectar ao DRE: "Qual linha isso mexe?"
- [ ] Ter linguagem de EMPRESARIO (nao dev)
- [ ] Quick Win em 15 minutos ou menos
- [ ] Artefato implementavel em 48h

### Perguntas de Validacao:

1. Um empresario de 45 anos que nunca programou entenderia?
2. O exemplo faz sentido pra quem nao e dev?
3. A motivacao conecta com resultado de negocio?
4. A linguagem evita jargoes de programacao?
