# 1.1 - Git vs GitHub

**Duração:** 5 min | **Nível:** Iniciante

## Objetivo

Entender a diferença entre Git e GitHub e por que empresas de tecnologia exigem proficiência em ambos.

## Por que isso importa

Confundir Git com GitHub é como confundir um motor com uma concessionária. Entender a diferença te ajuda a navegar conversas técnicas com confiança e saber exatamente qual ferramenta usar em cada situação.

## Conteúdo

### Git: O Sistema de Controle de Versão

**O que é:** Software que roda no seu computador e rastreia mudanças no código.

**Pense assim:** Um super "Ctrl+Z" infinito que:
- Guarda todas as versões do seu código
- Permite voltar no tempo
- Funciona 100% offline
- Foi criado por Linus Torvalds (criador do Linux) em 2005

**Exemplo prático:**
```bash
# Você está no seu computador, offline
git init                 # Cria repositório Git local
git add README.md        # Marca arquivo para salvar
git commit -m "Initial"  # Salva versão
```

Tudo isso acontece **só no seu computador**. Git não precisa de internet.

### GitHub: A Plataforma de Colaboração

**O que é:** Site (github.com) que hospeda repositórios Git na nuvem e adiciona ferramentas de colaboração.

**Pense assim:** O Google Drive do código, mas com superpoderes:
- Armazena seu código na nuvem
- Permite colaboração em equipe
- Oferece Pull Requests, Issues, Actions
- Serve como seu portfólio profissional
- Foi comprado pela Microsoft em 2018 por $7.5 bilhões

**Exemplo prático:**
```bash
# Agora você precisa de internet
git push origin main  # Envia código local para GitHub
```

### A Relação Entre Eles

```
┌─────────────────────────┐
│   Seu Computador        │
│                         │
│   Git (local)           │
│   ├── commits           │

│   └── histórico         │
│                         │
└────────┬────────────────┘
         │ push/pull
         ▼
┌─────────────────────────┐
│   GitHub (nuvem)        │
│                         │
│   Repositório remoto    │
│   ├── backup            │
│   ├── colaboração       │
│   └── portfólio         │
│                         │
└─────────────────────────┘
```

### Por Que Empresas Exigem GitHub

**1. Colaboração distribuída**
- Times globais trabalhando no mesmo código
- Code review antes de aceitar mudanças
- Histórico completo de quem fez o quê

**2. Portfólio vivo**
- Recrutadores verificam seu GitHub antes da entrevista
- Contribuições open source contam como experiência
- Código público demonstra habilidades reais

**3. Workflow padronizado**
- Git Flow é padrão na indústria
- Pull Requests são universais
- CI/CD integrado (GitHub Actions)

**Dados do mercado:**
- 100+ milhões de desenvolvedores no GitHub (2024)
- 90% das empresas de tech usam Git
- "GitHub profile" é requisito em 70% das vagas junior

### Alternativas ao GitHub

Existem outras plataformas que hospedam Git:
- **GitLab** - Popular em empresas, tem CI/CD forte
- **Bitbucket** - Integra com Jira, usado em empresas
- **Gitea** - Self-hosted, open source

**Mas aprenda GitHub primeiro** porque:
- Maior comunidade
- Melhor para portfólio
- Mais vagas exigem
