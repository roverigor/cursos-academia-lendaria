### [O QUE É GIT] - 1 minuto

[SLIDE 1 - Título: "Git = Ferramenta Local"]

**[PROFESSOR DIZ:]**

> Git é uma ferramenta. Um programa que você instala no seu computador.
>
> Pensa assim: Git é um super "Ctrl+Z" infinito pro seu código.
>
> Ele guarda TODAS as versões de todos os seus arquivos. Você pode voltar pra qualquer ponto no tempo. E o mais importante: funciona 100% offline.
>
> [SLIDE 2 - Terminal mostrando comandos]
>
> Olha esses comandos aqui:
>
> `git init` - cria um repositório
> `git add` - marca arquivos
> `git commit` - salva versão
>
> Tudo isso acontece só no seu computador. Você pode estar num avião sem internet e usar Git normalmente.
>
> Git foi criado pelo Linus Torvalds em 2005, o mesmo cara que criou o Linux.

---

### [O QUE É GITHUB] - 1 minuto

[SLIDE 3 - Logo do GitHub]

**[PROFESSOR DIZ:]**

> GitHub é outra coisa. GitHub é um site. Uma plataforma na nuvem.
>
> Pensa assim: GitHub é o Google Drive do código.
>
> Ele armazena seus repositórios Git na internet. Permite que outras pessoas vejam seu código. Permite colaboração em equipe.
>
> [SLIDE 4 - Interface do GitHub]
>
> Quando você faz `git push`, você tá enviando do seu computador pro GitHub.
>
> Quando você faz `git pull`, você tá baixando do GitHub pro seu computador.
>
> Esses comandos, sim, precisam de internet. Porque você tá se comunicando com o servidor do GitHub.
>
> O GitHub foi comprado pela Microsoft em 2018 por 7.5 bilhões de dólares. Bilhões. Isso mostra o quanto essa plataforma é importante.

---

### [ANALOGIA DEFINITIVA] - 45 segundos

[SLIDE 5 - Diagrama Carro vs Concessionária]

**[PROFESSOR DIZ:]**

> Deixa eu te dar a analogia que vai fixar isso pra sempre:
>
> Git é o motor do carro. Funciona sozinho, faz o trabalho pesado.
>
> GitHub é a concessionária, oficina, e clube de motoristas - tudo junto.
>
> Você pode ter um motor sem concessionária. Git funciona sem GitHub.
>
> Mas a concessionária precisa ter carros. GitHub precisa do Git pra funcionar.
>
> [Pausa]
>
> Então quando alguém perguntar:
>
> "Git ou GitHub?"
>
> Você responde: "Git é a ferramenta local, GitHub é o serviço na nuvem."
>
> Pronto. Entrevista passada.

---

### [O DIAGRAMA] - 30 segundos

[SLIDE 6 - Diagrama Local vs Remoto]

**[PROFESSOR DIZ:]**

> Visualmente fica assim:
>
> No seu computador, você tem o Git rodando. Ele guarda seus commits, suas branches, todo o histórico.
>
> Na nuvem, você tem o GitHub. Ele guarda uma cópia do seu repositório, permite colaboração, serve como backup e portfolio.
>
> A comunicação entre eles acontece com `push` e `pull`.
>
> Push: envia pra cima. Pull: puxa pra baixo.
>
> Simples assim.
