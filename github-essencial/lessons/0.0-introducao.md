# AULA - BLOCO 2: A HISTÓRIA DO GIT (6 min)

## 03:00 - 04:30 | Linus Torvalds e Linux

[SLIDE 5 - Foto de Linus Torvalds jovem]

[TIMING] 1 minuto 30 segundos

[FALA]

> "Pra entender Git, você precisa conhecer esse cara: Linus Torvalds.
>
> Em 1991, Linus era um estudante finlandês de 21 anos. E ele criou o Linux - o sistema operacional que roda em 90% dos servidores do mundo. Sério. Quando você acessa Google, Netflix, Amazon, Instagram - tudo isso roda em Linux por baixo.
>
> [SLIDE 6 - Diagrama de Sistemas Operacionais]
>
> Windows é da Microsoft, pago e fechado. macOS é da Apple, pago e fechado. Linux é gratuito e aberto - qualquer pessoa pode ver o código, modificar, contribuir.
>
> E aí que começa o problema. Linux cresceu. Milhares de desenvolvedores no mundo todo queriam contribuir. Como você gerencia código de milhares de pessoas?"

---

## 04:30 - 06:00 | O Caos Antes do Git

[SLIDE 7 - Diagrama do fluxo de contribuições por email]

[TIMING] 1 minuto 30 segundos

[FALA]

> "De 1991 a 2002, Linus gerenciava tudo na raça.
>
> Desenvolvedores mandavam contribuições por email. Linus juntava tudo manualmente. Não tinha histórico organizado. Conflitos eram resolvidos na mão.
>
> Por onze anos, o cara fez isso. Heroico, mas insustentável.
>
> Em 2002, Linux adotou uma ferramenta chamada BitKeeper - um sistema de controle de versão proprietário, mas que liberou licença gratuita pro Linux.
>
> [SLIDE 8 - Timeline da Crise]
>
> Funcionou bem por 3 anos. Até que em abril de 2005, a empresa dona do BitKeeper revogou a licença gratuita. Simplesmente cortou o acesso.
>
> Linux, um dos maiores projetos de software do mundo, ficou sem ferramenta de controle de versão."

---

## 06:00 - 08:00 | Git Nasce em 10 Dias

[SLIDE 9 - "3 de Abril de 2005"]

[TIMING] 2 minutos

[FALA]

> "E o que Linus fez? Ele sentou e criou o Git.
>
> 3 de abril de 2005: começa a desenvolver.
>
> 7 de abril: primeiro commit - Git já gerenciando a si mesmo.
>
> 18 de abril: primeiro merge de múltiplas branches.
>
> 16 de junho: Git já gerencia o lançamento oficial do Linux Kernel.
>
> [Pausa]
>
> Em aproximadamente 10 dias, Linus criou o protótipo funcional de uma ferramenta que revolucionou o desenvolvimento de software no mundo todo.
>
> [SLIDE 10 - Citação de Linus]
>
> Ele disse uma vez: 'O truque não foi tanto a codificação, mas descobrir como organizar os dados.'
>
> E sobre o nome Git? Em gíria britânica, 'git' significa pessoa teimosa e desagradável. Linus tava fazendo piada consigo mesmo.
>
> Oficialmente, também pode ser Global Information Tracker. Mas a gente sabe que ele escolheu pelo humor."

---

## 08:00 - 09:00 | E o GitHub?

[SLIDE 11 - Logo GitHub + Timeline]

[TIMING] 1 minuto

[FALA]

> "Git existia desde 2005. Era poderoso, rápido, revolucionário. Mas era difícil de usar pra colaborar.
>
> Em 2008, três desenvolvedores criaram o GitHub. A ideia era simples:
>
> **Git** + **Hub**. Hub significa centro, ponto de encontro.
>
> Git é o motor do carro - roda local, no seu computador.
>
> GitHub é a concessionária, oficina, e comunidade de motoristas - tudo na nuvem.
>
> [SLIDE 12 - Crescimento do GitHub]
>
> 2012: 1 milhão de usuários.
>
> 2018: Microsoft compra por 7.5 bilhões de dólares. Bilhões. Com B.
>
> 2024: mais de 100 milhões de desenvolvedores.
>
> Quando alguém fala 'manda o link do GitHub', é tipo falar 'manda o link do Google Drive' - virou sinônimo de repositório de código."

---

## 10:00 - 12:00 | Passo a Passo

[TERMINAL]

[TIMING] 2 minutos

[FALA]

> "Abre o terminal aí. No Windows é Git Bash ou PowerShell. No Mac ou Linux, o terminal normal.
>
> Digita:
>
> ```bash
> git --version
> ```
>
> Se aparecer algo como 'git version 2.43.0', você tá pronto. Se der erro, a gente resolve na próxima aula.
>
> [Pausa]
>
> Agora, se você ainda não tem conta no GitHub, vai lá em github.com e cria. Usa um email profissional. E escolhe um username decente - nada de xX_gamer_Xx. Esse nome vai te acompanhar na carreira.
>
> [TERMINAL - Mostrar comandos]
>
> Agora vamos criar o repositório de prática:
>
> ```bash
> mkdir github-essencial-practice
> cd github-essencial-practice
> git init
> ```
>
> Isso cria uma pasta e inicializa o Git dentro dela.
>
> Agora cria um arquivo README:
>
> ```bash
> echo '# Minha Jornada GitHub Essencial' > README.md
> ```
>
> E faz seu primeiro commit:
>
> ```bash
> git add README.md
> git commit -m 'docs: initialize learning repo'
> ```
>
> Pronto. Você acabou de fazer seu primeiro commit. Isso é história. Guarda esse momento."

---

## 12:00 - 13:00 | Verificando

[TERMINAL]

[TIMING] 1 minuto

[FALA]

> "Pra confirmar que deu certo:
>
> ```bash
> git log --oneline
> ```
>
> Você deve ver algo assim:
>
> ```
> abc1234 docs: initialize learning repo
> ```
>
> Esse código no começo é o hash do commit - tipo um CPF único pra essa mudança.
>
> Se apareceu isso, parabéns. Você fez seu primeiro commit. Esse repositório vai ser sua base de prática pra todo o curso."

---

## 13:00 - 14:00 | O Que Você Acabou de Fazer

[SLIDE 14 - Resumo Visual]

[TIMING] 1 minuto

[FALA]

> "Vamos recapitular o que você fez:
>
> `git init` - Inicializou um repositório Git. Criou aquela pasta invisível .git que guarda todo o histórico.
>
> `git add` - Adicionou arquivo pra área de staging. É tipo colocar na fila pra foto.
>
> `git commit` - Tirou a foto. Salvou o estado atual do código com uma mensagem explicativa.
>
> Esses três comandos são o coração do Git. Você vai usar eles milhares de vezes na carreira."
