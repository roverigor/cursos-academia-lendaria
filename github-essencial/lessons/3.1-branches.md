## ROTEIRO COMPLETO

### [ABERTURA] - 30 segundos

**[FALA]**

> Voce ja quis testar uma mudanca grande em um documento, mas teve medo de perder a versao que funcionava?
>
> Com o Git, voce pode criar uma versao paralela para testar. Se der certo, voce integra. Se der errado, voce simplesmente descarta.
>
> Isso se chama "branch". Nesta aula, voce vai aprender a usar.

---

### [O QUE SAO BRANCHES] - 2 minutos

[SLIDE 1 - Diagrama de branches]

**[FALA]**

> Branch significa "ramo" em ingles. Pense como um galho de arvore que sai do tronco principal.
>
> No Git, a versao principal do seu projeto se chama "main". E o tronco da arvore - a versao estavel que voce sabe que funciona.
>
> Quando voce quer fazer uma mudanca grande ou testar algo novo, voce cria um "ramo" separado. Uma copia do projeto onde voce pode experimentar sem medo.
>
> [SLIDE 2 - Fluxo visual]
>
> Se a mudanca funcionar, voce junta o ramo de volta ao tronco. Isso se chama "merge" (juntar).
>
> Se nao funcionar, voce simplesmente apaga o ramo. A versao principal continua intacta.
>
> [Pausa]
>
> Por que isso e util?
>
> Imagine que voce tem um manual de processos que funciona bem. Voce quer reorganizar completamente as secoes.
>
> Em vez de editar o manual e correr o risco de piorar, voce cria uma branch. Faz todas as mudancas la. Se ficar melhor, integra. Se ficar pior, volta para a versao original.

---

### [QUANDO USAR BRANCHES] - 1 minuto 30 segundos

[SLIDE 3 - Quando usar]

**[FALA]**

> Voce deve criar uma branch quando:
>
> **Quer testar uma mudanca grande.**
>
> Reorganizar um documento, mudar a estrutura de pastas, testar uma nova abordagem.
>
> **Vai pedir ajuda para alguem.**
>
> Se voce contratou um freelancer ou tem alguem da equipe que vai trabalhar no projeto, essa pessoa deve trabalhar em uma branch separada.
>
> Assim voce pode revisar o trabalho antes de integrar na versao principal.
>
> **Quer manter a versao principal sempre funcionando.**
>
> Enquanto voce trabalha em mudancas, a versao main continua estavel. Se precisar voltar, ela esta la.
>
> [Pausa]
>
> A regra e simples: para mudancas pequenas (corrigir um erro de digitacao), edite direto. Para mudancas maiores, crie uma branch.

---

### [NOMES DE BRANCHES] - 1 minuto

[SLIDE 4 - Nomes de branches]

**[FALA]**

> Toda branch precisa de um nome que explique o que voce esta fazendo.
>
> Alguns exemplos bons:
>
> - `nova-tabela-precos` - para testar uma nova tabela de precos
> - `reorganiza-manual` - para reorganizar o manual
> - `adiciona-faq` - para adicionar uma secao de perguntas
> - `corrige-processos-venda` - para corrigir processos de venda
>
> [SLIDE 5 - Nomes ruins]
>
> Evite nomes genericos como:
>
> - `teste` - teste de que?
> - `branch1` - o que tem nessa branch?
> - `nova` - nova o que?
>
> [Pausa]
>
> O nome deve dizer o que voce esta trabalhando. Assim quando voce olhar a lista de branches, sabe o que cada uma faz.

---

### [COMANDOS BASICOS] - 2 minutos

[SLIDE 6 - Comandos de branch]

**[FALA]**

> Vou mostrar os comandos que voce vai usar.
>
> **Ver suas branches:**
>
> ```
> git branch
> ```
>
> Mostra todas as branches do projeto. O asterisco indica em qual voce esta.
>
> **Criar uma nova branch e entrar nela:**
>
> ```
> git checkout -b nome-da-branch
> ```
>
> O `-b` significa "criar nova branch". Esse comando cria e ja entra nela.
>
> **Mudar para outra branch:**
>
> ```
> git checkout main
> ```
>
> Isso volta para a versao principal.
>
> [SLIDE 7 - Publicar branch]
>
> **Enviar a branch para o GitHub:**
>
> ```
> git push -u origin nome-da-branch
> ```
>
> O `-u origin` configura a conexao com o GitHub. Depois da primeira vez, pode usar so `git push`.
>
> **Apagar uma branch que nao precisa mais:**
>
> ```
> git branch -d nome-da-branch
> ```
>
> Use isso depois que integrar as mudancas.

---

### [PRATICA] - 3 minutos

[SLIDE 8 - Pratica]

**[FALA]**

> Vamos praticar. Abra o terminal na pasta do seu projeto.
>
> Primeiro, veja as branches atuais:
>
> ```
> git branch
> ```
>
> Provavelmente so tem `main`.
>
> Agora crie uma branch nova:
>
> ```
> git checkout -b teste-nova-secao
> ```
>
> Veja que mudou:
>
> ```
> git branch
> ```
>
> O asterisco agora esta na `teste-nova-secao`.
>
> [Pausa]
>
> Faca uma mudanca nessa branch. Abra o README.md e adicione uma nova secao no final do arquivo.
>
> Salve a versao:
>
> ```
> git add .
> git commit -m "Adiciona nova secao para teste"
> ```
>
> Agora o interessante. Volte para a main:
>
> ```
> git checkout main
> ```
>
> Abra o README.md. A secao que voce adicionou nao esta la! Ela so existe na branch de teste.
>
> [Pausa]
>
> Volte para a branch de teste:
>
> ```
> git checkout teste-nova-secao
> ```
>
> Agora a secao aparece de novo.
>
> Isso e o isolamento. Suas mudancas ficam separadas ate voce decidir integrar.
>
> Envie a branch para o GitHub:
>
> ```
> git push -u origin teste-nova-secao
> ```
>
> Va ate o GitHub no navegador. Clique no botao que mostra "main" e voce vera sua branch na lista.

---

### [FECHAMENTO] - 30 segundos

[SLIDE 9 - Resumo]

**[FALA]**

> Recapitulando:
>
> - Branch = versao paralela do projeto
> - Use para mudancas grandes ou testes
> - `git checkout -b nome` cria e entra na branch
> - `git checkout main` volta para a versao principal
> - `git push -u origin nome` envia para o GitHub
>
> A partir de agora, quando quiser testar algo novo, crie uma branch primeiro.
>
> Na proxima aula, vamos aprender Pull Request - como propor suas mudancas para serem integradas na versao principal.
>
> Te vejo la.

## SE DER ERRO

### "Please commit your changes before switching branches"

Voce tem mudancas nao salvas. Salve primeiro:

```
git add .
git commit -m "Salva mudancas antes de trocar de branch"
```

Depois troque de branch.

### "Branch already exists"

Ja existe uma branch com esse nome. Use outro nome ou delete a existente:

```
git branch -d nome-existente
git checkout -b nome-existente
```

### Fiz mudanca na branch errada

Se ainda nao salvou a versao, simplesmente troque de branch e as mudancas vem junto:

```
git checkout branch-correta
git add .
git commit -m "Salva mudancas na branch correta"
```
