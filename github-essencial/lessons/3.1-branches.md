### [O QUE SÃO BRANCHES] - 2 minutos

[SLIDE 1 - Diagrama de branches]

**[PROFESSOR DIZ:]**

> Branch significa "ramo" em inglês. É uma linha independente de desenvolvimento.
>
> Pensa assim: a branch `main` é a rodovia principal. Sempre estável, sempre funcionando.
>
> Quando você vai construir algo novo, você sai pra uma via paralela. Faz seu trabalho lá.
>
> [SLIDE 2 - Fluxo visual]
>
> Quando termina, você retorna pra rodovia principal. Isso é o merge.
>
> ```
> main:     A---B---C-------M
>                \       /
> feature:        D---E---F
> ```
>
> Enquanto você trabalha na feature, a main continua estável. Outros devs podem continuar trabalhando. Ninguém se atrapalha.
>
> [Pausa]
>
> Por que isso importa?
>
> Em times reais, você tem 5, 10, 20 devs trabalhando ao mesmo tempo. Se todo mundo commitasse na main direto, seria impossível.

---

### [POR QUE USAR BRANCHES] - 1 minuto 30 segundos

[SLIDE 3 - Benefícios]

**[PROFESSOR DIZ:]**

> Três razões principais.
>
> **Primeira: proteção.** A main nunca quebra. Código incompleto fica na sua branch.
>
> **Segunda: trabalho paralelo.** Alice trabalha na feature de login. Bob trabalha no dashboard. Carol trabalha na API. Cada um na sua branch. Sem conflitos.
>
> **Terceira: code review.** Antes de integrar, outros devs revisam seu código. Isso acontece via Pull Request, que a gente vai ver na próxima aula.
>
> [Pausa]
>
> Regra de ouro: nunca commite direto na main. Sempre crie uma branch.
>
> Em entrevistas técnicas, se você não souber usar branches, é desqualificante.

---

### [NOMENCLATURA PROFISSIONAL] - 1 minuto

[SLIDE 4 - Convenções de nome]

**[PROFESSOR DIZ:]**

> Branch precisa de nome claro. O padrão é: `tipo/descrição`.
>
> Os tipos principais:
>
> **feature/** - funcionalidade nova. `feature/user-authentication`
>
> **fix/** - correção de bug. `fix/login-redirect-loop`
>
> **refactor/** - refatoração. `refactor/extract-auth-service`
>
> **docs/** - documentação. `docs/update-readme`
>
> [SLIDE 5 - Bom vs Ruim]
>
> **Ruim:** `branch1`, `test`, `my-branch`. Não diz nada.
>
> **Bom:** `feature/dark-mode-toggle`, `fix/navbar-mobile-overflow`.
>
> [Pausa]
>
> Use kebab-case. Letras minúsculas, separadas por hífen. Em inglês, que é o padrão da indústria.

### [COMANDOS BÁSICOS] - 2 minutos

[SLIDE 6 - Comandos de branch]

**[PROFESSOR DIZ:]**

> Os comandos que você vai usar todo dia.
>
> **Ver branches:**
>
> ```bash
> git branch
> ```
>
> Mostra suas branches locais. O asterisco indica qual você está.
>
> **Criar e mudar pra nova branch:**
>
> ```bash
> git checkout -b feature/minha-feature
> ```
>
> O `-b` cria a branch e já muda pra ela. Comando mais usado.
>
> **Mudar de branch:**
>
> ```bash
> git checkout main
> ```
>
> Volta pra main.
>
> [SLIDE 7 - Push da branch]
>
> **Publicar branch no GitHub:**
>
> ```bash
> git push -u origin feature/minha-feature
> ```
>
> O `-u` configura a conexão. Depois disso, pode usar só `git push`.
>
> **Deletar branch local:**
>
> ```bash
> git branch -d feature/minha-feature
> ```
>
> Só funciona depois que a branch foi mergeada.

### [EXERCÍCIO AO VIVO] - 3 minutos

[SLIDE 8 - Terminal ao vivo]

**[PROFESSOR DIZ:]**

> Vamos praticar. Abre o terminal no seu repo.
>
> Primeiro, veja as branches atuais:
>
> ```bash
> cd github-practice-repo
> git branch
> ```
>
> Provavelmente só tem `main`.
>
> Agora cria uma branch nova:
>
> ```bash
> git checkout -b feature/add-greeting
> ```
>
> Veja que mudou:
>
> ```bash
> git branch
> ```
>
> Agora o asterisco tá na `feature/add-greeting`.
>
> [Pausa]
>
> Faz uma mudança nessa branch:
>
> ```bash
> echo "function greet(name) { return 'Hello, ' + name; }" > greeting.js
> git add greeting.js
> git commit -m "feat: add greeting function"
> ```
>
> Agora a parte importante. Volta pra main:
>
> ```bash
> git checkout main
> ls
> ```
>
> O arquivo `greeting.js` não existe na main! Ele só existe na sua branch.
>
> [Pausa]
>
> Volta pra branch:
>
> ```bash
> git checkout feature/add-greeting
> ls
> ```
>
> Agora o arquivo aparece de novo.
>
> Isso é isolamento. Suas mudanças ficam na sua branch até você integrar.
>
> Agora publica:
>
> ```bash
> git push -u origin feature/add-greeting
> ```
>
> Vai no GitHub. Clica no dropdown de branches. Sua branch vai aparecer lá.

## TROUBLESHOOTING

### Erro: "Please commit your changes before switching branches"

> Você tem mudanças não commitadas.
> ```bash
> git stash
> git checkout outra-branch
> git stash pop  # Depois, para recuperar
> ```

### Erro: "Branch already exists"

> Branch já existe. Use outro nome ou delete:
> ```bash
> git branch -d nome-existente
> git checkout -b nome-existente
> ```

### Esqueceu de criar branch antes de commitar

> Se ainda não fez push:
> ```bash
> git branch nova-branch
> git reset --hard HEAD~1
> git checkout nova-branch
> ```
