### [INTEGRANDO COM NOTION] - 1 minuto 30 segundos

[SLIDE 2 - Notion + GitHub]

**[FALA]**

> Primeira integracao util: Notion.
>
> Se voce usa Notion para organizar tarefas e projetos, pode conectar com o GitHub.
>
> Como funciona:
>
> Voce pode criar links entre paginas do Notion e repositorios do GitHub. Quando atualiza um, o outro mostra.
>
> [SLIDE 3 - Casos de uso Notion]
>
> Usos praticos:
>
> - Ter uma pagina no Notion que lista todos os repositorios importantes
> - Acompanhar o que foi feito em cada projeto
> - Manter documentacao no Notion e arquivos no GitHub
>
> [Pausa]
>
> A integracao nao e automatica, mas voce pode manter links entre os dois sistemas.
>
> Alguns usam: Notion para planejamento, GitHub para versoes dos arquivos.

---

### [INTEGRANDO COM GOOGLE DRIVE] - 1 minuto

[SLIDE 4 - Google Drive + GitHub]

**[FALA]**

> Segunda integracao: Google Drive.
>
> Muita gente pergunta: "Posso usar os dois?"
>
> Sim, mas para coisas diferentes:
>
> **Google Drive:** Documentos que voce edita em colaboracao (Google Docs, Sheets).
>
> **GitHub:** Arquivos de texto puro, manuais, processos, qualquer coisa que precise de historico de versoes.
>
> [SLIDE 5 - Quando usar cada um]
>
> Regra simples:
>
> Se e documento que varias pessoas editam ao mesmo tempo → Google Drive
>
> Se e documento que voce quer historico completo de mudancas → GitHub
>
> [Pausa]
>
> Os dois podem trabalhar juntos. Voce pode ter links no GitHub que apontam para Google Docs, e vice-versa.

---

### [INTEGRANDO COM FERRAMENTAS DE GESTAO] - 1 minuto

[SLIDE 6 - Ferramentas de gestao]

**[FALA]**

> Terceira integracao: ferramentas de gestao de projetos.
>
> Trello, Asana, Monday - qualquer uma dessas pode se conectar com GitHub.
>
> A integracao mais simples: usar links.
>
> No cartao da tarefa, voce coloca o link do repositorio. No README do repositorio, voce coloca o link do projeto.
>
> [SLIDE 7 - Issues do GitHub]
>
> Ou voce pode usar o proprio GitHub para gestao basica.
>
> O GitHub tem uma funcao chamada "Issues" - sao tarefas dentro do repositorio.
>
> Cada Issue e uma tarefa. Voce pode marcar como feita, comentar, atribuir para alguem.
>
> Para projetos simples, pode ser suficiente.

[SLIDE 8 - Proximos passos]

>
> [Pausa]

> A habilidade melhora com pratica. Continue usando.
