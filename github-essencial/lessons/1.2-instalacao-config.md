### [POR QUE CONFIGURAÇÃO IMPORTA] - 1 minuto

[SLIDE 1 - Comparativo de commits]

**[PROFESSOR DIZ:]**

> Olha essa diferença aqui.
>
> Commit sem configuração:
>
> `Author: unknown <unknown@unknown.com>`
>
> Commit com configuração:
>
> `Author: Maria Silva <maria@email.com>`
>
> A diferença parece cosmética, mas não é.
>
> [SLIDE 2 - GitHub Profile]
>
> Se você fizer commits com o email errado ou sem email, essas contribuições NÃO aparecem no seu perfil do GitHub.
>
> Sabe aquele gráfico verde bonito que mostra sua atividade? Ele fica vazio. Recrutador olha e pensa: "esse cara não código nada".
>
> Mas você codou. Só configurou errado.
>
> Então vamos fazer certo desde o início.

### [INSTALAÇÃO - WINDOWS] - 1 minuto 30 segundos

[SLIDE 3 - Git for Windows]

**[PROFESSOR DIZ:]**

> Se você tá no Windows, segue esses passos.
>
> Primeiro, acessa git-scm.com/download/win. Baixa o instalador 64-bit.
>
> Durante a instalação, aceita quase tudo padrão. Mas atenção em três telas:
>
> [SLIDE 4 - Opções de instalação]
>
> **Editor:** escolhe VS Code se você tiver. Senão, Nano. Não escolhe Vim se você não sabe sair dele.
>
> **Branch inicial:** marca "main", não "master". Main é o padrão atual da indústria.
>
> **Terminal:** usa Git Bash. É o melhor pra trabalhar com Git no Windows.
>
> Depois de instalar, abre o Git Bash e digita:
>
> `git --version`
>
> Se aparecer algo tipo "git version 2.43.0", você tá pronto.

---

### [INSTALAÇÃO - MAC] - 45 segundos

[SLIDE 5 - macOS]

**[PROFESSOR DIZ:]**

> No Mac, a forma mais fácil é usar Homebrew.
>
> Se você ainda não tem Homebrew, vai em brew.sh e instala. Demora uns 2 minutos.
>
> Depois, no terminal:
>
> `brew install git`
>
> Pronto. Verifica com:
>
> `git --version`
>
> Se você não quer instalar Homebrew, pode usar o Xcode Command Line Tools:
>
> `xcode-select --install`
>
> Também funciona, mas o Homebrew é mais útil no longo prazo.

---

### [INSTALAÇÃO - LINUX] - 30 segundos

[SLIDE 6 - Linux]

**[PROFESSOR DIZ:]**

> Linux é mais direto ainda.
>
> Ubuntu ou Debian:
>
> `sudo apt update && sudo apt install git`
>
> Fedora:
>
> `sudo dnf install git`
>
> Verifica:
>
> `git --version`
>
> Provavelmente seu Linux já veio com Git instalado. Só verifica a versão e atualiza se for muito antiga.

---

### [CONFIGURAÇÃO ESSENCIAL] - 2 minutos

[SLIDE 7 - Terminal mostrando comandos]

**[PROFESSOR DIZ:]**

> Agora a parte crucial: configurar sua identidade.
>
> No terminal, digita esses comandos. Substitui pelos SEUS dados.
>
> ```bash
> git config --global user.name "Seu Nome Completo"
> git config --global user.email "seu@email.com"
> ```
>
> [Pausa]
>
> Atenção aqui: o email TEM que ser o mesmo que você cadastrou no GitHub.
>
> Se você usou maria@gmail.com no GitHub, usa maria@gmail.com aqui.
>
> Se usar email diferente, os commits não vão conectar com seu perfil.
>
> [SLIDE 8 - Dica de email privado]
>
> Dica pro: se você não quer expor seu email nos commits, o GitHub oferece email privado.
>
> Vai em Settings, depois Emails, marca "Keep my email addresses private".
>
> Ele gera um email tipo `seuusername@users.noreply.github.com`. Usa esse no git config.
>
> Seus commits ficam conectados ao perfil, mas ninguém vê seu email real.

---

### [CONFIGURAÇÕES EXTRAS] - 1 minuto

[SLIDE 9 - Mais configurações]

**[PROFESSOR DIZ:]**

> Agora mais algumas configurações importantes.
>
> **Editor padrão:** quando você precisa escrever mensagem de commit maior, Git abre um editor. Configura o VS Code:
>
> `git config --global core.editor "code --wait"`
>
> O `--wait` faz o Git esperar você fechar o arquivo.
>
> **Branch padrão:** configura pra main:
>
> `git config --global init.defaultBranch main`
>
> **Cores no terminal:** facilita a leitura:
>
> `git config --global color.ui auto`
>
> Pronto. Com isso você tá configurado do jeito profissional.

---

### [VERIFICANDO TUDO] - 1 minuto

[SLIDE 10 - git config --list]

**[PROFESSOR DIZ:]**

> Agora vamos conferir se deu tudo certo.
>
> Digita:
>
> `git config --list`
>
> Você deve ver algo assim:
>
> ```
> user.name=Seu Nome
> user.email=seu@email.com
> core.editor=code --wait
> init.defaultbranch=main
> color.ui=auto
> ```
>
> Se aparecer isso, você tá pronto.
>
> [Pausa]
>
> Pra ver só uma configuração específica:
>
> `git config user.name`
>
> Retorna só o nome.
>
> Dica: essas configurações ficam salvas no arquivo ~/.gitconfig. Se precisar editar manualmente, é lá.
