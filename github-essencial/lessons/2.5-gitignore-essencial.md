### [O QUE NAO GUARDAR] - 1 minuto

[SLIDE 1 - Lista do que nao guardar]

**[FALA]**

> Existem tipos de arquivos que nunca devem ir para o GitHub.
>
> **Arquivos com senhas ou chaves de acesso:**
>
> Se voce usa servicos como ChatGPT, n8n, Make, ou outros, provavelmente tem chaves de API ou senhas em algum lugar. Esses arquivos nao podem ir para a nuvem.
>
> **Documentos com dados pessoais:**
>
> Listas de clientes com CPF, contratos com valores, qualquer coisa que seria problema se vazasse.
>
> **Arquivos temporarios:**
>
> Arquivos que seu computador cria automaticamente, como .DS_Store no Mac ou Thumbs.db no Windows. Eles nao tem utilidade no projeto.
>
> [SLIDE 2 - Arquivos grandes]
>
> **Arquivos muito grandes:**
>
> Videos, apresentacoes pesadas, backups de banco de dados. O GitHub tem limite de tamanho, e esses arquivos pesam o repositorio.
>
> [Pausa]
>
> A regra e simples: se e sensivel ou muito grande, nao vai para o GitHub.

---

### [CRIANDO A PROTECAO] - 1 minuto 30 segundos

[SLIDE 3 - Arquivo .gitignore]

**[FALA]**

> O Git tem uma forma de proteger arquivos automaticamente. E o arquivo .gitignore.
>
> Esse arquivo lista tudo que o Git deve ignorar. Qualquer arquivo listado ali nao vai para o GitHub.
>
> Vamos criar um .gitignore basico.
>
> Abra o terminal na pasta do seu projeto e digite:
>
> ```
> echo "# Arquivos sensiveis" > .gitignore
> echo "senhas.txt" >> .gitignore
> echo "*.env" >> .gitignore
> echo "" >> .gitignore
> echo "# Arquivos do sistema" >> .gitignore
> echo ".DS_Store" >> .gitignore
> echo "Thumbs.db" >> .gitignore
> ```
>
> [Pausa]
>
> Deixa eu explicar:
>
> - Linhas que comecam com # sao comentarios (explicacoes)
> - `senhas.txt` ignora um arquivo especifico
> - `*.env` ignora todos os arquivos que terminam com .env
> - `.DS_Store` e `Thumbs.db` sao arquivos do sistema
>
> [SLIDE 4 - Verificando]
>
> Para verificar se funcionou, crie um arquivo de teste:
>
> ```
> echo "minha-senha-123" > senhas.txt
> git status
> ```
>
> O arquivo senhas.txt NAO deve aparecer na lista. Ele esta sendo ignorado.
>
> Agora salve o .gitignore:
>
> ```
> git add .gitignore
> git commit -m "Adiciona arquivo de protecao"
> git push
> ```
>
> Pronto. Seu repositorio esta protegido.

### [DICA: ARQUIVO DE EXEMPLO] - 1 minuto

[SLIDE 5 - Arquivo de exemplo]

**[FALA]**

> Quando voce tem um arquivo com senhas, e util criar um arquivo de exemplo.
>
> Por exemplo, se voce tem um arquivo chamado `config.txt` com senhas:
>
> ```
> API_OPENAI=sk-xxxxx-sua-chave-real
> SENHA_BANCO=sua-senha-secreta
> ```
>
> Voce pode criar um `config-exemplo.txt` sem as senhas reais:
>
> ```
> API_OPENAI=coloque-sua-chave-aqui
> SENHA_BANCO=coloque-sua-senha-aqui
> ```
>
> [Pausa]
>
> No .gitignore, voce coloca:
>
> ```
> config.txt
> ```
>
> O arquivo real com senhas fica protegido. O arquivo de exemplo vai para o GitHub e mostra quais informacoes sao necessarias.
>
> Assim, se outra pessoa precisar usar o projeto, ela sabe o que preencher.

## SE DER ERRO

### Arquivo ja foi enviado antes

Se voce enviou um arquivo sensivel antes de criar o .gitignore, precisa remove-lo:

```
git rm --cached nome-do-arquivo.txt
git add .gitignore
git commit -m "Remove arquivo sensivel e adiciona protecao"
```

O `--cached` remove do Git mas mantem o arquivo no seu computador.

### .gitignore nao funciona

Verifique:
- O nome do arquivo e exatamente `.gitignore` (com ponto no inicio)?
- O arquivo esta na pasta principal do projeto?
- O arquivo que voce quer ignorar ja foi adicionado antes?

## COMANDOS DA AULA (COPIAR E COLAR)

```bash
# Criar .gitignore basico
echo "# Arquivos sensiveis" > .gitignore
echo "senhas.txt" >> .gitignore
echo "*.env" >> .gitignore
echo "" >> .gitignore
echo "# Arquivos do sistema" >> .gitignore
echo ".DS_Store" >> .gitignore
echo "Thumbs.db" >> .gitignore

# Ver se arquivo esta sendo ignorado
git status

# Salvar o .gitignore
git add .gitignore
git commit -m "Adiciona arquivo de protecao"
git push

# Remover arquivo que ja foi enviado (mas manter no computador)
git rm --cached nome-do-arquivo.txt
```
