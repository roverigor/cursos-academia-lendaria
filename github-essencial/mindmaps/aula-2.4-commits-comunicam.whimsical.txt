Commits que Comunicam
	Por Que Mensagens Importam
		Historico ruim
			fix bug
			changes
			update
			WIP
			asdf
			Nao entende nada
		Historico profissional
			feat: add user authentication
			fix: resolve memory leak
			docs: update API documentation
			Sabe o que cada commit faz
		Beneficios
			Code review: revisor entende antes de ler
			Debug: sabe qual commit causou problema
			Recrutadores: olham seu historico
			Voce mesmo: daqui a meses vai agradecer
	Conventional Commits
		Estrutura
			tipo: descricao curta
		Tipos principais
			feat: funcionalidade nova
			fix: correcao de bug
			docs: documentacao
			refactor: reescrita sem mudar comportamento
			test: adicionar testes
			chore: manutencao, dependencias
		Exemplos
			feat: add dark mode toggle
			fix: resolve button alignment on mobile
			docs: add installation instructions
			refactor: simplify authentication flow
			test: add unit tests for login
			chore: upgrade dependencies
	Regras de Mensagem
		Titulo
			Maximo 50 caracteres
			Curto e direto
		Imperativo
			Add, Fix, Update
			NAO: Added, Fixed, Updated
		Sem ponto final
			Titulo nao e frase completa
		Especifico
			Fix navbar alignment on mobile
			NAO: Fix bug
		Regra mental
			If applied, this commit will...
			add dark mode toggle
	Bom vs Ruim
		Ruim: fix bug
		Bom: fix: resolve null pointer in profile
		Ruim: update
		Bom: docs: update installation for Windows
		Ruim: add feature, fix bugs, update readme
		Bom: deveria ser 3 commits!
		Se precisa de virgula ou "e"
			Provavelmente mais de um commit
	Commits Atomicos
		Significado
			Uma mudanca logica completa
		Por que importa
			Se precisar reverter
			Consegue reverter so uma coisa
		Bom: Commits separados
			feat: add user authentication
			docs: update README with auth instructions
			fix: correct typo in login form
		Ruim: Tudo junto
			Add auth, update readme, fix typos
		Cada commit
			Pode ser revertido independentemente
	Mensagem com Corpo
		Quando usar
			Precisa explicar mais
		Como fazer
			git commit (sem -m)
			Abre editor
		Formato
			Titulo
			Linha em branco
			Corpo explicando o porque
		Exemplo
			fix: resolve race condition in payment
			(linha em branco)
			Payment requests were processed twice...
			Added debouncing and idempotency check.
			Closes #245
		Closes #245
			Fecha automaticamente a issue
	Resumo
		Conventional Commits: tipo: descricao
		Tipos: feat, fix, docs, refactor, test, chore
		Imperativo, maximo 50 caracteres
		Um commit = uma mudanca logica
		Especifico, nao generico
