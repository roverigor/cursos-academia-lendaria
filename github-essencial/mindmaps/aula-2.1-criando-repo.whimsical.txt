Criando Sua Central de Projetos
	Por Que Organizacao Importa
		Duas pastas no computador
			Baguncada: arquivos jogados
			Organizada: estrutura clara
		Repositorio bem organizado
			Economiza tempo
			Sabe onde procurar
	Arquivos Iniciais
		README.md
			Apresentacao do projeto
			Primeira coisa que aparece
			Explica o que tem ali
			.md = Markdown (formato simples)
		.gitignore
			Diz ao Git o que ignorar
			Arquivos que NAO devem ser salvos
			Temporarios, senhas, configs pessoais
			Protege voce de subir algo errado
		LICENSE
			Regras de uso
			Quem pode usar, como pode usar
			MIT: pode usar, mantenha creditos
	Criando no GitHub
		github.com → login
		Clique no + → New repository
		Preencher
			Nome: simples, descritivo
			Letras minusculas, tracos
			Exemplos: meus-projetos, documentos-empresa
		Descricao
			Uma frase do que vai guardar
		Visibilidade
			Private: so voce ve
			Public: qualquer pessoa ve
		Opcoes para marcar
			Add a README file
			Add .gitignore (opcional)
		Create repository
	Baixando para o Computador
		Clonar = baixar repositorio
		No GitHub
			Botao verde Code
			Aba SSH (nao HTTPS)
			Copiar endereco (git@github.com:...)
		No terminal
			Navegar para pasta desejada
			git clone [endereco]
			Exemplo: git clone git@github.com:usuario/repo.git
		Entrar na pasta
			cd nome-do-repo
			ls (ver arquivos)
	Verificando Conexao
		Comando
			git remote -v
		Resultado esperado
			origin git@github.com:usuario/repo.git (fetch)
			origin git@github.com:usuario/repo.git (push)
		Se aparecer git@github.com
			Conexao segura (SSH) funcionando
		Se aparecer https://
			Clonou diferente, vai pedir senha
	Resumo
		Repositorio criado no GitHub
		Arquivos basicos entendidos
		Projeto baixado no computador
		Conexao verificada
