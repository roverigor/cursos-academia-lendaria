Salvando Versoes (O Ciclo Basico)
	As 3 Areas do Git
		Area 1: Pasta de trabalho
			Pasta do projeto no computador
			Onde voce edita arquivos
		Area 2: Area de preparacao
			Caixa para arquivos que quer salvar
			Voce decide o que entra
		Area 3: Historico
			Versoes salvas permanentemente
			Fica para sempre
		Analogia
			Pasta de trabalho = Mesa com papeis
			Area de preparacao = Envelope
			Historico = Arquivo guardado
		Processo
			Editar → Preparar → Salvar
	Comando 1: git status
		O que faz
			Mostra situacao dos arquivos
			Raio-x do projeto
		Mensagens comuns
			Changes not staged for commit
				Arquivos modificados
				Precisa usar add
			Untracked files
				Arquivos novos
				Git nao conhece ainda
			Changes to be committed
				Prontos para salvar
			nothing to commit, working tree clean
				Tudo ja foi salvo
		Quando usar
			Sempre que quiser saber o status
	Comando 2: git add
		O que faz
			Prepara arquivos para salvar
			Move para area de preparacao
		Arquivo especifico
			git add nome-do-arquivo.txt
		Todos os arquivos
			git add .
			Ponto = todos desta pasta
		Desfazer add por engano
			git restore --staged nome-do-arquivo.txt
			Remove da preparacao
			Nao apaga mudancas
	Comando 3: git commit
		O que faz
			Salva a versao no historico
		Formato
			git commit -m "sua mensagem aqui"
			-m = message (mensagem)
		Mensagens
			Descricao do que mudou
			Fica no historico
			Exemplos bons
				Atualiza documento de processos
				Adiciona secao de perguntas frequentes
				Corrige erro no calculo
			Evitar mensagens vagas
				alteracoes
				update
	O Ciclo Completo
		1. git status
			Ver situacao atual
		2. Editar arquivos
			Fazer mudancas normalmente
		3. git status
			Ver o que mudou
		4. git add .
			Preparar arquivos
		5. git status
			Confirmar preparacao
		6. git commit -m "mensagem"
			Salvar versao
		7. git status
			Confirmar que salvou
		Resumo
			status, add, commit
			Com tempo fica automatico
	Ver Historico
		Comando
			git log --oneline
		O que mostra
			Uma linha por versao
			Mensagem que voce escreveu
	Checklist
		Entendi as 3 areas
		Sei usar git status
		Sei usar git add
		Sei usar git commit
		Fiz pelo menos 3 commits
