Pedindo Revisao (Pull Request)
	O Que e Pull Request
		Proposta de mudanca
		Pede revisao antes de integrar
		Mostra o que voce fez
		Outra pessoa aprova ou pede ajustes
	Por Que Usar
		Evitar erros
			Alguem revisa seu trabalho
			Pega problemas antes de integrar
		Documentar mudancas
			Historico do que foi feito
			Por que foi feito
		Colaborar
			Mesmo trabalhando sozinho
			Voce futuro vai agradecer
	Criando um Pull Request
		Ter branch com mudancas
			Ja enviada para GitHub (git push)
		No GitHub
			Botao verde "Compare & pull request"
			Ou: Pull requests â†’ New pull request
		Escolher branches
			base = onde quer integrar (main)
			compare = sua branch com mudancas
	Escrevendo Boa Descricao
		Estrutura simples
			O que mudou
			Por que mudou
			O que foi alterado
		Exemplo
			Titulo: Adiciona secao de perguntas frequentes
			Descricao:
				O que mudou: Adicionei uma nova secao ao manual
				Por que: Clientes sempre perguntam as mesmas coisas
				O que foi alterado: README.md
		Dica
			Voce mesmo vai agradecer depois
			Quando olhar o historico
	Depois de Criar
		Esperar revisao
			Outra pessoa vai olhar
			Pode fazer comentarios
			Pode pedir ajustes
		Se pedirem mudancas
			Faca as correcoes na sua branch
			git add, git commit, git push
			Mudancas aparecem automaticamente no PR
	Aprovacao e Merge
		Quando aprovarem
			Botao verde "Merge pull request"
			Clique e confirme
		Depois do merge
			Mudancas vao para main
			Pode apagar a branch
			GitHub oferece essa opcao
	Exercicio
		Criar branch nova
			git checkout -b minha-mudanca
		Fazer mudanca
			Editar README.md
			Adicionar informacao
		Salvar e enviar
			git add .
			git commit -m "Adiciona nova informacao"
			git push -u origin minha-mudanca
		Criar Pull Request
			Ir para GitHub
			Clicar em Compare & pull request
			Preencher titulo e descricao
			Clicar em Create pull request
		Revisar e aprovar
			Aba Files changed (ver mudancas)
			Botao Merge pull request
		Atualizar local
			git checkout main
			git pull
	Resumo
		Pull Request = proposta de mudanca
		Crie para suas branches antes de integrar
		Escreva descricao clara: o que, por que, o que mudou
		Espere revisao, faca ajustes se pedirem
		Merge quando aprovado
