Sincronizando com a Nuvem
	Computador vs Nuvem
		Duas copias do projeto
			Local: seu computador
			Remoto: GitHub (nuvem)
		Commit fica so no computador
			GitHub ainda nao sabe
		Sincronizacao
			Push: empurrar para nuvem
			Pull: puxar da nuvem
	Por Que Sincronizar
		Backup automatico
			Computador para de funcionar
			Arquivos seguros no GitHub
		Acesso de qualquer lugar
			Trabalhar de outro computador
		Colaboracao
			Outras pessoas veem atualizacoes
	git push (Enviar)
		O que faz
			Envia versoes para o GitHub
		Comando basico
			git push
		Primeira vez
			git push -u origin main
			-u configura conexao
			origin = nome do GitHub
			main = linha principal
		Depois da primeira vez
			Apenas: git push
		Quando funciona
			Writing objects: 100%
			main -> main
		Verificar
			Abrir GitHub no navegador
			Atualizar pagina
			Mudancas aparecem
	git pull (Receber)
		O que faz
			Baixa atualizacoes do GitHub
		Comando
			git pull
		Quando usar
			Editou algo no site do GitHub
			Outra pessoa fez mudancas
			Garantir versao mais recente
	Boas Praticas
		Pull no comeco do dia
			Comeca com versao atualizada
		Push no final do dia
			Mudancas ficam na nuvem
			Mesmo se nao terminou
		Habito
			Pull de manha, push a noite
			Protege seu trabalho
	Se Der Problema
		Erro no push
			GitHub tem mudancas que voce nao tem
			Solucao
				git pull
				git push
		Erro no pull
			Voce tem mudancas nao salvas
			Solucao
				git add .
				git commit -m "Salva mudancas"
				git pull
		Regra geral
			Se der erro no push
			Faca pull primeiro
	Fluxo Completo
		1. git add .
		2. git commit -m "mensagem"
		3. git push
		Editar no GitHub
		4. git pull
		Sincronizado
	Resumo
		git push = envia para GitHub
		git pull = baixa do GitHub
		Pull no comeco, push no final
		Arquivos seguros na nuvem
